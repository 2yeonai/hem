# MASTER_SETUP — 어떤 계정/모델에서도 이 시스템을 복원하는 법

> 전제: 이 볼트 폴더(hem)만 있으면 된다. 모델·계정·앱이 바뀌어도 **판단력은 파일에 저장돼 있고**, 새 모델은 이 파일들을 읽고 따라가기만 하면 된다.

## 0. 복원 순서 (새 환경에서 이대로)

1. 새 환경(Claude 데스크탑 Cowork / Claude Code / 타사 도구)에서 이 볼트 폴더를 작업 폴더로 선택.
2. 첫 프롬프트로 아래를 입력:
   ```
   이 폴더의 CLAUDE.md → 2. Areas/핵심맥락.md → ai공장짓기/CLAUDE.md →
   ai공장짓기/HANDOFF.md → 1. Projects/완전자동화_실행계획.md 순서로 읽고,
   내가 누구인지·무엇을 만들고 있는지·지금 어느 단계인지 요약해서 확인시켜줘.
   틀린 부분을 내가 고쳐주면 그때부터 작업 시작.
   ```
3. 요약이 맞으면 → `1. Projects/완전자동화_실행계획.md`의 현재 단계부터 재개.
4. 예약 작업(자동 기록)은 환경마다 다시 등록해야 한다 → 아래 4번 섹션의 프롬프트 사용.

## 1. 이 시스템의 "판단력"이 저장된 위치 (읽기 순서대로)

|---|---|

| `2. Areas/핵심맥락.md` | 내가 누구인지(mo,on 대표+AI공장 설계자), 4개 공장 전체 그림·현황 |

## 2. 절대 규칙 (모델·도구가 바뀌어도 유지)

1. 승인완료 아닌 문서는 발송 금지. 지원사업 접수·고객 발송 등 되돌릴 수 없는 행동은 항상 사람이.
2. 모델명 하드코딩 금지 — 티어(low_cost/mid/high)로만.
3. 완료 선언 = 증거 붙여넣기 (테스트 출력, diff, 검증 로그). quality_gate 없으면 미완성.
4. 큰 파일(대략 5KB 이상) 수정 시: 원본을 `.stale-타임스탬프`로 이름 변경 → bash heredoc으로 재작성 → `py_compile`/`yaml.safe_load`로 검증. (이 환경의 파일 잘림 버그 우회)
5. raw 원본(골든셋 원자료, 과거기록)은 수정 금지 — 컴파일된 노트만 갱신.
6. 크레딧·한도 소진 시: 어디까지 했는지 HANDOFF에 기록 후 중단. 재개는 기록 지점부터.

## 3. 모델 역할표 (다른 회사 모델로 바꿔도 티어로 매핑)

| 티어 | 현재 매핑 | 하는 일 |
|---|---|---|
| high (설계) | Fable 5 / Opus | 새 구조 결정, 설계 충돌 해소, 주간 종합. **사전 승인 필수** |
| mid (실행) | Sonnet | 문서 작성, 코드 작성·실행, 테스트, 인터뷰 진행 — 기본값 |
| low_cost (분류) | Haiku | 태그, 분류, 이름 추출, 일일 로그 컴파일 |


2. 받은 파일을 `2. Areas/Claude 세션로그/과거기록/`에 넣기 (수정 금지).
3. 프롬프트: "과거기록 폴더를 읽고 대화별 날짜/주제/결정사항/미해결을 날짜별 노트로 컴파일. 원본 수정 금지."

## 6. 컴퓨터를 바꿀 때 (이사 절차, 순서대로)

1. **볼트 복사**: `hem` 폴더를 통째로 외장하드/USB에 복사 (`.obsidian`, `.git` 포함). 클라우드 동기화로 옮길 거면 **동기화 시스템은 하나만** — 에이전트가 쓰는 도중 iCloud 등이 동시에 돌면 충돌 사본이 생긴다.
2. **새 컴퓨터 설치**: Obsidian + Claude 데스크탑 앱 설치 → 같은 계정으로 로그인.
3. **볼트 열기**: Obsidian에서 "Open folder as vault"로 복사한 hem 폴더 열기.
4. **Cowork 연결**: Cowork에서 작업 폴더로 hem 선택 → 0번 섹션의 복원 프롬프트 실행.
5. **예약 작업 재등록**: 예약 작업은 컴퓨터에 저장되므로 계정을 유지해도 새 컴퓨터에선 다시 등록해야 한다 → 4번 섹션 프롬프트 실행.
6. **커넥터 확인**: 커넥터(Gmail 등)는 계정에 붙어 있어 로그인하면 따라온다. 안 보이면 설정 > Connectors에서 재연결.
7. **검증**: "2. Areas/Claude 세션로그/의 최신 노트를 읽고 마지막 작업 지점을 말해줘"가 정상 동작하면 이사 완료.
