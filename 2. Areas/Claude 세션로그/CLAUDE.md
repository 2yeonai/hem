# 2. Areas/Claude 세션로그 — 폴더 규칙

## 목적과 역할
Claude(Cowork·Code·채팅) 세션의 작업 기록을 날짜별로 축적하는 지속 관리 영역.
세션·크레딧이 끊겨도 이 폴더만 읽으면 끊긴 지점부터 재개할 수 있게 하는 것이 존재 이유.
Areas에 두는 이유: 마감 없는 지속 책임(매일 기록)이기 때문. 프로젝트별 통찰은 여기 두지 않고 승격시킨다(아래 참조).

## 들어가는 노트
- `YYYY-MM-DD.md` — 일일 세션 요약 (예약 작업이 매일 22:30 자동 생성)
- `주간종합_YYYY-MM-DD.md` — 주간 종합 (예약 작업이 일요일 21:00 자동 생성)
- `과거기록/` — claude.ai 내보내기 원본. **읽기 전용, 절대 수정·삭제 금지**

## 일일 노트 형식 (자동 생성 시 이 형식을 따를 것)
```
---
type: session-log
date: YYYY-MM-DD
tags: [area/claude-log]
---
# YYYY-MM-DD 세션 기록
## 진행한 작업 (세션별)
- (세션 주제): 한 일 / 사용 모델 / 산출물 경로
## 결정사항
- (있으면. 설계 결정이면 [[decision-log_skill-factory-architecture]]에도 반영 지시)
## 미해결 / 다음에 이어서
- (끊긴 지점, 다음 행동)
```

## 작업 방식과 규칙
1. 원본(과거기록/)은 수정 금지. 컴파일된 날짜 노트만 갱신한다.
2. 같은 날짜 노트가 이미 있으면 새로 만들지 말고 그 노트에 추가한다.
3. 세션이 없었던 날은 노트를 만들지 않는다 (빈 노트 금지).
4. 재사용 가치가 있는 통찰(한 개 아이디어 단위)은 `5. Zettelkasten/00. Inbox/`로 승격시키고, 여기엔 링크만 남긴다.
5. 관련 공장 폴더가 있으면 `[[클로드 방역 ai]]` 식으로 링크를 건다.
6. 이 폴더의 노트는 사실 기록이지 해석이 아니다 — 과장·추측 없이 한 일만 적는다.
