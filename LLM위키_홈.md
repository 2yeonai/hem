# 🧠 LLM위키 홈 — 혜미의 뇌 지도

> 지시 10·11번으로 신설. 이 노트는 "기록 목록"이 아니라 **기록들 사이의 관계(왜 연결되는가)**를 보여주는 홈이다.
> Obsidian에서 이 노트를 열고 우측 그래프 뷰(로컬 그래프)를 켜면 아래 구조가 실제로 보인다.

## 한 장 요약 — 모든 기록의 인과 사슬

```
[정체성] mo,on 대표 (산후회복 서비스, 진주)
    │  "내 사업에 필요해서 직접 만들었다"
    ▼
[1차 공장] 정부지원사업 매칭 ──재사용──▶ [2차] 꽃집 ──재사용──▶ [3차] 방역
    │                                      (설계 재사용률 64~86%)
    ▼  세 공장의 공통 뼈대를 추출
[플랫폼] 범용 러너 (ai공장짓기/runner) ◀─ 공통 계약: manifest·quality_gate·human게이트
    │  러너 위에 신축
    ▼
[4차 공장] 콘텐츠 (블로그·스레드) ──확장──▶ [마케팅 앱 지도] ──고객채널──▶ [카톡·알림봇]
    │
    ▼
[수익화] 가게 홍보 → 고객 유입 → mo,on 매출  ← 결국 출발점(정체성)으로 되돌아옴
```

**이 볼트의 단 하나의 문장**: "혜미는 자기 사업을 굴리기 위해 만든 자동화를, 같은 계약 언어로 일반화해 다른 소상공인에게 파는 플랫폼을 짓는 중이다."

## 지금 맥락이 얇은 곳 (지시 11번 "맥락이 적어 보임"에 대한 답)

1. **꽃집** — 설계는 있는데 "왜 12봇→14봇인지" 결정 서사가 흩어져 있음 → 꽃집 착수 시 decision-log 방식 소급 적용 예정.
2. **mo,on 사업 자체** — 공장 기록은 두꺼운데 정작 본업(서비스 설계, 고객, 가격)의 노트가 얇음 → 2. Areas에 "mo,on 사업" 영역 노트 신설 권장 (혜미 결정 대기).
3. **Zettelkasten 20. Permanent** — 공장에서 얻은 통찰("승인은 발송 직전에 재검사한다" 같은 원칙)이 아직 영구노트로 승격 안 됨 → 주간 예약작업에 "이번 주 통찰 1개 승격" 추가 권장.
