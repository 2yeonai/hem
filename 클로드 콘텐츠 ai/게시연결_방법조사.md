# 게시 연결 방법 조사 (2026-07-12, 웹 조사 기반)

> 혜미 지시: "다른 사람들은 다 하던데, 방법까지 찾아오고 연결이 필요하면 나한테 요청해"
> 결론 먼저: **셋 다 가능. 시작은 Chrome 게시 대행 하나로 통일하고, 스레드만 나중에 공식 API로 승격하면 예약발행까지 완전 자동이 된다.**

## 채널별 조사 결과

| 채널 | 공식 API | 실제로 다들 쓰는 방법 | 우리 선택 |
|---|---|---|---|
| **Threads** | ✅ 있음 (Meta Threads API) — 게시물 발행 2단계(컨테이너 생성→발행), 개발모드 앱이면 본인 계정은 심사 없이 사용 가능. 토큰 60일마다 갱신 필요 | 공식 API로 예약발행 (Claude/코드와 연결해 몰아서 점검 후 자동 발행하는 사례 다수) | 1단계: Chrome 대행 → 2단계: 공식 API 승격(예약발행) |
| **네이버 블로그** | ❌ 없음 — 글쓰기 API 미제공(구 API는 네아로+심사 필요했고 사실상 막힘) | 로그인된 브라우저 자동화(다들 이 방식) | Chrome 게시 대행 (승인 후) |
| **티스토리** | ❌ 종료 — Open API 2024년 완전 종료 | 로그인된 브라우저 자동화 | Chrome 게시 대행 (승인 후) |

## "월/수/금 오후 5시 자동 발행"의 현실적인 단계

1. **지금 (1단계)**: 예약작업이 오후 5시 전에 초안을 준비 → 혜미가 승인 → 같은 세션에서 Claude가 Chrome으로 3채널 게시. 컴퓨터가 켜져 있고 Chrome 확장이 연결돼 있어야 함.
2. **나중 (2단계, 스레드만)**: Meta 개발자 앱 + 토큰을 만들면 스레드는 승인된 글을 컴퓨터 앞에 없어도 API로 자동 발행 가능. 네이버·티스토리는 API가 없어서 계속 Chrome 방식.
3. 어느 단계든 **승인 없는 게시는 공장 구조상 불가능** (게시봇 이중방어).

## 혜미에게 요청하는 연결 (이것만 해주면 됨)

| # | 요청 | 소요 | 왜 필요한지 |
|---|---|---|---|
| 1 | **Claude Chrome 확장 연결** (Chrome 열고 확장 로그인) | 2분 | 게시 대행 + 블로그·스레드 골든셋 읽기 전부 이걸로 함. **가장 중요** |
| 2 | Chrome에서 네이버·스레드 **로그인 상태 유지** | 1분 | 로그인된 창에서만 게시 가능 (비밀번호는 Claude가 다루지 않음 — 로그인은 항상 혜미가) |
| 3 | **티스토리 개설** (아이디 생성) | 5분 | 계정 생성은 보안상 Claude 대행 불가. 개설 절차는 화면으로 안내 가능 |
| 4 | (나중, 선택) Meta 개발자 앱 + Threads 토큰 발급 | 20분 | 스레드 완전 자동 예약발행용. 첫 실전 몇 건 해본 뒤에 해도 늦지 않음 |
