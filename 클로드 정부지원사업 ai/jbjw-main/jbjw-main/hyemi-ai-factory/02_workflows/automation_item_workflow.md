# automation_item_workflow.md — E라인: 업무자동화 아이템 설계 공장 워크플로우

목적: 반복 수작업 업무를 "입력자료 → 처리과정 → 출력물"로 정의된 자동화 MVP 설계서로 변환. (설계까지가 이 라인의 범위, 구현은 후속 확장)

## 처리 순서

1. **업무 분해**: 현재 수작업을 단계별로 나열 (예: 주문 확인 → 라벨 작성 → 배송 준비 → 안내문 발송)
2. **입력 정의**: 자료 형식, 출처, 발생 주기, 개인정보 포함 여부
3. **처리 정의**: 규칙 기반인지 판단 필요인지 구분. 판단 필요 지점 = 사람 검수 지점
4. **출력 정의**: 파일 형식, 사용 목적, 품질 기준
5. **MVP 범위 확정**: 첫 버전에서 자동화할 것 / 수동으로 남길 것 / 후속 확장
6. **검수 지점 설계**: "자동 생성 후 사람 확인" 구조 명시 (무인 운영 설계 금지)

## MVP 설계서 형식

| 항목 | 내용 |
|---|---|
| 자동화 대상 업무 | |
| 현재 소요 시간/오류 유형 | |
| 입력자료 (형식·출처·주기) | |
| 처리 과정 (단계별) | |
| 출력물 (형식·품질 기준) | |
| 사람 검수 지점 | |
| 개인정보 처리 방식 | |
| MVP 범위 / 후속 확장 | |
| 기대 효과 (시간 절감·오류 감소, 측정 방법) | |

## 예시 아이템 (설계 대상 후보)

- 주문정보 정리 → 라벨 PDF 생성 → 배송 체크리스트 생성
- 안내문 대기열 생성 (발송 목록 자동 생성 후 확인하여 발송)
- 후기글 수집 → 콘텐츠 변환 대기열 (D라인 연계)

## 주의

- "자동 발송·무인 운영"으로 설계하지 않는다. 항상 사람 확인 단계를 넣는다.
- 개인정보(주소·연락처)가 흐르는 지점을 명시하고 보관·삭제 방침을 함께 설계한다.
