# presentation_workflow.md — B라인: 발표자료 공장 워크플로우

전제: 최종 제출본 LOCK(14번) 이후에만 시작. 발표자료는 제출본 요약이 아니라 **심사표·예상 질문·예산 방어 중심의 재구성**이다.

## 입력 확인

제출 과제명 / 핵심 아이템 / 예산 / 성과목표 / 증빙 / 평가항목·배점 / 발표시간 / Q&A 방식 / 공격 예상 지점. 없으면 `[확인 필요]`.

## 1. 목차 생성

기본 흐름(공고 기준에 맞게 조정):
1) 과제명·한 줄 요약 → 2) 문제·필요성 → 3) 기존 방식의 한계 → 4) 핵심 해결방안 → 5) 수행기간 내 실행계획 → 6) 예산 사용계획·산출물 → 7) 성과목표 → 8) 고객·수혜자·확산 계획 → 9) 신청자 역량 → 10) 리스크 관리 → 11) 마무리 메시지

배점 상위 항목을 핵심 슬라이드(중반부, 시간 배분 최대)에 배치한다.

## 2. 슬라이드별 핵심 주장 작성

규칙: 1슬라이드 1주장 / 글자 짧게 / 도식은 구조 이해용, 표는 정리용, 이미지는 증빙용 / 발표자가 실제로 말할 수 있는 언어.

슬라이드 설계표:

| # | 슬라이드 제목 | 핵심 주장(1개) | 연결 평가항목 | 연결 증빙 | 예상 질문 | 권장 시간 |
|---|---|---|---|---|---|---|

## 3. 방어 구조 반영

- 예산 슬라이드: 항목→목적→산출물→성과 흐름이 그림으로 보이게
- 성과 슬라이드: 정량 목표와 측정 방법 병기
- 역량 슬라이드: "왜 신청자인가" 답이 되게

## 4. 불일치 감지 (필수 게이트)

제출본과 대조: 과제명 / 숫자(예산·성과목표·기간) / 산출물 목록 / 증빙 목록.
불일치 1건이라도 있으면 발표자료 LOCK 불가. 불일치 리포트 작성 후 수정.
