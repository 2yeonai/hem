# grant_workflow.md — A라인: 정부지원사업 공장 워크플로우

역할 전환: 공고 분석가 → 리스크 심사관 → 전략가 → 작성자 → 심사위원 (AGENTS.md)

## 0단계. 기준자료 확인 [→ 공고 기준 LOCK]

01_inputs/input_form.md 기준으로 확인: 공고문 / 신청자격 / 지원대상 / 지원제외 / 제출양식 / 페이지·글자수 제한 / 심사표·평가항목·배점 / 예산 기준·자부담 / 필수 제출서류 / 가점·감점·탈락 조건 / 발표·면접 여부·시간.
**없는 자료는 임의 확정하지 않고 `[확인 필요]` 표시.** 출력: notice_summary.md (03_templates/notice_summary_template.md)

## 1단계. 프로젝트 분리 [→ 프로젝트 분리 LOCK]

중심 아이템 1개 확정. 신청자·사업자 형태·팀 구성 확인. 다른 아이템은 증빙/협력/확장으로만 등장.

## 2단계. 자격·리스크 검토 [→ 자격·리스크 LOCK, 사람 확정]

자격 충족, 업력, 폐업 이력, 중복수혜, 지원제외 업종·비용, 체납·채무, 특수관계자 거래, 외주 의존·단순 구매로 보일 위험, 법률·개인정보 리스크, 기간 내 실현 가능성. 출력: applicant_risk_check.md. **탈락급 리스크 발견 시 STOP하고 사용자 보고.**

## 3단계. 아이디어 후보 정리 [→ 후보 LOCK]

후보를 버리지 않고 모두 등록: 아이디어명 / 해결하는 문제 / 대상 / 산출물 / 필요 예산 / 보유 증빙 / 리스크 / 공고 적합도.

## 4단계. 심사표·배점 분석 [→ 심사표 분석 LOCK]

배점 최고 항목, 감점 위험 항목, 증빙 필요 항목, 발표 질문 예상 항목, 정량/정성 구분, 가점 항목. 출력: scoring_strategy.md 초안.

## 5단계. 아이디어 평가·선정 [→ 최종 아이디어 LOCK, 사람 확정]

evaluation_rubric.md의 10축 5점 척도 비교표 작성. 출력: idea_comparison_table.md.
선정 기준: 재미가 아니라 **현재 심사표에서 점수 가능성 최고**인 것.

## 6단계. 핵심 프레임 LOCK [→ 과제명 LOCK]

최종 아이디어 / 과제명(=[고객·현장]+[문제]+[해결수단]+[결과물], 3~5초 내 이해) / 한 줄 요약 / 첫 고객 / 핵심 문제·해결·산출물·성과목표 / 최강조 평가항목 / 최경계 리스크. 출력: selected_idea_lock.md.

## 7단계. 작성전략 수립 [→ 작성전략 LOCK]

전략표: 평가항목 / 배점 / 심사위원이 보는 것 / 작성전략 / 필요한 증빙 / 분량 비중. 출력: scoring_strategy.md 확정.
배점 높은 항목 = 분량·근거·증빙·도식 집중. 낮은 항목 = 압축.

## 8단계. 본문 구조 작성 [→ 본문 구조 LOCK]

공고 양식 우선. 기본 흐름: 문제정의(시간·비용·오류로 수치화) → 기존 대안의 한계(비난 금지, 인정 후 한계) → 해결방안(기능·절차·산출물·수행범위 표) → 실행계획(기간 내 일정·자원·검증·보완계획) → 성장전략(현재 단계와 장기 분리) → 신청자 역량.

## 9단계. 증빙자료 연결 [→ 증빙 LOCK]

evidence_checklist.md 적용. 주장-증빙 연결표. 이미지마다 200~300자 설명문.

## 10단계. 예산표 작성 [→ 예산표 LOCK, 사람 확정]

budget_alignment_checklist.md 적용. 형식: 예산 항목 / 사용 목적 / 산출물 / 산정 근거 / 연결 평가항목 / 기대성과.

## 11단계. 페이지·글자수 조정 [→ 분량 LOCK]

핵심 주장 앞으로, 중복 삭제, 감성보다 근거, 배점 낮은 내용 과감히 축소.

## 12단계. 위험 표현 제거 [→ 위험표현 LOCK]

danger_words.md 전체 스캔 → replacement_words.md로 대체 → 0건 확인.

## 13단계. 최종 제출본 정리 [→ 제출본 LOCK, 사람 확정]

final_review_checklist.md 전 항목 통과 → 복사·제출 가능한 상태로 완성 → 심사위원 모드 검수(05_reviews) → 사람 확정.

## 14~16단계. 발표 단계 (발표 있는 공고만)

presentation_workflow.md → qna_workflow.md로 인계.
