# content_workflow.md — D라인: 콘텐츠 변환 공장 워크플로우

목적: 원문 1개 → 채널별 콘텐츠 세트로 반복 변환. 브랜드 말투 유지.

## 전제 입력 [확인 필요 — 아직 미입력]

- 브랜드 말투 가이드: 실제 글 샘플 2~3개에서 추출 (문장 길이, 어미, 이모지 사용, 금지 표현)
- 채널 목록과 규격

## 처리 순서

1. **원문 핵심 추출**: 대상 독자 / 핵심 메시지 1개 / 근거·에피소드 / 행동 유도(CTA)
2. **채널별 변환**:

| 채널 | 형식 규칙 |
|---|---|
| 블로그 글 | 제목+소제목 구조, 검색 키워드 반영, 1,000~2,000자 |
| 인스타 캡션 | 첫 줄 후킹, 줄바꿈 리듬, 해시태그 세트 |
| 릴스 대본 | 3초 후킹 → 본문 → CTA, 30~60초 분량, 장면 지시 포함 |
| 카드뉴스 문구 | 표지 1장 + 본문 5~8장, 장당 1메시지 2줄 이내 |
| 썸네일 문구 | 후보 5개, 10자 내외 |
| 스레드 글 | 1포스트 500자 이내 연작, 첫 포스트에 결론 |
| 후기글 변환 | 고객 언어 유지, 과장 제거, 사실 기반 |

3. **말투 통일**: 브랜드 말투 가이드 적용
4. **안전 검수**: danger_words.md + 광고 심의 리스크(효능·효과 단정, 최상급 표현) 스캔. 가격·일정·사실관계는 사람 확인.

## 출력

04_outputs/<프로젝트>/content_set_<날짜>.md — 채널별 섹션 + 검수 체크 결과 포함

## 품질 기준

- 채널 규격 준수 / 핵심 메시지 1개 유지 / 브랜드 말투 일치 / 과장 0건
