# qna_workflow.md — C라인: 발표연습기 공장 워크플로우

전제: 발표자료 LOCK(15번) 이후 시작. 목적은 암기가 아니라 **자연스럽고 과장 없는 방어**.

## 1. 대본 생성 [→ 발표 대본 LOCK]

- 전체 대본: 서류 문장 낭독 금지. 발표자 말투, 한 문장 짧게, 정책 용어 최소화, 실제 경험이 드러나게.
- 버전 세트: 전체 대본 / 시간 압축 대본 / 슬라이드별 핵심 문장 / 쉬운 말 버전 / 마무리 멘트.

## 2. 시간 압축

발표시간 기준 압축: 중복 삭제 → 배점 낮은 내용 축소 → 예산·성과·차별성·실행계획은 사수. 슬라이드별 권장 시간표 작성. 시간 부족 시 반드시 남길 문장 지정.

## 3. 슬라이드별 연습표

| 슬라이드 | 반드시 말할 문장 | 빼도 되는 문장 | 예상 질문 | 위험 표현 | 연결 증빙 |
|---|---|---|---|---|---|

## 4. 공격 질문 훈련 [→ 발표 Q&A LOCK]

00_rules/presentation_attack_questions.md의 필수 10대 질문 + 슬라이드별 질문 + 예산 방어 질문 전체에 답변 작성.

- 예산 방어 구조: 예산 항목 → 사용 목적 → 산출물 → 성과목표 → 평가항목 연결
- WHY 3종: 아이템(문제+기존 한계) / 지금(공고 취지+환경 변화) / 신청자(경험+자원+현장 이해)
- 모르는 질문 안전 문장: "현재 단계에서는 제출한 수행범위 안에서 먼저 검증하고, 결과를 바탕으로 다음 단계에서 보완하겠습니다."

답변 피드백 기준: 직접 답변 / 간결성 / 무과장 / 증빙 언급 / 예산-산출물 연결 / 실제 발화 가능성.

## 5. 위험 표현 감지

대본·답변 전체를 danger_words.md로 스캔. "완벽히, 무조건, 100%, 확정, 독점, 자동으로 전부, 실패하지 않음, 검증 완료, 효과 보장" 발견 시 대체.

## 6. 최종 리허설

- 실제 발표시간 기준 전체 발표 (사람이 소리 내어 읽고 시간 측정 — AI 대체 불가)
- 예상 질문 5~10개 랜덤 공격 → 답변 피드백
- 심사표 기준 점수표 + 마지막 수정 우선순위
