# 증빙 연결표 (9단계): sample-demo

⚠️ 규칙 기반 예비 판정 — AI 검수·사람 확정 전 참고용. 상태: DRAFT / 2026-07-04
이미지·도식에는 200~300자 설명문 — 심사위원이 해석하게 두지 않는다

| 본문 주장 | 증빙 후보 | 상태 | 사용 방법 |
|---|---|---|---|
| 문제가 반복된다 | 작업시간 실측 기록, 주문서·장부 샘플, 작업 과정 사진 | 확보 | 주문서·장부 샘플, 작업 과정 사진 |
| 고객이 실재한다 | 매출·판매 내역, 고객 후기, 판매 채널 화면 | 확보 | 매출·판매 내역, 판매 채널 화면 |
| 신청자가 실행할 수 있다 | 사업자등록증, 포트폴리오, 교육 수료자료, 자격증 | 확보 | 사업자등록증 |
| 예산이 적정하다 | 견적서, 예산 산출근거, 시장 단가 자료 | 예정 | 견적서 — 본문에는 '확보 후 추진'으로만 |
| 기존에 시도해 봤다 | 시제품·목업, 랜딩페이지, 상세페이지 | 없음 | 이 주장 단정 금지 — 보완계획으로 대체 |

## 증빙 공백 (확보 전 해당 주장 단정 금지)

- '기존에 시도해 봤다' 증빙 없음: 시제품·목업 / 랜딩페이지 중 확보 필요

- 확보: 사업자등록증, 작업 과정 사진, 주문서 엑셀 샘플, 판매 내역
- 예정: 개발 견적서, 수요조사, 작업시간 실측 기록
