# budget_risk_review.md — 예산 위험 검토 (mvp_test_01)

⚠️ 최상위 지적: **예산표가 아직 없다.** 아래는 selected_idea_lock의 방향에서 예상되는 예산 항목 기준의 사전 위험 검토다. 실제 예산표 작성(grant_workflow 10단계) 시 이 표를 먼저 반영할 것.

| 예산 항목 (예상) | 위험 이유 | 산출물 연결 여부 | 대체 표현 | 수정 방향 |
|---|---|---|---|---|
| 자동화 도구 개발·구축비 | **외주개발 의존으로 보일 위험** — 총액 대비 비중이 크면 "외주 사업" 프레임 | 라벨 PDF 도구·체크리스트 생성기 — 연결 가능 | "외주개발" → "요구사항 정의·검수는 신청자 직접, 구현 위탁은 [범위]에 한정" | 견적서 2곳 + 신청자 수행 범위 병기 |
| 소프트웨어·서비스 이용료 | **구독료로 보일 위험** — 공고상 지원제외 가능성 [확인 필요] | 약함 | 구독료 항목은 원칙 배제, 필요 시 "수행기간 내 검증용 최소 이용" | 실제 공고 허용 항목 대조 후 결정 |
| 장비 구매 (프린터 등) | **단순 구매성** — 범용 사무기기는 제외 항목 [가정] | 라벨 출력에 연결되나 범용성이 문제 | "장비 구매" 단독 항목 금지 → 시스템 구축의 부속으로, 필요 최소 | 공고 허용 여부 확인, 불허 시 자부담 |
| 교육·역량강화비 | 낮음 | 운영 매뉴얼·정착에 연결 | - | 산출물(매뉴얼) 명시 |
| 마케팅비 | 공고 목적(업무 효율)과 거리 — 배점 없는 곳에 예산 배치 위험 | 이 과제 산출물과 연결 약함 | 이번 과제에서는 배제 또는 최소화 | 후속 계획으로 분리 |
