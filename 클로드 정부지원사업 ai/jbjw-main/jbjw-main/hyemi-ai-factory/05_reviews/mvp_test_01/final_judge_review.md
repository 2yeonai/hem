# 심사위원 총평: mvp_test_01

작성일: 2026-07-03 / 심사 모드: 떨어뜨릴 이유를 찾는 심사위원 / 검수 대상: 04_outputs/mvp_test_01 전체

## 1. 총평

이 산출물은 지금 상태로 제출하면 떨어진다. 구조는 잡혔지만 **모든 핵심 수치가 가정값이고, 공고문 원문이 없어 자격·예산·지원제외 판단 전체가 허공에 떠 있다.** 아이디어 선정 논리와 감점 회피 설계는 방향이 맞으나, 심사위원이 점수를 줄 "근거"(실측치·견적서·기준치)가 아직 하나도 실물로 없다. 공장 구조 테스트로는 합격, 제출물로는 불합격이다.

## 2. 강점 (근거 있는 것만)

| 강점 | 근거 | 연결 평가항목 |
|---|---|---|
| 아이디어 선정이 배점 논리로 이루어짐 | 후보3(매력적이나 공고 불일치)을 탈락시키고 근거를 남김 | 공고 적합성 |
| 문제가 시간·오류로 수치화되는 구조 | 주당 10시간·오배송이라는 측정 가능한 프레임 | 필요성 20점 |
| 반자동(검수 후 처리) 설계 | "자동으로 모두 처리" 함정을 사전 회피 | 실현 가능성 30점 |
| 확보/미확보 증빙 구분 | 견적서·시간기록을 "확보 필요"로 정직하게 표기 | 증빙 신뢰도 |

## 3. 약점

| 약점 | 심각도 | 근거 |
|---|---|---|
| 공고문 원문 부재 — 지원제외·예산 허용 항목을 가정으로 판단 | **탈락급** | 지원제외 조항 하나로 접수 무효 가능 |
| 성과목표 기준치가 임의 가정 (주 10시간, 월 주문 300건) | **탈락급~감점급** | 발표에서 "그 수치 근거가 뭔가요" 한 방에 무너짐 |
| 중복수혜·체납 미확인 | 탈락급 | 요건 미비는 심사 전 컷 |
| 예산표가 아예 없음 | 감점급 | 예산 적정성 15점을 통째로 비워둔 상태 |
| 차별성 답변이 한 줄뿐 | 감점급 | "기존 배송 프로그램과 뭐가 다른가" 공격에 대비 부족 |
| 외주 범위 미정의 | 감점급 | "직접 하나 외주하나"에 답이 없음 |

## 4. 탈락 위험 (심사위원 화법)

"신청자분, 주당 10시간이라는 수치는 어디서 나온 겁니까? 측정 기록 있습니까? 그리고 이 도구, 시중 택배 프로그램 사서 쓰면 되는 것 아닌가요? 2천만원 중 개발비가 얼마고 누가 개발합니까? 본인이 합니까, 외주입니까? — 이 네 질문에 지금 답이 없습니다."

## 5. 보완 가능성

실측 기록 2주치 + 견적서 2건 + 공고 원문 대조만 이루어지면 필요성·실행계획·예산 세 항목이 동시에 살아난다. 구조가 이미 배점 정렬돼 있어 보완 효율이 높다. 예상 상승폭: score_estimation_table.md 참조 (61→85점대).
