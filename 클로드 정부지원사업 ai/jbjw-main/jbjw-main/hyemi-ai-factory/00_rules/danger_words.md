# danger_words.md — 위험 표현 목록 (삭제 불가, 추가만 가능)

모든 제출용 산출물·발표 대본에서 아래 표현을 감지하면 즉시 replacement_words.md로 대체한다.

## 1. 과장·단정형

| 표현 | 위험 유형 |
|---|---|
| 완벽히 해결 / 완벽한 | 실현 불가능 단정 |
| 100% 가능 / 100% | 근거 없는 확률 단정 |
| 무조건 가능 / 무조건 | 근거 없는 단정 |
| 검증 완료 | 증빙 없으면 허위로 간주될 위험 |
| 확정 (계약·판매·제휴 앞에서) | 증빙 없는 확정 표현 |
| 실패하지 않음 | 리스크 인식 부재로 감점 |
| 효과 보장 | 보장 불가 + 광고심의 리스크 |

## 2. 시장·지위 과장형

| 표현 | 위험 유형 |
|---|---|
| 독점 | 시장 이해 부족으로 보임 |
| 최고 수준 | 비교 근거 요구받음 |
| 세계 최초 / 국내 최초 | 검증 요구 + 반례 시 신뢰 붕괴 |
| 폭발적 성장 / 대규모 확장 (근거 없이) | 실현 가능성 공격 |
| 제휴 확정 / 판매 확정 (증빙 없이) | 허위 기재 위험 |

## 3. 자동화 과신형 (AI 사업에서 특히 위험)

| 표현 | 위험 유형 |
|---|---|
| 자동으로 모두 처리 | 검수 부재 → 품질·책임 공격 |
| AI가 판단 | 책임 소재 불명 → 감점 |
| 자동 발송 | 오발송 사고 책임 질문 |
| 무인 운영 | 신청자 실행역량 무의미화 + 사고 책임 |
| 사람 개입 불필요 | 위와 동일 |

## 4. 법률·행정 리스크형

| 표현 | 위험 유형 |
|---|---|
| 치료 / 완치 / 의학적 효과 (근거 없이) | 의료법·표시광고법 |
| 수익 보장 / 원금 보장 | 법적 리스크 |
| 개인정보를 수집·활용 (처리 방침 없이) | 개인정보보호법 질문 |

## 감지 시 처리 형식 (필수)

- 기존 표현 / 위험 이유 / 대체 표현 / 수정 후 문장 — 4열 표로 기록
- 기록 위치: 05_reviews/<프로젝트>/dangerous_expression_review.md
