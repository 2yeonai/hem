# evidence_checklist.md — 증빙자료 체크리스트

원칙: 증빙은 첨부용 장식이 아니라 본문 주장의 근거다. 주장마다 증빙을 연결한다.

## 연결 규칙

- [ ] 본문의 모든 핵심 주장에 증빙이 연결돼 있는가 (증빙 연결표 작성)
- [ ] 확보된 증빙과 예정 증빙이 구분 표기돼 있는가
- [ ] 증빙 없는 주장을 단정형으로 쓰지 않았는가 ("예정", "확보 후 추진"으로)
- [ ] 이미지·도식마다 200~300자 설명문이 붙어 있는가 (심사위원이 해석하게 두지 않기)

## 증빙 유형별 점검

| 유형 | 예시 | 점검 |
|---|---|---|
| 신청자 자격 | 사업자등록증, 업력 증명 | [ ] |
| 시장·수요 | 수요조사, 설문, 시장자료, 고객 후기, 인터뷰 | [ ] |
| 실적 | 판매자료, 매출자료, 포트폴리오 | [ ] |
| 현장 | 작업 과정 사진, 현장 사진, 제품 이미지 | [ ] |
| 협력 | MOU, 구매의향서, 거래처 자료 | [ ] |
| 실행역량 | 자격증, 교육 수료, 기존 운영 채널 | [ ] |
| 제품·서비스 | 시제품, 목업, 랜딩페이지, 상세페이지 | [ ] |
| 예산 근거 | 견적서, 시장 단가 자료, 산출근거표 | [ ] |

## 증빙 연결표 형식

| 본문 주장 | 필요한 증빙 | 상태(확보/예정) | 증빙 위치 | 설명문 |
|---|---|---|---|---|

## 금지

- 존재하지 않는 증빙을 "확보"로 표기 (허위 기재 → 탈락+제재 위험)
- AI가 증빙 실물 존재를 확정 판단 (사람 확인 필수)
