# final_review_checklist.md — 최종 제출 전 검수 체크리스트

제출본 LOCK(14번) 전에 전 항목 통과 필수. 하나라도 실패하면 revision_order 생성.

## A. 공고 적합성

- [ ] 현재 공고 목적과 맞는가
- [ ] 신청자 조건과 맞는가 (업력, 사업자 형태, 지역)
- [ ] 지원대상에 포함되는가
- [ ] 지원제외 조건에 걸릴 위험이 없는가
- [ ] 예산 기준(허용 항목·자부담)과 충돌하지 않는가

## B. 심사표·배점

- [ ] 배점 높은 항목이 충분한 분량·근거로 반영됐는가
- [ ] 배점 낮은 항목에 과도한 분량을 쓰지 않았는가
- [ ] 심사위원이 점수 줄 근거(수치·증빙)가 항목마다 있는가
- [ ] 정량 목표와 정성 목표가 구분되는가

## C. 스토리 일관성

- [ ] 과제명이 본문 전체와 일치하는가
- [ ] 문제 → 해결 → 실행 → 예산 → 산출물 → 성과 → 역량 → 증빙 흐름이 끊기지 않는가
- [ ] 성과목표가 측정 가능한가
- [ ] 하나의 문서에 하나의 중심 아이템만 있는가

## D. 실행 가능성

- [ ] 수행기간 안에 가능한가
- [ ] 신청자가 직접 실행 가능해 보이는가
- [ ] 외주 의존으로 보이지 않는가
- [ ] 단순 구매성 사업으로 보이지 않는가
- [ ] 산출물이 명확한가

## E. 예산·증빙

- [ ] budget_alignment_checklist.md 전 항목 통과
- [ ] evidence_checklist.md 전 항목 통과

## F. 표현·형식

- [ ] danger_words.md 스캔 결과 0건
- [ ] 페이지·글자수 제한 준수
- [ ] 확보 증빙과 예정 증빙이 구분 표기됨
- [ ] 복사해서 바로 제출 가능한 상태인가
