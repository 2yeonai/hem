# APP_USAGE.md — Hyemi Grant Factory 사용법

## 화면 흐름

```
대시보드 ── + 새 프로젝트 ──→ 입력 폼 ──→ 분석 실행
   │                                        ↓
   └─ ▶ 샘플 실행 ──────────→ 프로젝트 화면 (11탭 + 공정 0~16 진행바)
        ① 공고 분석 → ② 자격 리스크 → ③ 아이디어 비교 → ④ LOCK 판정
        → ⑤ 심사위원 검수 → ⑥ 재작업 지시서 → ⑦ 발표 Q&A
        → ⑧ 서류 초안 → ⑨ 발표자료(PPTX) → ⑩ 발표연습기 → ⑪ Export/HANDOFF
```

모든 탭 상단의 **공정 0~16 진행바** = 혜미 방식 17단계. 색: 초록(완료)/주황(진행)/회색(대기)/빨강(잠김)/파랑(사람 몫). 번호에 마우스를 올리면 단계명·상태가 보인다.

## 탭별 보는 법 (위험이 먼저 보이게 설계됨)

| 탭 | 보는 것 | 여기서 할 일 |
|---|---|---|
| 상단 배너 | ⛔탈락급 / ⚠️원문 없음 / 위험 표현 | 빨간 배너가 있으면 다른 탭보다 먼저 해소 |
| ① 공고 분석 | 요약, 배점 최고 항목, [확인 필요] 목록 | 원문 미확보 항목을 공고에서 찾아 보완 |
| ② 자격 리스크 | 체납·중복수혜·업종 판정 | 서류(납세증명·수혜이력·등록증)로 확인 |
| ③ 아이디어 비교 | 6축 점수, 추천/보류/탈락, 감점 근거 | 탈락 후보를 본문에 섞지 않기 |
| ④ LOCK 판정 | 불가 사유, 승인 체크박스, 과제명 후보 | **서류로 확인한 것만 체크** — 체크가 책임 기록 |
| ⑤ 심사위원 검수 | 예상 점수표, 위험 표현, 예산 리스크, 누락 증빙 | 80점 미만이면 제출 금지 |
| ⑥ 재작업 지시서 | 1~3순위 + 혼입 금지 | 1순위부터 처리 |
| ⑦ 발표 Q&A | 10대 질문 발화 가능/증빙 필요/발화 금지 | 발화 금지 답변은 근거 확보 전 연습도 금지 |
| ⑧ 서류 초안 | 작성전략표, 본문 6절 초안, 증빙 연결표, 연결형 예산표, 최종 점검 9종 | [확인 필요] 대괄호를 실제 내용으로 채우기 — 초안은 항상 DRAFT |
| ⑨ 발표자료 | 슬라이드 11장 구성 + **PPTX 다운로드**, 발표 대본(풀·압축·쉬운말) | PPTX를 PowerPoint에서 열어 디자인·이미지 보강 |
| ⑩ 발표연습기 | 시간 측정 타이머, 공격 질문 뽑기, 위험 표현 실시간 감지, WHY 3종·예산 방어 | 발표 시작 → 슬라이드 넘기며 소리 내 연습 → 시간 리포트 확인 |
| ⑪ Export | 전 파일 링크, 다음 실행 프롬프트 | AI 정밀 검수로 넘길 때 프롬프트 복사 |

## 서류·발표 라인 (아이디어 선정 후 자동 생성)

아이디어가 평가돼 추천(또는 사람 확정)되면 7~16단계 산출물이 자동 생성된다:
- `writing_strategy.md` (작성전략표) / `application_draft.md` (본문 초안) / `evidence_link_table.md` (증빙 연결)
- `budget_table.md` (연결형 예산표) / `final_check.md` (분량·최종 점검) / `slides_outline.md` + **`slides.pptx`**
- `presentation_script.md` (대본 3종) / `qna_defense.md` (WHY 3종·예산 방어·슬라이드별 예상 질문)

생성 문장은 위험 표현 치환기를 자동 통과한다(12단계). 아이디어 미선정(0단계)이면 이 라인은 잠긴다 — 섞임 방지 설계.

## 0단계 모드 (공고만 있고 신청자·아이디어가 아직 없을 때)

새 프로젝트에서 아이디어를 비워두고 저장하면 **0단계**로 등록된다:
- 공고 분석 + 배점 전략(①탭)과 **신청자 입력 질문지**(`04_outputs/<p>/applicant_input_form.md`)가 생성된다
- 심사 판정은 "판정 불가", LOCK은 잠김 — 노란 배너의 **"신청자·아이디어 입력 →"** 버튼(또는 하단 입력/수정 버튼)으로 폼을 채우면 0단계를 벗어나 전체 분석이 돌아간다
- 실전 예시: `2026_ai_support` 프로젝트 (2026 혁신 소상공인 AI 활용지원 공고)

## LOCK이 안 될 때 (정상이다)

LOCK 가능 조건 7가지(app_completion_criteria.md): 공고 원문 / 탈락급 0건 / 자격 승인 / 아이디어 확정 / 예산 승인 / 위험 표현 0건 / 확보 증빙 1건 이상.
전부 충족해야 REVIEWED가 되고, **LOCKED 기재는 사람이 06_locks/LOCK_STATUS.md에 직접** 한다. 앱이 LOCKED를 만들지 않는 것은 버그가 아니라 설계다.

## 입력 수정 후

입력을 바꿨으면 프로젝트 화면 하단 "입력 변경 후 재분석" 버튼. 결과 파일은 덮어써진다 (입력이 바뀌면 결과가 바뀌는 것이 정상).

## AI 정밀 검수로 넘기기

앱의 판정은 규칙 기반 예비 판정이다. ⑧ Export 탭의 "다음 실행 프롬프트"를 Claude 세션에 붙여넣으면 10축 정밀 채점·8종 정성 검수로 이어진다.
