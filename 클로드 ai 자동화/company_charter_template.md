# [AI 회사 이름] 헌장 (Company Charter)

> 이 문서는 AI직원 플랫폼과 모든 도메인 스킬이 참조하는 최상위 정의입니다.
> 여기 없는 결정은 개별 스킬이 임의로 내리면 안 되고, 이 문서를 먼저 업데이트합니다.

## 1. 정체성 — 우리는 뭘 하는 회사인가
- 한 문장 정의: (예: "소상공인 실무를 AI직원이 대신 처리해주는 운영 자동화 회사")
- 지금 단계: 셀프 사용 / SaaS 전환 준비 / SaaS 운영 중 (택 1)
- 다루는 도메인: (예: 정부지원사업, 꽃집 실무, 방역사업 — 계속 추가될 리스트)

## 2. AI직원이란 무엇인가
- 정의: (예: "각 도메인 스킬을 실행하고 결과를 보고하는 자동화 실행 단위")
- 할 수 있는 것:
  - (예: 스킬 실행, 진행상황 보고, 반복작업 처리)
- 할 수 없는 것 (명시적으로):
  - (예: 최종 승인 없이 외부 제출/결제/삭제)

## 3. 권한 경계선 (3단계) — 스킬 설계 시 반드시 이 기준을 따름
| 등급 | 설명 | 예시 |
|---|---|---|
| 자동실행 가능 | 사람 확인 없이 바로 실행 | 공고 수집, 데이터 정리, 초안 작성 |
| 승인 필요 | 결과를 사람이 확인 후 다음 단계 진행 | 신청서 제출, 고객 데이터 변경 |
| 절대 자동화 금지 | 이 회사는 이걸 자동화하지 않음 (최소 1개 이상 명시 필수) | (예: 법적 자문 필요한 최종 판단, 결제/계약 체결) |

## 4. 성장 경로
- 지금: (예: 나 혼자 사용, 3개 도메인 스킬 구축 중)
- SaaS 전환 조건: (예: 3개 스킬이 각각 최소 N회 무사고 실행되면 전환 검토)
- 전환 시 바뀌는 것: (예: 멀티테넌트 데이터 격리, 과금 로직 추가)
