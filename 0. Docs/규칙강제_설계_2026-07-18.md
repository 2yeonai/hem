# 규칙 강제 설계 (2026-07-18) [k9cjhmw7z9]

> 작성: Fable 5 (혜미 지시: "규칙을 강제하게 해줘, 폴더별 역할을 꼭 준수하게, 완료된 건 완료처리 되게 — 몇 번을 말했는지 모르겠어").
> 한 줄 요약: **지금까지 규칙은 "읽고 기억해서 지키는 문장"이었다. 이제부터는 "저장(커밋)하는 순간 컴퓨터가 자동으로 검사해서, 어기면 저장 자체가 안 되는 장치"로 바꾼다.**

## 1. 왜 계속 안 지켜졌나 (진단)

1. **규칙이 문장으로만 존재** — CLAUDE.md·운영표·명령어사전에 흩어진 규칙을 세션이 안 읽으면 그만. 읽었는지 확인하는 장치가 없다.
2. **어겨도 아무 일도 안 일어남** — 폴더 역할을 어기고 파일을 만들어도, 완료된 걸 안 지워도, 그 순간엔 티가 안 난다. 나중에 혜미가 헷갈릴 때에야 문제가 드러난다.
3. **규칙 총량이 사람(세션)의 기억 용량을 초과** — 볼트에 CLAUDE.md만 10개 이상. 다 기억하고 일하는 세션은 없다.

이건 2026-07-17 기록체계 재설계(원칙 3 "규칙은 기억이 아니라 스크립트로 검사")에서 handoff_check.py로 처음 검증된 접근을 **볼트 전체 규칙으로 확장**하는 것이다. 원자노트 [[5. Zettelkasten/20. Permanent/규칙은 스크립트가 검사하게 한다]]의 적용 2탄.

## 2. 3단 강제 체계

### 1단 — 차단 (어기면 저장이 안 됨): git pre-commit hook
git 커밋(저장) 버튼을 누르는 순간 `vault_check.py --pre-commit`이 자동 실행된다. 아래 "하드 위반"이 있으면 **커밋 자체가 거부**된다. 이 볼트에 유입되는 모든 계정·세션은 결국 커밋을 하므로, 문서를 안 읽었어도 강제로 걸린다 — 이게 핵심이다.

하드 위반 4종 (기계가 확실히 판정 가능한 것만 — 오탐이 나면 신뢰를 잃으므로 보수적으로):
| # | 위반 | 근거 규칙 |
|---|---|---|
| H1 | 볼트 루트에 허용 목록 밖의 새 md 파일 생성 | 폴더 역할(PARA) — 산출물은 1. Projects 아래 |
| H2 | 커밋되는 md 파일에 sentinel(`<!-- ok -->`) 누락 | truncation 방지 규칙(2026-07-13 C1) |
| H3 | `1. Projects/` 직속에 새로 만드는 프로젝트 노트에 type/status/due 누락 | 1. Projects/CLAUDE.md 프론트매터 규칙 |
| H4 | `.env`·`*.key`·`credentials` 등 비밀 파일 커밋 시도 | 보안(API 키 평문 유출 방지) |

긴급 탈출구: 정말 예외적으로 통과해야 할 때만 `VAULT_CHECK_SKIP=1 git commit ...`. 사용하면 커밋 메시지에 사유를 남길 것(훅이 사용 사실을 경고로 출력).

### 2단 — 경고 (저장은 되지만 잔소리): 같은 훅의 WARN 출력

## 3. 신설 규칙 2개 (CLAUDE.md에 반영)

### 규칙 A — 완료 이관 규칙
할 일 목록("지금 볼 것", "혜미가 실행할 것", T-테이블, HANDOFF의 다음 할 일)의 항목이 완료되면, 완료시킨 세션이 **그 자리에서 즉시** 표시한다:
- 체크박스: `- [x] ... (완료 YYYY-MM-DD)`
- 표: 상태 칸을 `완료`로
- 줄글: 끝에 ` → ✅ 완료(YYYY-MM-DD, 근거: [[문서]])`
완료 여부가 애매하면 표시하지 말고 "확인 필요"로 보고한다. 할 일의 정본은 `1. Projects/완전자동화_실행계획.md` T-테이블 하나다(정본 지정 원칙).

### 규칙 B — 비개발자 실행 지침 규칙
혜미에게 직접 실행을 요청하는 모든 지침은 반드시 3단계로 쓴다:
① **어떤 프로그램을 여는지부터** — "시작 버튼 → 'PowerShell' 검색 → 파란 창" / "시작 버튼 → 'cmd' 검색 → 검은 창" / "파일 탐색기에서 ○○ 폴더 열기"처럼 여는 방법까지.
② 복사해 붙여넣을 명령(한 줄씩, 통째로 복붙 가능하게).
③ 성공했는지 확인하는 방법("○○라고 뜨면 성공").
