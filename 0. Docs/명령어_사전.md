# 혜미 간단 명령어 사전

> 목적: 혜미가 스킬 이름을 몰라도 **짧은 말 한마디**로 작업이 시작되게 한다.
> 이 사전은 루트 CLAUDE.md에 연결돼 있어 **모든 Claude 세션이 자동으로 읽는다.**
> Claude는 아래 명령을 받으면 확인 질문을 최소화하고 바로 해당 흐름으로 간다.

## 콘텐츠 (계정별)

| 혜미가 이렇게 말하면 | Claude가 하는 일 |
|---|---|
| "마시땅 글 써줘" (+가게 이름/사진) | 콘텐츠 공장 실행 — 진주 맛집 글, @maasittang용. 톤은 콘텐츠_작업스타일.md 따름 → 승인 요청 → 승인 시 Chrome으로 게시 |
| "이연 글 써줘" / "육아 글 써줘" (+사진) | 콘텐츠 공장 — @2yeon_sz용. 육아·**아기코디**(바이럴 결)·엄마바이브 중심 → 승인 → 게시 |
| "블로그 글 써줘" (+주제) | 네이버(+티스토리 개설 후) 장문 글 — 같은 분석카드에서 파생, 수익화(검색 유입) 구조 |
| "오늘 발행 준비해줘" | 콘텐츠_주제백로그.md 맨 위 대기 주제로 3채널 초안 생성 → 승인 요청 |
| "게시해줘" | 승인완료된 초안만 Chrome으로 게시 (승인 안 된 건 구조상 게시 불가) → 발행 장부 기록 |

## 제작물 (기존 라우팅 + 스킬 연계)

| 혜미가 이렇게 말하면 | Claude가 하는 일 |
|---|---|
| "PPT 만들어줘" (+주제) | 볼트에서 관련 내용 수집 → pptx 스킬로 .pptx 제작 → 해당 프로젝트 폴더 저장 |
| "카드뉴스/이미지 만들어줘" | Canva 커넥터(연결돼 있으면) 사용, 아니면 SVG/HTML 시안 제작 후 안내 |
| "이 캐릭터로 ○○ 만들어줘" | 설계노트 5-7(3D캐릭터·굿즈)·5-6(이미지) 공장 결 따름 — 저작권 게이트 먼저 확인 후 제작 |
| "엑셀로 정리해줘" | xlsx 스킬 — 표/시트 제작 |
| "문서로 만들어줘" | docx 스킬 — 워드 문서 |
| "PDF로 만들어줘" | pdf 스킬 |
| "매주/매일 ○○해줘" | schedule 스킬 — 예약작업 등록 (등록 전 요일·시간 확인) |

## 공장 운영

| 명령 | 동작 |
|---|---|
| "이어서 해줘" | 최신 세션로그(2. Areas/Claude 세션로그/) + 해당 폴더 HANDOFF 읽고 기록된 지점부터 재개 |
| "상태 점검해줘" | 러너 테스트 3종 + validate_manifest 재실행, PASS 여부 보고 |
| "오류 고쳐줘" | ai공장짓기/failure_log.md의 open 카드부터 처리 |
| "볼트 기준으로 평가해줘" (+새 자료) | 핵심맥락 운영규칙 1번 — 기존 시스템과 비교 평가만, 세팅 변경은 승인 후 |

## 규칙 (Claude용)
- 명령어에 해당하면 **긴 되묻기 없이 시작**하되, 되돌릴 수 없는 일(게시·발송·구매)은 반드시 승인 게이트.
- 새 명령 패턴이 생기면 이 표에 **추가**하고 세션로그에 남길 것 (삭제 금지).
- 진짜 "스킬"(Cowork 설정의 skill) 등록은 설정 > 기능(Capabilities)에서만 가능 — 세션에서는 이 사전이 그 역할을 대신함.
