# 혜미 간단 명령어 사전

> 목적: 혜미가 스킬 이름을 몰라도 **짧은 말 한마디**로 작업이 시작되게 한다.
> 이 사전은 루트 CLAUDE.md에 연결돼 있어 **모든 Claude 세션이 자동으로 읽는다.**
> Claude는 아래 명령을 받으면 확인 질문을 최소화하고 바로 해당 흐름으로 간다.

## 스킬 전체 지도 (설치된 스킬 전수 — 위 표에 없는 것만)

> 위 "콘텐츠"·"제작물"·"공장 운영" 표에 이미 나온 스킬(PPT/엑셀/워드/PDF/예약작업/결정 원자노트)은 여기 다시 안 적음. 여기는 **누락분만**.

| 혜미가 이렇게 말하면 | 스킬(정식 이름) | 하는 일 | 예시 |
|---|---|---|---|
| "이런 일 반복되는데 스킬로 만들어줘" | skill-creator | 새로 반복되는 작업 패턴을 위한 전용 스킬(SKILL.md)을 설계·제작 | "견적서 자동 작성 스킬 하나 만들어줘" |
| "기억 정리해줘" / "지금까지 쌓인 메모리 중복 정리해줘" | consolidate-memory | 세션마다 쌓인 기억 파일을 검토해 중복 병합·오래된 정보 수정·정돈 | "기억 파일들 중복되는 거 정리해줘" |
| "이 플러그인 우리 사업에 맞게 바꿔줘" | cowork-plugin-management (cowork-plugin-customizer) | 이미 설치된 Claude 플러그인의 커넥터·설정을 혜미의 도구/워크플로우에 맞춰 조정 | "이 플러그인 방역 업무용으로 맞춰줘" |
| "플러그인 하나 새로 만들어줘" | cowork-plugin-management (create-cowork-plugin) | 처음부터 새 플러그인(도구 묶음)을 설계·제작해 .plugin 파일로 전달 | "방역 견적 자동화 플러그인 하나 만들어줘" |
| "Cowork 처음 설정하는 거 도와줘" | setup-cowork | 역할에 맞는 플러그인 설치·도구 연결·스킬 체험까지 안내하는 최초 설정 가이드 | "Cowork 처음 쓰는데 세팅 좀 도와줘" |
| (혜미가 직접 부를 일 없음) | init / review / security-review | 코드 저장소용 개발자 스킬(CLAUDE.md 초기화, PR 리뷰, 보안 점검) — ai공장짓기 스크립트 작업 중 Claude가 내부적으로 쓸 수는 있으나 혜미가 일반어로 명령할 대상이 아님 | — |

## 규칙 (Claude용)
- 명령어에 해당하면 **긴 되묻기 없이 시작**하되, 되돌릴 수 없는 일(게시·발송·구매)은 반드시 승인 게이트.
- 새 명령 패턴이 생기면 이 표에 **추가**하고 세션로그에 남길 것 (삭제 금지).
- 진짜 "스킬"(Cowork 설정의 skill) 등록은 설정 > 기능(Capabilities)에서만 가능 — 세션에서는 이 사전이 그 역할을 대신함.
- **(2026-07-13, C1 잘림방지)** 새 md 전면 작성/재작성은 Edit/Write 대신 bash heredoc + `wc -c` 확인. 모든 md의 마지막 줄은 `<!-- ok -->`(sentinel) 고정 — Edit/Write 직후 `python3 ai공장짓기/scripts/verify_write.py --sentinel-check <경로>`로 PASS 확인할 것.
