# 혜미 자동화 운영표 (2026-07-16)

> 한 줄 목적: **혜미 손이 닿는 지점을 "① 승인 클릭 ② 사람만 아는 정보 입력" 딱 2가지로 줄인다.** 나머지는 전부 이 표대로 Claude가 분배·실행한다.

## 0. 기본 규칙 (2026-07-16 신설, 모든 세션 적용)

1. **프로젝트 시작 위치 = HEM 볼트 고정.** 다른 지시가 없으면 모든 새 프로젝트·산출물은 `C:\Users\82106\Desktop\hem` 안에서 시작한다. 새 프로젝트 폴더는 `1. Projects/` 아래, 산출물은 해당 프로젝트 폴더에 저장. (루트 CLAUDE.md에도 고정됨)
2. **모델은 혜미가 고르지 않는다.** 분배는 이 표 + [[prompts/routing-policy|routing-policy]]가 정한다. 기본 Sonnet, 단순 분류·요약은 Claude가 알아서 경량(Haiku 서브에이전트)으로 내리고, 설계 충돌·새 구조 판단만 Claude가 먼저 "Fable로 승격할까요?" 승인을 구한다.
3. **긴 자료가 들어오면 저장 전 평가.** §4 인테이크 파이프라인을 따른다. 평가 전엔 세팅을 바꾸지 않는다(핵심맥락 운영규칙 1번).

## 1. 작업스타일 → 운영 전략

핵심맥락의 진단(2026-07-09)을 실행 규칙으로 변환한 것.

| 구분 | 내용 | 운영에 이렇게 반영 (증폭/보완) |
|---|---|---|
| 강점 ① | 기록 습관 (HANDOFF/decision-log/세션로그) | **증폭**: 모든 세션이 기록에서 시작("이어서 해줘")하고 기록으로 끝남(22:30 자동 로그). 기록이 곧 인수인계라 혜미가 맥락 재설명 0회 |
| 강점 ② | 승인 게이트 감각 (되돌릴 수 없는 건 사람이) | **증폭**: 게시·발송·구매·삭제·설정변경만 승인 요청. 그 외 확인 질문은 생략하고 진행 — 승인 요청이 적을수록 게이트의 무게가 유지됨 |
| 강점 ③ | 구조적 사고 (계약·게이트·티어 개념 체득) | **증폭**: 새 업무는 전부 "입력→판단→출력→승인→인계" 5칸으로 쪼개 이 표에 추가. 공장 manifest와 같은 언어 |
| 병목 | **판단 과부하** (정보 유입량 ≫ 판단 처리량) | **보완**: 판단을 파일에 위임 — 순서=실행계획, 모델=routing-policy, 중요도=감사_로드맵, 새 자료=§4 인테이크. Claude는 항상 "선택지+추천+모델/시간/혜미 몫"으로 가져오고 혜미는 고르기만 |

## 3. 세션 운영 프로토콜 (Claude용 — 매 세션 동일)

1. **시작**: HEM 볼트에서 시작 → 최신 세션로그 + 해당 HANDOFF 읽기 ("이어서 해줘"와 동일 동작).
2. **분배**: 받은 일을 local(코드)/model(AI)/human(승인) 단계로 쪼개고, 티어 배정(§0-2). 여러 단계면 앞 단계 결과를 다음 모델이 이어받게 TASK 순서를 세움 — 병렬 가능한 것은 서브에이전트로 동시 실행.
3. **보고**: 결과 먼저 한 줄 → 상세. 제안할 땐 항상 모델/소요시간/혜미 몫 3종 명시.
4. **종료**: HANDOFF 갱신 + 새 명령 패턴 생겼으면 명령어_사전에 추가. (세션로그는 22:30 예약작업이 자동 작성)

## 4. 정보 인테이크 파이프라인 (AI 소식·가이드·설정 자료)

혜미가 긴 자료(글 전문, 설치 가이드, 공략집, 링크)를 들고 오면 — **혜미는 던지기만, 분류·판단은 Claude가**:

```
[혜미: 자료 + 한마디]
   ├─ "아카이브해줘" → 3~5줄 요약 + 원본 링크 → 5. Zettelkasten/00. Inbox (보관만, 끝)
   └─ "정리해줘" / "볼트 기준으로 평가해줘" / 그냥 붙여넣기 →
        ① 평가: 볼트 기존 시스템과 대조 → 채택/보류/폐기 판정표
           → 0. Docs/외부트렌드자료_평가_YYYY-MM-DD.md
        ② 보관: 원문 정리본(실행 명령어·체크리스트 보존) → 3. Resources/AI도구_외부자료/
        ③ 반영: 채택분만 이 운영표·명령어_사전·routing-policy에 반영 (세팅 변경은 승인 후)
```

- 근거: 7/13 평가에서 트렌드 6건 중 실질 채택 2건 — **유입 자료 대부분은 이미 가진 것보다 못하다.** 안 읽은 자료에 죄책감 불필요, 밀린 건 일요일 주간종합이 배치 처리.
- 실적: 2026-07-16 ECC 설치가이드 + Fable5 공략집 2건 처리 → [[0. Docs/외부트렌드자료_평가_2026-07-16|평가문서]].
