# 혜미 운영지침서 (2026-07-13)

> 결론부터. 각 절의 "실행"은 Sonnet이 판단 없이 따라 하면 되고, **혜미 몫은 굵게** 표시.

## 0. 우선순위 판정 — 혜미 제안에 대한 답

혜미 제안: 옵시디언 → 정부지원 완성 → AI회사 → mo,on. **판정: 절반 동의, 절반 수정.**

| 순위  | 항목                      | 판정                           | 근거                                                                                                                   |
| --- | ----------------------- | ---------------------------- | -------------------------------------------------------------------------------------------------------------------- |
| ⓪   | **방역 골든셋 인터뷰 (혜미 1시간)** | 제안에 빠져 있어서 추가 — 혜미 시간 기준 최우선 | 어제(감사_로드맵_2026-07-13) 확정한 전체 라인의 유일 병목. 딱 1시간이면 방역 검증→꽃집→플랫폼 라인 전체가 풀림                                               |
| ①   | 옵시디언 맥락·정리              | 동의 — 단 **반나절 범위로 제한**        | "매번 맥락 설명"은 모든 작업에 붙는 세금이라 최우선 맞음. 단 완벽 정리가 아니라 §1+§5 실행이면 충분 — 그 이상은 정리를 위한 정리                                      |
| ②   | 정부지원 완성 **= mo,on**     | 동의 — 단 ②와 ④는 같은 작업           | "완성"의 지금 의미 = 탈락분석(B2B 품질관리 SaaS 피벗 수렴) 반영 + 다음 공고 재투입. 그 검증 대상이 mo,on 신청서 그 자체 — 분리 불가. 스크래핑·PPT는 이미 결정대로 8월/선정 트리거 |
| ③   | AI회사 플랫폼                | **4위로 내림** (mo,on과 순서 교체)    | 수익은 mo,on에서만 남. 플랫폼 총설계는 방역 검증 결과가 입력이라 지금 당기면 재작업 위험 — 어제 결정 유지: 8월 초 Fable 총설계 1회, 9월 F4 후 착수                      |

**병렬 가능 (이번 주에 세 라인 동시):**
- **라인 A (Sonnet 단독, 혜미 0분)**: §1 폴더정리 + §5 스킬사전 + §6 유지규칙 — 세션 1~2개로 전부 처리
- **라인 B (혜미 1시간)**: 골든셋 인터뷰 → 이후 배치테스트는 Sonnet
- **라인 C (혜미 부분 참여)**: 정부지원 다음 공고 탐색 + 피벗 반영 재투입
- 강의(§4)는 이미 진행 중인 일정 그대로 병행.

## 1. 폴더 정리 (중복 버그 해소 + PARA 재배치)

**실행 (Sonnet, 2시간, 혜미 몫: 이동 매핑표 승인 1회):** 순서대로.
1. **감사_로드맵_2026-07-13의 E1 먼저** (sentinel 잘림방지 — 정리 중 또 잘리면 안 되니 최우선).
2. **E2 실행**: _inbox 중복 인덱스 → Archive 이동 / 깨진 링크 ~150개 재매핑 / 바이너리 링크 스캔 제외.
3. **R4 꽃집 폴더**: 루트의 정부지원 사본(SKILL.md, manifest.yaml, scripts/, test/, gov-support-skill/)을 정부지원 원본과 diff로 동일 확인 후 `4. Archive/꽃집폴더_정부지원사본_2026-07/`로 **이동** (삭제 금지). diff 결과 붙여넣기.
4. **루트 떠돌이 정리**: `2026-07-08.md` → `2. Areas/Claude 세션로그/과거기록/`. `도구/`·`정부지원금/`은 내용 목록을 뽑아 PARA 매핑표(어디로 옮길지)를 만들고 **혜미 승인 후** 이동. `러너앱_실행.bat`·`_inbox/`는 루트 유지(실행 편의/수집함).
5. `.stale-*` 파일: 기존 규칙 유지 — 방치. 옵시디언 그래프에 안 나오므로 미관 문제 없음.
- **완료 기준**: 루트에 남는 것 = PARA 폴더 7개 + `ai공장짓기` + `클로드 ai 자동화` + `_inbox` + CLAUDE/AGENTS/LLM위키_홈 + bat 파일뿐.

## 2. 그래프 뷰 "혜미 중심"으로 (혜미 직접, 5분 — 클로드가 대신 못 함)

1. 그래프 뷰 열기 → 좌상단 톱니(설정) → **필터**에 입력: `-path:"4. Archive" -path:"_inbox" -path:"7. Attachments"`
2. 같은 설정의 **그룹**에서 색상 추가: `path:"1. Projects"`(초록), `path:"ai공장짓기"`(보라), `path:"2. Areas"`(주황), `path:"5. Zettelkasten"`(파랑)
3. 표시 옵션에서 **"존재하지 않는 파일" 끄기** (깨진 링크 점 제거 — §1-2 후 크게 줄어듦)
4. **"나" 중심 그래프의 정체**: 전역 그래프가 아니라 — `2. Areas/핵심맥락.md`를 열고 우측 패널 **로컬 그래프**(깊이 2)를 켜는 것. 그게 혜미 중심 뷰다. 전체 관계는 `LLM위키_홈.md`의 로컬 그래프.

## 3. 쏟아지는 AI 정보 — 다 수용하지 않는 게 정답

**새 규칙 없음 — 이미 있는 규칙(핵심맥락 운영규칙 1번)을 그대로 쓰면 됨.** 혜미가 할 일은 판단이 아니라 던지기:
- 보관만: **"아카이브해줘" + 링크** → 3~5줄 요약과 함께 Inbox 저장. 끝.
- 채택 판단: **"볼트 기준으로 평가해줘" + 자료** → 클로드가 기존 시스템과 대조해 채택/폐기 표로 답함.
- 근거(실증): 7/13 트렌드 6건 평가 결과 실질 채택 2건뿐 — **쏟아지는 정보의 대부분은 이미 가진 것보다 못하다.** 안 읽은 자료에 죄책감 가질 필요 없음. 밀린 건 일요일 주간종합이 배치 처리.

## 4. 강의 + 모델 선택 — 둘 다 이미 답이 있음

- **강의: 새로 고민하지 말 것.** [[3. Resources/내일배움카드_AI코딩_강의가이드|강의가이드]] 하단 "최종 선별 4개"(2026-07-12 확정)가 답: 모두의연구소 바이브코딩 2개(7/10~8/8, 진행 중) + Zapier(7/15~) + 패캠 AI비즈니스(8/19~). **단 하나만 결정하면 됨: 7월 3개 동시는 진도 80% 미달 위험 → Zapier를 다음 회차로 미루는 것을 권장** (가이드의 기존 권장과 동일). 새 강의 후보가 생기면 §3대로 "볼트 기준으로 평가해줘".
- **모델: 고르지 마라.** 한 줄 규칙 — **"고민되면 그냥 Sonnet."** 단순 분류·요약은 클로드가 알아서 Haiku로 내리고, 설계 충돌·모호함이 생기면 클로드가 먼저 "Fable로 승격할까요?"라고 승인을 구한다(ai공장짓기/CLAUDE.md 기존 정책). 혜미가 모델명을 기억할 필요가 없는 게 정상 상태다.

## 5. 스킬 ↔ 일반어 연결 (기억하지 말고 사전에 위임)

이미 그 용도의 파일이 있음: [[0. Docs/명령어_사전|명령어_사전]]. 부족한 건 "설치된 스킬 전수 목록"뿐.
- **실행 (Sonnet, 1시간, 혜미 몫: 없음)**: 설치된 스킬 전수 조사(공식: docx/pptx/xlsx/pdf/schedule/skill-creator + `.claude/skills/` 자작 스킬 + 플러그인) → 명령어_사전에 **"스킬 전체 지도"** 섹션 추가 — 각 스킬당 한 줄: "이렇게 말하면 됨"(일반어) / 뭘 하는 스킬인지 / 예시 1개. 이미 매핑된 건 중복 기재하지 않고 누락분만.
- **완료 기준**: 혜미가 스킬 이름을 하나도 몰라도 명령어_사전 한 페이지로 전부 시킬 수 있는 상태.

## 6. 앞으로도 깔끔 유지 (예방 규칙 — 오늘부터)

- **규칙 1**: 새 파일은 반드시 PARA 폴더 안에. 루트 신규 생성 금지 (기존 4파일 제외).
- **규칙 2**: 새 노트는 태그 + LLM위키_홈 해당 클러스터 입구에 연결 (기존 명령어_사전 규칙 재확인).
- **규칙 3**: 같은 주제 노트가 있으면 **업데이트, 새 파일 금지** (기존 규칙 재확인).
- **실행 (Sonnet, 15분)**: 일요일 21:00 주간종합 예약작업 프롬프트에 "볼트 위생 3종" 추가 — ①루트 떠돌이 파일 검사 ②_inbox 잔량 보고 ③깨진 링크 스캔(비-md 제외). E1의 sentinel 스캔과 같은 배치로.
- 이 4개가 돌면 **자료가 늘어도 진입점은 항상 2개**(핵심맥락 → LLM위키_홈)로 유지됨 — "많은데 못 찾는" 상태가 구조적으로 재발 안 함.
