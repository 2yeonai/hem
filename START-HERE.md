# START-HERE — 한 문장에서 승인까지 (비개발자용)

배치 위치: `vault/` 루트 · 함께 쓰는 파일: routing-policy.md(판정) · TASK-PACKET.md(발주) · RESULT-CARD.md(회수)
원칙: **당신이 하는 일은 3가지뿐이다 — 한 문장 쓰기, 붙여넣기, 판정하기.** 나머지는 AI가 한다.

---

## 0. 준비물 (최초 1회)

- vault 폴더에 이 파일 + routing-policy.md(`vault/prompts/`) + 템플릿 2종(`vault/templates/`)
- `usage.md` 템플릿을 `vault/ops/usage.md`로 복사 (제품별 신호등 🟢🟡🔴 표)
- Claude Cowork에 vault 폴더 연결 (사무국 역할: 패킷 폴더 생성·파일 정리·usage 갱신 담당)

## 1. 작업을 한 문장으로 쓴다

> 예: "정부지원사업 공고 3건을 비교해서 지원서 초안을 만들고 싶다"

두 문장이 나오면 작업이 2개라는 뜻이다. 쪼갠다.

## 2. 판정한다 (30초)

routing-policy.md의 §1→§2→§3 순서로: **시작 제품 / 티어 / 병렬 여부 / 검수자**가 나온다.
직접 보기 귀찮으면 Cowork에게: *"routing-policy로 이 작업 판정해줘: {한 문장}"*

## 3. 패킷을 만든다

Cowork(또는 총괄 ChatGPT)에게:
> "이 문장으로 TASK-PACKET 만들어줘: {한 문장}. 판정 결과는 {…}"

→ `vault/ops/tasks/{오늘날짜}_{작업명}/00_PACKET.md` 생성.
**병렬 작업이면** 총괄 AI에게 파생 분할까지 시킨다: *"이 패킷을 GPT·Gemini용 파생 패킷으로 분할해줘. 비범위가 서로 배타적이어야 한다"* → `10_`, `13_` 파일 생성.

## 4. 배포하고 실행시킨다 (붙여넣기)

- 각 담당 제품을 열고 해당 패킷(파생본)을 붙여넣는다. **패킷 밖 설명을 덧붙이지 않는다** — 덧붙이고 싶은 말이 있다면 패킷이 불완전한 것이므로 패킷을 고친다.
- 코드 작업이면 Claude Code에 패킷을 주면 내부 오케스트라(planner→developer→tester→reviewer)가 알아서 돈다. 교차 검증은 같은 repo에서 Codex를 열어 같은 패킷+결과 카드를 준다.
- 파일·문서 작업이면 Cowork에 준다.

## 5. 회수한다

- 각 AI의 마지막 지시는 항상 같다: **"결과를 RESULT-CARD 형식으로 반환해줘."**
- 돌아온 카드를 작업 폴더에 `2x_RESULT-{도구}.md`로 저장 (Cowork에 붙여넣으며 "저장해줘"라고 해도 된다).

## 6. 종합·검수시킨다

- 카드가 1장이면 바로 §7로.
- 카드가 2장 이상이면 총괄 ChatGPT에게 카드들만 주고: *"RESULT-CARD 취합 절차대로 종합해서 90_FINAL을 만들어줘"* → 결론 비교·상충 판정·나에게 묻는 질문이 정리되어 온다.
- 검수자는 routing-policy §3 규칙대로 (실행자와 다른 AI).

## 7. 판정한다 (사람의 유일한 고유 권한)

- 승인 → `99_APPROVED.md`에 한 줄 메모. 재사용 가치 있는 결정은 `vault/decisions/`로 승격 (Cowork에 위임 가능).
- 보완 → FINAL의 미해결 항목으로 **보완 패킷**을 만들어 §4부터 반복.
- 반려 → 폴더째 보관, decisions에 "왜 버렸는지" 한 줄.

---

## 예외 규칙: 절차를 생략할 때

**저난도·단발·되돌릴 수 있는 작업**(문구 몇 개, 간단 질문, 1회성 요약)은 패킷 없이 해당 제품에 바로 묻는다. 판단 기준 한 줄: *"결과가 틀려도 5분 안에 다시 시킬 수 있는가?"* — 예라면 절차 생략. **절차가 작업보다 무거우면 오케스트레이션이 아니라 관료제다.**

## 자주 막히는 지점

| 증상 | 처방 |
|---|---|
| 결과가 계속 어긋난다 | 패킷의 "완료 기준"이 체크 불가능한 문장인 경우가 대부분. 기준을 숫자/체크박스로 다시 쓴다 |
| 어떤 AI가 나은지 모르겠다 | 같은 패킷을 2노드에 주고 카드를 비교한다 — 고민보다 병렬이 싸다 |
| 한도 초과로 멈춤 | usage.md 🔴 표시 → routing-policy R8 대체표로 갈아탄다 |
| 대화가 길어져 맥락이 샌다 | 그 대화를 버리고, 현재 상태를 패킷으로 다시 만들어 새 대화에서 시작한다. 패킷이 있으면 대화는 일회용이다 |
| 폴더가 지저분해진다 | 주 1회 Cowork에게 "ops/tasks 정리해줘" (완료 폴더 아카이브) |
