# 6월 세미나 : 지원사업 빌드업과 합격전략 실전 정리본

> 작은 지원사업부터 사업화·R&D까지, 업종과 아이템에 맞게 단계별로 공략하는 정부지원사업 활용 전략

## 핵심 주제: 정부지원사업은 한 번에 큰돈 받는 게 아니라, 단계별 빌드업이다

이번 6월 세미나의 전체 메시지는 명확해.
**처음부터 1억짜리 사업을 노리는 게 아니라, 작은 지원사업 → 중간 사업화 지원 → R&D·스케일업 지원으로 쌓아가는 구조**야. 자료 첫 장에서도 “정부지원사업은 빌드업이다”라고 잡고 있고, 이후 전체 흐름이 그 논리로 이어져.

# 1. 정부지원금 공략 3단계 로드맵

## STEP 01. 진입장벽 낮은 50~100만 원 지원사업

이 단계는 **처음 정부지원사업에 진입하는 사람용**이야.
목적은 큰돈이 아니라 **이력 만들기, 결과물 만들기, 신청 경험 쌓기**야.

대표 유형은 다음이야.

| 구분     | 내용                                 |
| ------ | ---------------------------------- |
| 지원금 규모 | 50~100만 원                          |
| 대표 사업  | 경영안정 바우처, 고용보험료 환급, 기초 마케팅 지원      |
| 활용 목적  | 상세페이지, SNS 마케팅, 라이브커머스, 컨설팅, 점포 개선 |
| 핵심     | “나도 지원사업 수행 경험이 있다”는 기록 만들기        |

즉, 이 단계는 **합격 이력과 최소 결과물 확보 단계**야.
처음부터 “1억 주세요” 하면 기관 입장에선 “누구세요?”가 되니까, 일단 작은 사업으로 **수행 이력**을 만드는 구조.

---

## STEP 02. 1천만~5천만 원 사업화 지원

이 단계부터는 **실제 사업 고도화**가 들어가.
자료에서는 로컬기업육성, 소공인 판로개척지원, IP창출 종합패키지 등이 제시돼 있어. 

| 구분     | 내용                               |
| ------ | -------------------------------- |
| 지원금 규모 | 1천만~5천만 원                        |
| 대표 사업  | 로컬기업육성, 소공인 판로개척지원, IP창출 종합패키지   |
| 활용 목적  | 시제품, 브랜드, 패키지, 판로개척, 온라인몰, 지식재산권 |
| 핵심     | 제품·서비스를 실제 팔 수 있는 형태로 만드는 단계     |

이 단계는 혜미네 **온천꽃식물원**이랑 잘 맞아.
테라리움, 따오 캐릭터, 패키지, 지역성, 온라인 판매, 관광기념품화 전부 이 구간이야.

---

## STEP 03. 1억 이상 R&D·스케일업 지원

여기부터는 단순 사업화가 아니라 **기술성, 성능지표, 검증, 시장확장**이 필요해.
자료에서는 생활문화 혁신지원, AI 동행바우처, 강한소상공인 성장지원 등이 언급돼. 

| 구분     | 내용                                    |
| ------ | ------------------------------------- |
| 지원금 규모 | 1억~5억 이상                              |
| 대표 사업  | 생활문화 혁신지원, AI 동행바우처, 강한소상공인 성장지원, R&D |
| 활용 목적  | 기술개발, 성능검증, 플랫폼 개발, 대규모 판로, 글로벌 진출    |
| 핵심     | “좋은 아이디어”가 아니라 “검증 가능한 기술사업”이어야 함     |

여기는 **AI·플랫폼·소재·제조 등 기술검증형 아이템** 쪽이 더 잘 맞아.
특히 mo,on은 플랫폼·AI·운영관리 SaaS 쪽으로 연결 가능하고, 강사님 예시였던 Sunvera/UV 패치처럼 소재·성능검증형 아이템은 R&D형으로 연결할 수 있어.

# 2. 업종별 맞춤형 지원사업 매트릭스

자료에서 업종별로 맞는 지원사업을 정리해줬는데, 이게 되게 중요해.
왜냐하면 우리는 자꾸 “어떤 사업이 좋아?”를 보는데, 강의 기준은 반대야.

**내 업종과 현재 단계에 맞는 지원사업을 골라야 한다.**

| 업종/상황       | 맞는 지원사업                               |
| ----------- | ------------------------------------- |
| 음식점·카페      | 경영안정 바우처, 스마트상점, 고효율기기 지원, 지자체 경영환경개선 |
| 미용·네일·피부    | 특색간판, 경영환경개선, 온라인마케팅, 두루누리            |
| 온라인몰·제조판매   | 온라인판로 지원, 크라우드펀딩 지원, 서울/부산 판로개척       |
| 공방·소공인·제조   | 클린제조환경조성, 스마트제조, 소공인 판로개척             |
| 전통시장·골목상권   | 지역상권육성, 화재공제료 지원, 협업활성화 판로지원          |
| 직원 있는 소상공인  | 두루누리, 고용보험료 환급, 지자체 고용장려금             |
| 수출 가능 제품 보유 | K-소상공인 육성, 수출바우처 연계, 글로벌 마케팅          |
| 폐업 고민 사업자   | 희망리턴패키지, 재기사업화, 특화취업지원                |

혜미 기준으로 보면 이렇게 매칭돼.

| 적용 대상   | 맞는 지원사업 방향                                    |
| ------- | --------------------------------------------- |
| 온천꽃식물원  | 로컬기업육성, 생활문화 혁신지원, 소공인 판로개척, IP창출종합패키지        |
| mo,on   | AI 인재 실증형 창업패키지, AI 동행바우처, 디딤돌 R&D, 지역성장 예비창업 |
| 강사님 예시: Sunvera | 디딤돌 R&D, 초격차 뷰티, 글로벌협력형 R&D, 기술이전 사업화지원       |

결론은 이거야.
**온천꽃식물원은 ‘로컬·소상공인·생활문화·판로’ 쪽.**

**강사님 예시였던 Sunvera는 ‘기술개발·성능검증·뷰티 R&D’ 작성법을 이해하기 위한 참고사례.**

이 셋을 같은 지원사업에 억지로 넣으면 안 돼. 사업끼리 싸운다. 서류 안에서 내전 나면 심사위원은 바로 덮어.

# 3. 디딤돌 R&D는 창업지원금이 아니라 국가 R&D 과제다

6월 세미나 중간부터 핵심은 **디딤돌 R&D 합격전략**으로 넘어가.
자료에서는 디딤돌 R&D를 “아이디어를 연구개발계획서로 바꾸는 것”이라고 정리하고, 공고분석·성능지표·사업계획서 작성법을 중심으로 설명해. 

여기서 제일 중요한 관점은 이거야.

> 디딤돌은 “사업화 돈 주세요”가 아니라
> “이 기술을 개발하고, 성능지표로 검증하고, 시장에서 매출로 연결하겠습니다”를 증명하는 사업이다.

자료 결론에서도 **기술 + 성능지표 + 사업화 = 합격**이라고 정리돼 있어. 

즉, 디딤돌에 넣으려면 말투가 달라져야 해.

| 안 좋은 표현           | 좋은 표현                             |
| ----------------- | --------------------------------- |
| 플랫폼을 고도화하겠습니다     | 주요 기능 정상동작률 95% 이상을 달성하겠습니다       |
| AI 추천 성능을 개선하겠습니다 | 추천정확도 Precision@5 80% 이상을 달성하겠습니다 |
| 시스템을 안정화하겠습니다     | 100회 반복테스트 예외 발생률 1% 이하를 검증하겠습니다  |
| 빠르게 처리되게 하겠습니다    | 평균 API 응답속도 1초 이하를 달성하겠습니다        |

그러니까 R&D 문서에서는 감성 문장 금지.
“좋아질 거예요” 말고 **몇 %, 몇 초, 몇 회, 누가, 어디서 검증**이 나와야 해.
아… 이게 진짜 귀찮은데, 돈 주는 사람들은 숫자를 좋아해. 인간미 없는 척하지만 예산 앞에서는 다 숫자파야.

# 4. 성능지표는 합격의 핵심이다

이번 자료에서 가장 실무적으로 중요한 부분이 성능지표야.
성능지표는 **“해당 R&D 과제가 성공했는지 실패했는지를 객관적 수치로 판가름하는 최종 측정기준”**이라고 정의돼 있어. 

성능지표는 반드시 6요소로 정리해야 해.

| 요소      | 의미                        |
| ------- | ------------------------- |
| 지표명     | 무엇을 측정할 것인가               |
| 단위      | %, 초, MPa, TPS, 점/5점 등    |
| 현재수준    | 개발 전 현재 기술 수준             |
| 목표치     | R&D 종료 후 반드시 달성할 수치       |
| 목표치 근거  | 논문, 벤치마크, 경쟁사 데이터, 사용자 요구 |
| 평가방법·환경 | 누가, 어디서, 어떤 방식으로 검증할지     |

자료에서도 좋은 성능지표는 **측정 가능하고 재현 가능해야 한다**고 강조해. “지표를 읽는 순간 어떤 데이터로, 몇 번, 누가, 어떻게 측정하는지 떠올라야 한다”는 문장이 핵심이야.

# 6. 기술개발 목표는 4문장 구조로 쓴다

자료에서는 기술개발 목표를 4문장 구조로 정리해. 

| 순서 | 내용            |
| -- | ------------- |
| 1  | 고객문제 제시       |
| 2  | 기술적 한계 명시     |
| 3  | 핵심기술 정의       |
| 4  | 최종산출물 및 목표 제시 |

이걸 공식으로 쓰면 이렇게 돼.

> 기존 고객은 ○○ 문제를 겪고 있다.
> 현재 시장의 기존 제품·서비스는 ○○ 한계로 이를 충분히 해결하지 못한다.
> 본 과제는 ○○ 기술을 개발하여 이 문제를 해결하고자 한다.
> 최종적으로 ○○ 시제품/시스템을 개발하고, ○○ 성능지표를 달성한다.

mo,on에 적용하면:

# 8. 선행연구 자산은 “자랑”이 아니라 개발 리스크를 낮추는 증거다

| 보유자료        | 사업계획서에서의 역할         |
| ----------- | ------------------- |
| MVP / 프로토타입 | 개발기간 단축, 시행착오 감소 근거 |
| 원천데이터       | AI 모델 학습·검증 데이터셋    |

즉, “우리 이거 만들어봤어요”가 아니라
**“그래서 이번 과제 성공 가능성이 높습니다”**로 연결해야 해.

# 9. 사업화 전략은 4개 박스로 정리한다

자료에서는 구체적 사업화 전략을 다음 4개로 정리해. 

| 영역                  | 써야 할 내용                         |
| ------------------- | ------------------------------- |
| Market & Customer   | TAM/SAM/SOM 기반 목표시장과 타깃고객       |
| Revenue Model       | 구독료, 판매가, 라이선스 등 수익모델과 5개년 매출산식 |
| Sales Channels      | 자사몰, B2B 파트너십, 조달, 판로확보 방안      |
| Employment & Growth | 신규채용, 후속투자, TIPS, VC 투자유치       |

여기서 핵심은 매출표를 그냥 쓰면 안 된다는 거야.
자료에서도 **단가 × 수량 × 성장률** 같은 구체적 산식을 제시해야 한다고 강조해. 

예를 들면:

| 나쁜 매출계획      | 좋은 매출계획                                                          |
| ------------ | ---------------------------------------------------------------- |
| 1년차 매출 1억 달성 | 완제품 2만 원 × 2,000개 + DIY 키트 3만 원 × 1,000개 + 체험 2만 원 × 1,500명 = 1억 |
| 플랫폼 매출 3억    | 업체 SaaS 월 9.9만 원 × 100개사 × 12개월 + 예약수수료 8% × 거래액 2억 = 3억         |

이렇게 써야 심사위원이 “아, 계산은 해봤네”라고 봐.
안 그러면 그냥 희망사항. 희망사항은 예산서에서 힘이 없다. 너무 현실적이라 미안한데, 내가 현실 담당이라 어쩔 수 없어.

# 최종 정리

이번 6월 세미나는 이렇게 정리하면 돼.

**정부지원사업은 빌드업이다.**
처음엔 작은 지원사업으로 이력과 결과물을 만들고, 이후 1천만~5천만 원 사업화 지원으로 제품·판로·브랜드를 고도화한다. 1억 이상 R&D로 가려면 단순 아이디어가 아니라 기술개발 목표, 성능지표, 검증방법, 사업화 산식이 필요하다. 특히 디딤돌 R&D는 창업지원금이 아니라 국가 R&D 과제이므로, “무엇을 만들 것인가”보다 “어떻게 성능을 입증하고 매출로 연결할 것인가”가 핵심이다.

혜미야, 이 강의 기준으로 보면 이제 감이 딱 와.
**온천꽃식물원은 생활문화·로컬·판로형으로 가고, mo,on은 AI·플랫폼·SaaS형으로 가야 해. Sunvera/UV 패치는 강사님 예시라서 R&D·성능검증형 작성법 참고사례로만 분리하면 돼.**
셋 다 가능성은 있는데, 같은 말투로 쓰면 망하고 각자 다른 심사 언어로 써야 돼.
사업계획서도 결국 외국어야. “심사위원어”. 귀찮지만 우리가 이제 좀 알아듣기 시작했다.
