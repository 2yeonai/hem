# Fable 5 에이전트 설계 공략집 정리본 (lilcherry)

> 볼트 판정: 채택 3개(agent 운영표 양식 / 실패감지·월간리뷰 / 무료 마감일), 나머지는 볼트가 이미 상회 — [[0. Docs/외부트렌드자료_평가_2026-07-16|평가문서]] 참고.

## 핵심 사실
- Fable 5 = Opus 위 Mythos급. 유료 플랜 주간 사용량의 50%까지 무료 포함, **프로모션 종료 7/19 PT = 한국시간 7/20(월) 15:59**. 이후 100만 토큰당 입력 $10 / 출력 $50 크레딧 과금.
- 요지: 검색창처럼 쓰면 비쌈. "오래 걸리고 input 많고 반복될 일"에, 완성 기준을 먼저 말하고 맡겨야 값어치.

## 레벨 구조 (Lv.1→5)
1. **INPUT 리딩** — 사진·서류·초안 작은 input으로 모델이 읽는 방식 확인, 잘 나온 프롬프트 저장.
2. **업무 시스템화** — 회의록·피드백을 요약 말고 "연결해서" 패턴·판단기준·산출물 틀 추출 → 프롬프트 라이브러리(언제 쓰는지/input/완성 기준까지 같이 저장).
3. **SKILL 만들기** — 표로 끝내지 말고 재실행 가능한 산출물(비교 슬라이더, 대시보드)로. 수정 규칙까지 남기기.
4. **AGENT 설계** — 입력→판단→출력→승인→인계 5칸. 자동 실행 전 사람 승인, 처음 3회 시범운행, 3회 연속 무수정 통과 전 자동발송 금지. 실패 신호(잘못된 input/애매한 판단/틀린 출력/늦은 인계) 사전 정의. **월간 리뷰**로 수정 요청을 "앞으로 항상 이렇게 한다" 규칙으로 축적.
5. **레퍼런스 분석** — 해외 사례는 따라하기가 아니라 구조 분해→내 업무로 번역→작게 실험. 공식 사례(Stripe 5천만 줄 마이그레이션)와 소셜 주장(마인크래프트 클론/폴리마켓 봇)을 구분할 것.

## 기본 프롬프트 공식
"너는 [역할]처럼 봐줘. 원하는 결과는 [산출물]. 방법은 네가 정해도 됨. [포함 내용]은 필수. 끝내기 전에 스스로 검토하고 놓칠 만한 것까지 보완한 최종본만."
- 별첨 R.A.C.E. 틀: Role(역할) / Action(과업) / Context(대상·제약) / Expectation(형식·톤·금지사항).
