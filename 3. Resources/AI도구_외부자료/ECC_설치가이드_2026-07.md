# ECC (Everything Claude Code) 설치가이드 정리본

> 볼트 판정: **설치 보류** — [[0. Docs/외부트렌드자료_평가_2026-07-16|평가문서]] 참고. 설치는 혜미 승인 후에만. 아래는 나중에 그대로 쓸 수 있게 보존한 실행 절차.

## 핵심 3줄
- ECC = Claude Code용 오픈소스 확장팩(MIT 무료, ECC Pro만 유료). 네이티브 스킬 278개 + 서브에이전트 67개 + 훅 7종 (2026-07-15 기준, 숫자 자주 바뀜).
- 가장 쉬운 설치는 플러그인 방식이지만 **전체 설치 시 상시 컨텍스트 약 23,844 토큰** — 가볍게 쓰려면 minimal 프로필 또는 선택 스킬만.
- **플러그인 설치와 수동 설치를 절대 겹치지 말 것** (명령·훅 중복 발생).

## 방법 A — 플러그인 설치 (1분)
Claude Code 안에서:
```
/plugin marketplace add https://github.com/affaan-m/ECC
/plugin install ecc@ecc
/reload-plugins
/plugin list
```
- 플러그인은 rules를 자동 배포하지 못함 — 필요하면 공통 룰 + 쓰는 언어 룰만 수동 복사:
```powershell
git clone https://github.com/affaan-m/ECC.git ; cd ECC
New-Item -ItemType Directory -Force -Path "$HOME/.claude/rules/ecc" | Out-Null
Copy-Item -Recurse rules/common "$HOME/.claude/rules/ecc/"
# 언어 룰은 실제 쓰는 것만 (typescript/python 등)
```

## 방법 B — 가벼운 수동 선택 설치 (권장 경로, 설치하게 된다면)
```powershell
git clone https://github.com/affaan-m/ECC.git ; cd ECC
.\install.ps1 --profile minimal --target claude --dry-run   # 계획 확인
.\install.ps1 --profile minimal --target claude             # 실제 설치
```
- 프로필: minimal(낮은 컨텍스트, 훅 제외) → 기본 품질 워크플로 → 보안 중심 → 리서치 순.
- 특정 스킬만: `--skills tdd-workflow,security-scan,continuous-learning-v2 --dry-run` 확인 후 재실행.

## 검증·삭제
- 검증: `claude plugin details ecc@ecc` (스킬·훅·예상 컨텍스트 비용 표시) → 비중요 프로젝트에서 계획→TDD→보안스캔 순서로 시범.
- 보안 스캔(AgentShield)은 Node.js 22+ 필요: `npx ecc-agentshield --version`.
- 삭제: 플러그인은 `/plugin uninstall ecc@ecc`, 수동은 `node scripts/uninstall.js --dry-run` 확인 후 실행.
- continuous-learning은 "전부 영구 기억"이 아니라 반복 패턴을 instinct로 쌓는 별도 레이어 — 중요한 규칙은 CLAUDE.md에 직접 명시.

## 자주 막히는 지점 (요약)
- 명령 안 보임 → `claude update` 후 재시작 → `/reload-plugins` → Errors 탭.
- 룰 미적용 → 플러그인은 rules 자동설치 불가, 위 수동 복사 필요.
- 명령이 두 번씩 → 플러그인+수동 겹침. 플러그인 제거 → uninstall.js → 하나만 다시 선택.
