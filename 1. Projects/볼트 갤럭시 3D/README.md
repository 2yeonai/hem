# 볼트 갤럭시 3D (Vault Galaxy)

옵시디언 볼트의 그래프 뷰를 실제로 날아다닐 수 있는 3D 은하로 바꿔주는 사이트입니다.
Vite + Three.js(WebGL) 기반이고, 사용된 유일한 "에셋"은 아바타(다트/블롭) 하나뿐이며 나머지 환경(별, 노드, 오션, 오로라 등)은 전부 코드로 생성됩니다.

## 처음 실행하는 법

1. 터미널에서 이 폴더로 이동합니다.
2. `npm install` — 딱 한 번만 하면 됩니다 (Three.js, Vite 설치).
3. `npm run dev` — 볼트를 스캔해서 그래프 데이터를 만들고(`public/data/`), 로컬 서버를 켭니다.
4. 터미널에 뜨는 주소(보통 `http://localhost:5173`)를 브라우저(크롬 추천)에서 엽니다.

볼트 안의 노트를 수정/추가/삭제한 뒤 그래프에 반영하고 싶으면 `npm run build-graph` 를 다시 실행하고 페이지를 새로고침하면 됩니다. (그래프 구조는 미리 만들어두는 방식이라 실시간으로 자동 반영되지는 않습니다.)

기본적으로 이 프로젝트 폴더가 `볼트/1. Projects/볼트 갤럭시 3D/` 위치에 있다고 가정하고 볼트 루트를 자동으로 찾습니다. 다른 위치로 옮기면 이렇게 지정하세요:

```
VAULT_PATH="/원하는/볼트/경로" npm run build-graph
```

## 조작법

- **클릭**: 화면을 클릭하면 마우스가 잠기고(포인터 락) 조종을 시작합니다. 다시 풀려면 Esc.
- **마우스**: 좌우로 움직이면 진행 방향(heading)이 바뀌고, 위아래로 움직이면 시야만 기울어집니다(기체가 구르지는 않음).
- **W/A/S/D**: 현재 진행 방향 기준으로 전진/후진/좌우 이동.
- **R / F**: 상승 / 하강.
- **Space (누르고 있기)**: 하이퍼드라이브 — 전방으로 확 가속하면서 화면 가장자리에 속도선이 흐르는 연출.
- **속도 슬라이더**: 최고 속도 조절 (기본값은 낮게 설정해서 나중에 더 빠르게 갈 여지를 남겨둠).
- **퍼짐 정도 슬라이더**: 노드 사이 기본 링크 길이 — 그래프가 얼마나 넓게 퍼지는지.
- **허브/첨부파일 표시 토글**: 켜고 끄면 즉시 반영됩니다 (다시 빌드할 필요 없음).
- **V**: 가까이 있거나 조준 중인 노트로 "다이빙" — 그 노트의 텍스트로 이루어진 밤바다·오로라 연출로 들어갑니다. 다시 V 또는 Esc를 누르면 원래 있던 은하 위치로 정확히 돌아옵니다.
- **Z**: 젠 모드 — 모든 UI를 숨기고 환경만 봅니다.
- **크로스헤어를 노트에 맞추고 클릭** (또는 노트에 충분히 가까이 가면): 그 노트의 실제 텍스트를 보여주는 패널이 열립니다.

## 그래프가 어떻게 만들어지는가

`scripts/build-graph.mjs` 가 볼트를 한 번 스캔해서:

- 노트(`.md`) 각각을 노드로 만들고, `[[위키링크]]` 를 엣지로 연결합니다.
- 링크가 걸려있지만 실제 파일이 없는 이름(자주 인용하는 허브성 문서 등)은 **가상 허브 노드**로 만들어서, 옵시디언 그래프 뷰처럼 그런 이름들이 클러스터 중심이 되게 합니다.
- `![[이미지.png]]` 같은 첨부파일 임베드는 **첨부파일 노드**로 분류하고, 기본적으로 숨겨둡니다 (토글로 켤 수 있음).
- 각 노드의 연결 수(연결 많은 노트/허브일수록 큰 노드)를 계산합니다.
- 노트 본문 텍스트는 `graph.json` 에 다 넣지 않고, `public/data/notes/<번호>.json` 에 노트별로 따로 저장해서 패널을 열 때만 그때그때 불러옵니다.

## 테마 / 아바타 바꾸기

- 테마는 `src/config/themes.js` 에 색상표만 추가하면 새 테마가 됩니다 (딥 스페이스, 네뷸라 기본 제공).
- 아바타는 `src/avatar/avatars.js` 의 `REGISTRY` 에 새 모양을 코드로 추가하면 됩니다 (다트/블롭 기본 제공, 전부 코드로 그린 저폴리곤 도형이라 이미지 파일이 필요 없습니다). 다이빙 장면 안의 아바타도 같은 방식으로 바꿀 수 있습니다.

## 알려진 한계 (2026-07-14 기준)

- 이 세션 환경에는 브라우저/GPU가 없어서 **실제 화면에서 렌더링 테스트를 하지 못했습니다.** 문법 검사(`node --check`)와 빌드(`vite build`)는 전부 통과했지만, 처음 `npm run dev` 로 열어봤을 때 셰이더나 카메라 감각(속도감, 시야각 등)은 실제로 보면서 미세조정이 필요할 수 있습니다.
- 첨부파일 노드는 노트 안에서 옵시디언 임베드 문법(`![[파일명]]`)으로 직접 참조된 것만 인식합니다. 마크다운 이미지 문법(`![]()`)이나 `7. Attachments/` 폴더에 있지만 어디서도 링크되지 않은 파일은 그래프에 나타나지 않습니다.
- 링크 이름이 같은 노트가 여러 개 있으면(동명이인 문서) 그중 경로 순으로 하나만 자동 선택합니다 — 옵시디언의 실제 동명 해석 로직과 100% 같지는 않습니다.
- 물리 시뮬레이션은 노드 수가 수천 개 이상으로 크게 늘어나면 (현재 볼트 기준 약 670개 노드는 문제없음) 느려질 수 있습니다.
- 다이빙 장면(밤바다·오로라)의 "글자가 흩어졌다 다시 모이는" 연출은 개별 글자를 물리적으로 움직이는 대신, 텍스트가 입혀진 바다 표면을 번개가 칠 때마다 충격파 형태로 밀어내는 셰이더 방식으로 구현했습니다 — 멀리서 보면 의도한 느낌과 비슷하지만 글자 하나하나가 튀어오르는 것과는 결이 조금 다릅니다.
