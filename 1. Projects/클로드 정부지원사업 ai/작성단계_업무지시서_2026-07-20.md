# 작성단계 업무지시서 — 하위모델(Sonnet) 병렬 작성용 TASK-PACKET #정부지원 [gombeck1]

> **왜 이 문서가 필요한가**: 지금까지 공장은 "검수"만 있고 "작성 직원"이 없어서
> 짧은 선언문 수준의 부실 초안이 나왔다(외부감사 2026-07-19 결론).
> 이 문서는 사업계획서 4개 섹션을 **Sonnet 세션 4개에 나눠 동시에(병렬)** 쓰게 하는
> 업무지시서다. manifest v0.7.0의 작성_문제인식/실현가능성/성장전략/팀 stage의 실행 방법.

## 혜미 사용법 (3단계)

1. **입력표 채우기** : `입력표_템플릿.yaml`을 열어 빈칸을 채운다 (특히 대표자 경력 확인).
2. **병렬 작성** : 새 Cowork/클로드 세션(기본 모델, Sonnet급) 4개를 열고,
   아래 "공통 규칙 + 잠금값 카드 + 해당 시나리오 카드 + TASK-PACKET ①~④"를
   각각 하나씩 복붙한다. 4개가 동시에 돌아간다.
3. **통합·검증** : 4개 결과를 모아 TASK-PACKET ⑤(통합)를 실행하고, 그 세션에
   "content_gate 검증 실행해줘: `1. Projects/클로드 정부지원사업 ai/scripts/content_gate.py`"
   라고 말한다. `[PASS]`가 나와야 다음(심사→수정본)으로 넘어간다.

성공 확인 방법 : 게이트 실행 결과에 `arithmetic_error`, `program_clone_error`,
`customer_mix`가 하나도 없으면 성공. 하나라도 있으면 해당 섹션 세션에 오류 내용을
그대로 붙여넣고 "이 오류만 고쳐줘"라고 지시한다.

## 공통 규칙 (모든 작성 세션에 반드시 포함)

```
[역할] 너는 정부지원 사업계획서의 한 섹션만 담당하는 작성 직원이다. 담당 밖 섹션은 쓰지 않는다.

[서식] 모든 서술은 "○ 소제목(3~10자) : 내용(1~3문장, 구체 수치·근거 포함)" 불릿으로 쓴다.
긴 산문 단락 금지. 표는 경쟁사 비교·KPI 등 불가피한 경우만.

[상태값 규칙 — 가장 중요]
- 완료형("~함", "~보유함")으로 쓸 수 있는 것: 입력표에서 status가
  verified_fact / user_confirmed / (초창패 가상문서에 한해) simulation_assumption 인 항목뿐.
- planned_target 항목은 반드시 미래형("~할 계획임", "~을 목표로 검증함")으로 쓴다.
- 입력표에 없는 숫자를 새로 만들지 않는다. 필요하면 [근거 필요]로 표시하고 멈춘다.
- 존재하지 않는 인터뷰·계약·매출·MOU·특허를 완료된 것처럼 쓰면 안 된다.

[계산 규칙] 곱셈·합계가 들어가는 문장은 산식을 함께 쓰고 스스로 역산 검증한다.
(예: 300개소×5만원×12개월 = 1억8천만원 — 자릿수 틀리면 전체 문서가 탈락함)

[잠금값] 아래 잠금값 카드의 항목은 절대 바꾸지 않는다. 특히 1차 MVP 4종을
임의로 줄이거나 늘리지 않는다(강사 확정 사항).

[금지 표현] "책임 공방 판별"(→ "고객 응대 기준 확보·서비스 개선 자료"로),
"피부미용업 신고"(업종은 소프트웨어 개발·공급업), "어차피 차등 지급되니 상한선으로 신청",
의료 표현(진단·치료·처방·회복 효과 측정), 출처 없는 홍보 숫자("7년 연속 1위" 등).

[출력] 섹션 본문만 출력한다. 내부 메모·작성 과정 설명·경고 박스를 본문에 넣지 않는다.
```

```

과제명 : 회차별 서비스 이력과 돌봄 일정 연동을 통한 방문형 산후케어 운영품질 관리 SaaS, mo,on
지불고객(단일, 변경금지) : 소속 관리사 3~20명 규모로 고객 일정·회차 기록을
  전화·카카오톡·수기 메모로 관리하는 지역 중소 방문형 산후케어 업체
최종사용자 : 산모(조리원 퇴소 2주~3개월) + 회차별 방문업무를 수행하는 관리사
연계정보(고객 아님) : 산후도우미 근무시간·배우자/가족 돌봄 시간 = 예약 가능 시간을
  정하는 생활상 참고조건. 산후조리원·출장마사지·육아돌봄 = 확장시장(본문 전면 배치 금지)
중심 문제(1개) : 고객 일정과 회차별 서비스 정보가 전화·카카오톡·개인 수기 메모에
  분산되어, 관리사 변경·연속 방문 시 인수인계 누락과 일정 착오가 발생한다
파생 손실(4개) : ①산모 반복 설명 ②불만 발생 시 기록 부재 ③관리사별 품질 편차 ④일정 변경·취소
1차 MVP 4종(축소 금지, 하나의 흐름으로 서술) :
  이용 가능 시간 입력 → 일정 조율 → 회차별 수행기록 → 관리사 인수인계 → 고객 응대 자료 활용
확장 기능(본문 1줄로만) : 달빛등급제·우수 관리사 노출·교육·제휴 — 데이터 축적 이후
비의료 범위 : 진단·치료·처방·회복효과 측정 없음, 최소정보 수집, 체크형 입력 중심
시장규모(재계산 확정) : SAM = 전국 출장형 업체 300~500개소 × 구간제 구독료 —
  단일 5만원 기준 연 1.8억~3억원 / 구간제(9.9만~19.9만) 기준 연 3.6억~11.9억원(검증 가설)
가격(확정 아님, 검증 가설) : 시작형 9.9만(관리사 5명 이하)/운영형 19.9만(6~20명)/지점형 39.9만~
  — 인터뷰에서 5만·10만·20만 구간 지불의사 검증
대표자 경력(user_confirmed 필요) : 미용사(피부) 국가자격 + 뷰티 3종 제품 온라인 마케팅
  누적 100만개 판매 + AI 협업 도구로 화면·데이터 구조 직접 설계

```

```
  (서비스 1회 단가 10~15만원 시장에서 클레임 1건의 손실 > 월 구독료)
```

## TASK-PACKET ⑤ 초안통합

```
입력 : 섹션 초안 4개
할 일 :
1. 회사명·과제명·지불고객·MVP·KPI·예산 수치가 4개 섹션에서 완전히 일치하는지 교차 검사,
   불일치는 잠금값 카드 기준으로 통일
2. 문제(1-1) → 기능(2-1) → 검증방법(2-2) → KPI(3-2)가 1:1로 연결되는지 확인 —
   해결 기능 없는 문제, 문제와 무관한 기능, 측정 방법 없는 KPI는 삭제
3. 중복 문장 제거, 섹션 간 이음새 정리 (내용 추가 금지)
출력 : 통합본 1개 + "발견한 불일치 목록"(내부검토본용, 본문과 분리)
```
