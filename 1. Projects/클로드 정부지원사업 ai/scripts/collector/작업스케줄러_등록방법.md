# bizinfo 수집기 — 혜미 PC에서 매일 자동 실행하기 #정부지원 #ai공장

## 왜 Claude가 아니라 내 PC에서 돌려야 하나
기업마당(bizinfo) API 키는 발급 신청할 때 등록한 IP 또는 URL에서만 응답한다. Claude 작업 환경(샌드박스)의 접속 IP는 매번 바뀔 수 있어서 애초에 등록이 불가능하다 — 그래서 Claude 쪽에서는 이 API가 계속 빈 응답만 준다. 발급받은 그 IP(집/사무실 인터넷)에서 직접 돌려야 정상 응답을 받는다. 그게 이 컴퓨터다.

**좋은 소식**: 새 프로그램을 설치할 필요 없다. 이미 만들어진 `collect_bizinfo.py`를 그대로 쓰면 된다.

## 준비물 확인
1. 이 컴퓨터에 Python이 설치돼 있는지 확인 — 명령 프롬프트(cmd)를 열고 `python --version` 입력. 버전이 나오면 설치돼 있는 것.
   - 없으면 https://www.python.org/downloads/ 에서 설치(설치 시 "Add python.exe to PATH" 체크박스 꼭 체크).

## 1단계 — 실행용 배치파일 확인
`scripts\collector\run_daily_collect.bat` 파일을 이미 만들어뒀다(이 폴더에 있음). 더블클릭하면 바로 한 번 실행해볼 수 있다.

## 2단계 — 작업 스케줄러에 등록 (매일 07:00 자동 실행)
1. 시작 메뉴에서 "작업 스케줄러"(Task Scheduler) 검색해서 열기.
2. 오른쪽 "기본 작업 만들기" 클릭.
3. 이름: `기업마당 공고 수집` 입력 → 다음.
4. 트리거: "매일" 선택 → 다음 → 시작 시간 오전 7:00 설정 → 다음.
5. 동작: "프로그램 시작" 선택 → 다음.
6. "프로그램/스크립트" 칸에 아래 경로를 그대로 붙여넣기:
   ```
   C:\Users\82106\Desktop\hem\1. Projects\클로드 정부지원사업 ai\scripts\collector\run_daily_collect.bat
   ```
7. 마침.

## 3단계 — 정상 동작 확인
등록 다음 날, `scripts\collector\` 폴더 안에 `run_log.txt` 파일이 새로 생겼는지 확인. 그 안에 `EXIT_CODE: 0`이 보이면 정상이다(신규 공고 있음). `EXIT_CODE: 10`은 정상인데 새 공고가 없었다는 뜻(에러 아님). `EXIT_CODE: 1`이 3일 연속 뜨면 그때 알려달라 — 다음 Claude 세션에서 원인을 확인한다.

## 그다음은 어떻게 되나
수집된 결과는 `scripts\inbox\candidates_YYYY-MM-DD.json`에 쌓인다. 이건 아직 "사람이 확인 안 한" 후보 상태다 — Claude가 이 후보들을 실제 신청서 매칭에 쓰려면, 혜미가 각 카드를 보고 `reviewed: true`로 표시해야 한다(이 검수 단계는 아직 화면(UI)이 없어서, 다음에 Claude 세션을 열었을 때 "오늘 수집된 공고 후보 보여줘"라고 말하면 그 자리에서 같이 확인하고 표시하는 방식으로 처리하면 된다).
