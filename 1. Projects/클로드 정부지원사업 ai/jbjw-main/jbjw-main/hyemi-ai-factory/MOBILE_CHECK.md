# MOBILE_CHECK.md — 모바일 확인 방법

이 앱은 **PC에서 도는 로컬 Python 서버**다 (표준 라이브러리만, 설치 0). APK 단독 실행은 구조상 불가 —
휴대폰에서는 **같은 와이파이로 PC 서버에 접속**해 전 기능을 쓴다. (안드로이드 단독: Termux에서 `python app.py` 실행 가능)

## 접속

1. PC에서 실행: `python3 08_factory_tools/app.py`
   — 서버가 `0.0.0.0`으로 열리므로 실행하면 터미널에 **휴대폰용 주소가 바로 출력된다** (예: `http://192.168.0.12:8787`). 별도 IP 조회 불필요.
2. Windows에서 처음 실행 시 "Windows Defender 방화벽" 허용 창이 뜨면 **허용**(개인 네트워크) 클릭 — 클릭 안 하면 휴대폰에서 접속 안 됨.
3. 휴대폰 브라우저에서 터미널에 출력된 그 주소를 그대로 입력 (PC와 휴대폰이 **같은 와이파이**여야 함 — 모바일 데이터 불가).
4. 접속 안 되면: ①공유기의 "기기 간 통신 차단(AP 격리)" 옵션 확인 ②회사·카페 와이파이는 보안상 차단하는 경우가 많음 — 가정용 와이파이로 시도.
5. 이 방식은 **집 안에서만** 접속 가능 (인터넷에 공개되지 않음 — 보안상 의도된 동작).
6. APK: **없음** (이 앱은 서버형 — 꽃집 앱과 무관). 필요 시 차기 과제로 별도 결정

## 모바일 검수 순서

| # | 화면 | 확인 |
|---|---|---|
| 1 | 대시보드 | 프로젝트 목록·상태 태그 표시 |
| 2 | 📄 공고 업로드 | PDF/HWPX/DOCX 선택 → 업로드 → 프로젝트 자동 생성 |
| 3 | ① 공고 분석 탭 | 배점표 자동 추출·배점 전략·**공고 성격 분석**(목적/심사의도/위험전략) |
| 4 | 공정 진행바 | 0~16단계 색상 (초록/주황/회색/빨강/파랑) |
| 5 | 신청자·아이디어 입력 | 폼 저장 → 자동 재분석 → ③~⑦ 탭 갱신 |
| 6 | ⑧ 서류 초안 | 본문 6절이 빈 템플릿이 아닌지([확인 필요] 표시 포함) |
| 7 | ⑪ Export | application_draft.docx / slides.pptx / print_draft.html 다운로드 |
| 8 | 🤖 AI 설정 | 모드 저장, 키 암호화 저장, 예상 비용(원) 표시 |

## PDF 업로드 테스트

공고 PDF 1개 업로드 → ①탭에서 배점표·마감일이 자동으로 채워졌는지 → 틀리면 원문 대조 후 수정 입력.
스캔 PDF(텍스트 없음)면 경고가 뜬다 → 텍스트 붙여넣기 또는 AI 모드.

## 오류 시 캡처할 화면

① 오류 배너(빨간 박스) 전체 ② 해당 탭 상단(프로젝트명·탭 이름 포함) ③ 업로드 직후면 파일명이 보이는 화면 ④ PC 터미널의 마지막 10줄
