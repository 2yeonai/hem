실행일: 2026-07-03 / 도구: 08_factory_tools/mvp_notice_lock_generator.md / 입력: 07_samples/sample_input_mvp_test_01.md

## 1. 입력값 요약

- 공고: 소상공인 디지털 전환 지원사업 [전체 샘플 가정 — 실제 공고 아님]
- 신청자: 수제품 온라인 판매 3년차 개인사업자 [주요 수치 임의 가정]
- 후보 3개: ①주문·배송 자동화 ②후기→콘텐츠 변환 ③서류 AI 서비스
- 증빙: 5종 확보 가정, 3종 미확보

## 2. 출력 결과 요약

6종 전부 생성됨: notice_summary / applicant_risk_check / idea_comparison_table / scoring_strategy / selected_idea_lock / next_step_prompt.
추천 결과: 후보1 선정 추천(42/50), 후보3 부적합 판정(18/50) 후 확장 아이디어로 분리. 과제명·한줄요약 각 5개 생성. LOCK은 조건 미충족으로 DRAFT 유지 — 의도대로 동작.

## 3. 잘 작동한 부분

- 공고 목적 키워드 → 후보 평가 → 추천 근거가 배점 기준으로 일관되게 연결됨
- "재미있는 아이디어(후보3)"를 Gemini식 매력에도 불구하고 Monday식 기준으로 탈락시키고 보존 처리함
- 가정값 전체에 [확인 필요] 태그 유지, 사람 확정 지점에서 LOCK 차단이 작동함
- 탈락급 리스크(중복수혜·체납 미확인)를 STOP성 경고로 표면화함

## 4. 누락된 부분

- 예산표 초안 없음 — MVP 범위(11단계)에 예산표 작성이 없어서 4차 예산 검수가 가정 기반이 됨 → MVP v2에서 "예산 방향 초안" 단계 추가 검토
- 가점 항목 전략 공란 (입력 자체에 없었음)
- 증빙-주장 연결표 미생성 (9단계 워크플로우 범위라 MVP에는 없음 — 경계는 의도된 것이나 next_step에 명시 필요)

## 5. 사람이 확인해야 할 부분

① 실제 공고문 원문 (전 산출물의 전제) ② 중복수혜·체납·업종코드 ③ 수작업 시간·오배송 실측치 ④ 최종 아이디어 확정 ⑤ 견적서

## 6. 다음 수정 우선순위

1. 실제 공고문 입력 확보 (이것 없이는 전부 가정) 2. 성과목표 기준치 실측 3. MVP v2에 예산 방향 초안 단계 추가 4. 가점 전략 입력 항목 보강
