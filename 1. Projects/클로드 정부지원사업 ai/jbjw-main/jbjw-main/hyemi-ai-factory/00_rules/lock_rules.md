# lock_rules.md — 단계별 LOCK 규칙

## LOCK이란

한 단계의 결정을 잠정 고정하고 다음 단계로 넘어가는 것. LOCK 없이 다음 단계 진행 금지.

## 상태 정의

- `DRAFT`: 작성 중. 자유 수정 가능.
- `REVIEWED`: 심사위원 모드 검수 통과. 수정 시 검수 재실행.
- `LOCKED`: 사용자 확정. **사용자 승인 없이 변경 금지.** 변경 시 DECISIONS.md에 기록.

## LOCK 순서 (17단계)

1. 공고 기준 LOCK — 공고문·심사표·배점·양식·예산 기준 확인 완료
2. 프로젝트 분리 LOCK — 중심 아이템 1개 확정, 다른 아이템은 증빙·협력·확장으로만
3. 자격·리스크 LOCK — 신청 가능 판정 (사람 확정 필수)
4. 아이디어 후보 LOCK — 후보 목록 확정, 이후 새 후보는 별도 등록
5. 심사표 분석 LOCK — 배점 전략 확정
6. 최종 아이디어 LOCK — (사람 확정 필수)
7. 과제명 LOCK
8. 작성전략 LOCK
9. 본문 구조 LOCK
10. 증빙자료 LOCK
11. 예산표 LOCK — (사람 확정 필수)
12. 페이지·글자수 조정 LOCK
13. 위험 표현 제거 LOCK — danger_words 스캔 0건
14. 최종 제출본 LOCK — (사람 확정 필수)
15. 발표자료 LOCK
16. 발표 대본 LOCK
17. 발표 Q&A LOCK
