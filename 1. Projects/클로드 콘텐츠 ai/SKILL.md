# 콘텐츠 공장 (주제→발행) — SKILL

## 언제 이 스킬을 쓰나 (트리거)
- 혜미가 **주제 1개**를 주면서 "블로그/스레드 글 만들어서 올려줘"라고 할 때 (event: topic)
- 채널은 blog / threads 중 선택 가능 (기본: 둘 다)

## 언제 쓰지 않나
- 이미 있는 영상/글을 여러 채널로 **재가공**할 때 → 그건 별도의 "콘텐츠재생산 공장"(설계노트 5-8, 원본→숏폼/블로그/카드뉴스) 영역. 이 공장은 "주제→새 글 창작→발행" 전용.
- mo,on 사업 공식 문서(지원사업 신청서 등) → 정부지원사업 스킬 사용.

## 무엇을 하나 (한 문단)
주제를 받으면 자료를 조사하고, 모든 채널의 공통 뼈대인 **분석카드** 하나를 만든 뒤(채널 간 파생 금지 — 5-8 원칙), 블로그 초안과 스레드 초안을 각각 카드에서 파생시키고, 글자수·금지어 형식검사를 거쳐 **대표승인**(사람 게이트)을 받는다. 반려하면 사유별로 되돌아가 다시 만들고(내용→분석카드, 형식→형식게이트, 제약위반→자료조사), 승인된 것만 게시봇이 발행한 뒤 발행 장부에 기록한다. **승인 안 된 글은 게시봇이 자체 검증으로 한 번 더 차단**한다(이중방어 — company_charter 준수).

## 알려진 한계 (2026-07-12 기준 — 완성 선언 아님)
1. **전 stage가 mock** — 실제 글쓰기(LLM)·실제 게시 호출 없음. 구조 검증 단계.
2. **"진짜 자동화"의 현재 정확한 의미**: 주제 입력→초안 생성→혜미 승인→게시까지 이어지는 **공장 구조는 완성·검증됨**. 다만 실제 게시는 (a) 블로그 플랫폼 확정, (b) Threads 게시 수단 확정(공식 API 또는 Cowork 세션에서 Claude가 Chrome으로 게시)이 돼야 연결 가능. 어느 쪽이든 **대표승인 없이 자동 게시되는 일은 구조적으로 불가능**하게 설계돼 있고, 이는 혜미 본인의 운영 원칙(company_charter)이라 풀지 않는 게 맞음.
3. 톤·금지어·해시태그 규칙이 기본값 — 혜미의 기존 글 3~5개(골든셋)로 보정 필요.
4. 미결정 사항은 manifest.yaml의 open_questions 참고.
