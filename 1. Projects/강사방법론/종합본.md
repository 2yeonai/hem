# 강사방법론 종합본 — 강의 6회차 누적 정리 #정부지원

> 원 녹취(`강의녹취_원문/`)는 수정하지 않는다. 이 문서는 회차마다 ①판단기준 ②프로세스 ③화법 3층으로 뽑아 누적한 것. [[1. Projects/강사방법론/홍재우_페르소나카드|페르소나 카드]] v2가 이 내용 + 코칭원문 골든셋(G1~G8)을 합쳐 만들어졌다.

---

## 1강 — 판단기준
- "아무도 준비가 안 됐어요"라고 드러나는 사업계획서는 탈락 — 준비 부족이 그대로 보이면 안 됨.
- 무인 아이스크림 가게·백반집처럼 너무 일반적인 아이템은 감점. 단, 차별화 요소가 있으면 흔한 업종도 합격 가능("백반집도 잘 만들면 받을 수 있어요").
- 경쟁자 대비 최소 1~2가지 차별점이 있어야 함 — 핵심 판단기준.
- 모방이 어려운 기술·특허·전문자격이 있으면 가점. 민간자격증(예: 아로마테라피)은 경쟁력이 약함.
- 지원금은 "기술 업그레이드" + "홍보/사업화" 두 마리 토끼를 다 포함해야 함.
- 경쟁사가 전혀 없다는 주장은 위험 신호(수요 없음 또는 근거 없는 과장으로 읽힘).
- 인테리어·컴퓨터 등 처분 가능한 자산 매입 비용은 지원 배제 대상.
- 특허는 등록 완료가 아니라 "출원"만으로 충분 (목표는 사업 합격이지 특허 등록이 아님).
- 예비창업자 자격은 사업자 등록 시 영구 상실(비가역적 기준).
- 업력 3~5년 이상인데 매출 실적이 없으면 도약단계 지원 통과가 어려움.
- 예창패·초창패·청창사 등은 생애 1회 한정 — 재수혜 불가 이력이 자동 탈락 사유.
- 지원자격·이력은 법인/상호가 아니라 대표 개인 기준으로 판단됨.
- 자격 미달로 보여도 자가판단으로 포기하지 말고 반드시 접수할 것("안된다, 된다를 절대 스스로 판단하지 마십시오").

## 1강 — 프로세스
1. 강좌 구조: 1주 아이템(과제명) 선정 → 2주 특허출원/온라인 준비 → 3주 사업계획서 작성 → 4주 사업비 집행 노하우.
2. 자격 판단: 예비/초기/도약 여부 확인 → (필요시) 폐업·신규등록 전략 → K-스타트업 가입 → 창업기업확인서·중소기업확인서 발급 → 공동인증서 등록 → 신청서 제출.
3. 아이템 개발: AI로 경쟁사 조사 → 소비자 페인포인트 서사화 → 해결기술 도출 → 한 문장 과제명 → 10페이지로 확장.
4. 특허 출원: 특허고객번호 등록 → 공동인증서 → 셀프 접수(대리 접수 불가).
5. 접수 요령: 마감 임박 전 미리 접수, 마감 전까지 무제한 수정 가능.
6. 과제 제출: 게시판에 기재 → 강사 피드백 → 승인까지 반복 제출.

## 1강 — 화법
퀴즈식 진행("1번, 2번"), 핵심어 리듬감 있게 반복("접수, 접수, 접수"), "그죠?"류 동의 확인 태그질문, 실패를 정상화하는 안심 화법("괜찮아요"), 자기 실수를 유머로 노출, 스토리텔링 비유, 강한 명령형("무조건 접수"), "왜?"를 스스로 묻고 답하는 화법, 과장된 확률 비유, 난이도 상승 예고("한 단계 심화 들어갑니다").

## 2강 — 판단기준
- **구체성이 합격/불합격의 유일한 기준**: "계획이 구체적인 사람은 붙는 것이고, 모호하고 둥실하면 답이 없다."
- "내가 해결할 수 있는 것만이 문제다" — 고령화 같은 거대 사회문제는 안 됨. 문제-해결 1:1 구조 필수.
- 막연한 미래형 서술("~일지도 몰라요")은 감점. 이미 완료된 것처럼 서술해야 실행가능성 항목 통과.
- 흔한 클리셰 문제 제기(소상공인 폐업률 그래프 등)는 "지겹다"는 평가. 좁고 구체적인 문제로 좁혀야 신선함.
- 힘든 사연 위주 감성 서사보다 "나는 이미 이렇게 잘 해왔다"는 외부동기형 자기어필이 더 설득력 있음.
- 정량적 비교 지표가 있어야 좋은 솔루션 서술("보습 지속력 30분 → 내 걸 쓰면 ~").
- 표는 평가위원이 잘 안 읽음 — 문장 설명이 핵심, 표는 최소화(경쟁사비교·KPI 등 불가피한 경우만).
- 글씨 크기 11~12pt 이상 권장(평가위원 연령대 고려).
- 특허는 "출원사실증명"만 있으면 충분, 개수는 3개가 평균(너무 적으면 성의부족, 너무 많으면 부자연스러움).

## 2강 — 프로세스
1. 과제명(아이템) 확정.
2. 문제인식: 외부동기(경쟁사·고객 문제) → 내부동기(창업자 자신의 이유) 순으로 작성.
3. AI 프롬프트 순서 실행: 유사서비스 분석 → 기존 서비스 한계 비교 → 우리 솔루션 핵심특징 → 최신기술 반영 → 보유 데이터/파트너십/브랜드 정리.
4. 산출물을 웹사이트 제작용으로 재정리 → 러버블/스카이워크 등에 붙여 홈페이지·발표자료 생성.
5. 특허 주제 3개 AI 생성 → 임시명세서 작성 → 특허로(키프리스)에서 임시출원 접수.
6. 결과물을 PSST 양식에 옮기고 이미지마다 설명 문장 추가.
7. 일정표·예산은 마지막(4주차)에 작성.
8. 이번 시간 목표: PSST 1(문제인식)·2(솔루션)까지 완료.

## 2강 — 화법
"그죠?/아시겠죠?" 반복, 자기 낮추는 유머("귀찮으니까 저한테 시키는 거"), 가상 수강생 대사를 연기하듯 인용 후 답변, "빨리빨리/그냥 그냥" 반복 강조, 안심 고정문구("걱정하지 마세요, 다 할 수 있게 해드릴게요"), 난이도를 의도적으로 낮춰 말함, 돈에 비유한 절박함 강조("지면 한 장 한 장이 다 돈"), 촌철살인 격언식 문구, 직설적 지적 후 유머로 완화.

## 3강 — 판단기준
- 경제성/수익성 없으면 무조건 불합격 — 사회적 가치·공익성은 합격 기준이 아님("정부는 돈 많이 버는 거 좋아합니다").
- 시장 규모는 반드시 커야 함(작으면 감점).
- 표만 있고 설명 문장이 없으면 평가자가 안 읽음.
- 회사소개·연혁·상호 등은 넣지 않음(블라인드 방식).
- 경쟁사는 약한 곳이 아니라 시장 탑티어를 넣어야 유리.
- 그림/포지션맵/도식은 반드시 설명 문장을 같이 넣어야 함(그림만 두면 평가자가 안 봄).
- 계량목표(매출·고용·투자유치·수출) 4가지는 실제 계획이 없어도 반드시 명시해야 붙음.
- 숫자는 구체적일수록 허풍이 아니라 계획으로 인정됨.
- 기존 매출이 없으면 빈칸이 아니라 해당 표 자체를 삭제.
- 얼굴 사진·개인 프로필은 넣는 것이 유리(통념과 반대).
- 대표자(팀) 경력/네트워크/파트너사가 전혀 없으면 무조건 탈락.
- 사업계획서는 서류평가 통과용 소모품 — 평가위원은 사후 실제 이행 여부를 확인하지 않음.
- 분량은 짧아야 함(너무 길면 안 읽힘).
- 업무협약서·구매계약서 등 진짜 근거자료가 있으면 설득력 급상승. 단 조작은 금지.
- 초기 사업계획서는 B2C보다 B2B/B2G(확정계약)가 더 설득력 있음.

## 3강 — 프로세스
1. 전체 뼈대: PSST(문제→해결→성장전략→팀) 고정.
2. Scale-up 파트 순서: 시장규모 → 시장기회 → 비즈니스모델 → 경쟁사비교 → 수익모델 → 매출산정 → 계량목표 → 시장진출전략 → 사업추진일정.
3. 시장규모: TAM→SAM→SOM("탐샘섬")으로 좁힌 뒤 다시 확장 스토리로 풂.
4. 타겟 고객: 처음엔 좁고 뾰족하게, 이후 단계적으로 확장하는 서사.
5. 강좌 순서: 1강 PS → 2강 특허출원 → 3강 Scale-up+팀 → 4강 일반현황/요약/예산.
6. 제출 실무: 과제명 확정 → (임시)특허출원 → 사업계획서 작성 → 게시판 제출 → 강사 첨삭.

## 3강 — 화법
핵심어 겹쳐 말하기("돈이죠 돈 돈이"), 실패를 "꽝이다"로 단정, 실명 호명+즉석 퀴즈, 평가위원 관점을 대변하는 화법, 과장된 유머 비유, "그죠?/그쵸" 반복, 단호한 규범 제시 후 이유 설명, 명령형 지시("쓰세요/찍어주세요"), 부정적 피드백 후 곧바로 해법 제시.

## 4강 — 판단기준
- 협약기간(보통 6개월~1년) 내 실현 가능한 수준으로만 목표를 써야 함.
- 아이디어 수준으로만 쓰면 반드시 탈락.
- 준비가 안 돼 있어도 이미 다 준비된 것처럼 써야 함.
- 문제와 솔루션이 정확히 1:1 매칭돼야 함(안 그러면 논리구조 붕괴로 실패).
- 요약(초록)이 평가자가 가장 먼저 보는 부분 — 여기서 매력을 못 심으면 불리.
- 시제품·실물·이미지·목업이 있으면 합격률이 크게 오름.
- 가장 흔한 탈락 사유 두 가지: 시장 규모가 작아 보임 / 경쟁사 대비 차별성 안 보임.
- 매출/사업성이 최우선 기준 — 공익적이어도 돈이 안 되면 탈락.
- 실제로 없는 기술(AI·로봇 등)을 억지로 끼워 넣으면 기술평가위원에게 "물어뜯긴다."
- 구매의향서·계약서 등 강제성 없는 문서라도 있으면 합격 가능성이 훨씬 높아짐.
- 사업자명이 업종과 안 맞으면 어색함 — 애매한 영문/신조어 상호가 유리.
- 예산 비율·금액은 상세히 맞출 필요 없음("요건만 맞으면 넘어간다").
- 동일 대표가 다른 사업자로 같은 아이템을 각각 신청하면 중복지원 제재 대상.

## 4강 — 프로세스
1. 한 줄 과제명 확정 → PSST 본문 작성 → 본문 완성 후 과제명 재수정.
2. PSST 초안 → 제출 게시판 → 강사 피드백 → 첨삭 루프 반복.
3. 접수 → 서류평가 → 발표평가(약 1~1.5개월) → 협약·계좌개설 → 자금입금 → 사업비 집행(그해 12월 말 마감) → 중간보고 → 결과보고.
4. 산출물은 처음엔 5~6개 넉넉히 쓰고, 합격 후 실제 가능한 1~2개만 남겨 조정.
5. 요약 작성: 본문에 없는 임팩트 포인트를 새로 추가하는 방식을 추천(단순 AI 요약보다).
6. 예산 설계: 총사업비 결정 → 정부지원금 비율(약 70%) → 현금자부담(약 10%)·현물자부담(약 20%) 역산.
7. 사업비는 합격 직후 최대한 빨리 집행(재료비→외주비 순)해야 중간보고에 유리.
8. 배점 구조: 문제인식+실현가능성(PS)이 70%로 최대 비중, 시장성/팀(ST)은 상대적으로 작음.
9. 지속 사이클: 아이템 확정 → 계획서 완성 → 공고 확인하며 반복 지원 → 성과 쌓아 단계적 확장.

## 4강 — 화법
"그죠?" 반복, "제가 뭐라 그랬어요" 식 반복강조, "고수는/하수는" 이분법 대비, "이거는 경쟁이다" 반복, 실명 사례 스토리텔링, "저는 그렇게 봅니다"로 의견 명시, 극단적 반례로 경고성 일화, "그래서 이제" 화제전환, "붙는 사람/떨어지는 사람" 이분법 프레이밍, "합격할 수 있는 서류를 먼저 만들어라" 반복.

## 6월세미나 — 판단기준
- R&D 계획서에서 성능지표(정량 결과값)가 평가의 약 70% — "연구개발 지표 못 잡으면 꽝이다."
- 성능지표는 반드시 비교 대상(선행기술/제품)이 있어야 함.
- 비교 기준은 "국내 최고"로 잡을 것 — 무리하게 "세계 최고"급으로 잡지 말 것.
- 목표 수치를 허풍으로 잡으면 결과 제출 시 공인시험성적서로 검증되어 스스로 발목 잡음. 지표는 4~5개 중 1~2개만 난이도를 줌.
- 사업계획서 형식(페이지 수)은 합격과 무관 — 질이 중요.
- 기본 구조는 PSST + 성능지표(기술목표)를 더 첨예하게 넣은 것.
- 중복 수혜 제한 조항을 반드시 확인(예: 예비창업패키지 수혜자는 생활문화혁신 중복 불가).
- 매출액·근로자수 기준 초과 시 접수 자체 불가. 도박·유흥·약국·보험·부동산·수의업 등은 지원 대상 제외.
- 기술개발 항목에 예산 50% 이상 배정 등 배분 규정 준수 필요.
- 지원자가 적은(재공고·생소한 사업명) 사업이 실질적으로 합격률이 높음.

## 6월세미나 — 프로세스
1. 아이템 선정 → 표준 PSST 완성(이게 "준비 완료"의 기준).
2. R&D 신청 시 전자책 프롬프트 S1~S9를 단락별로 순서대로 실행 → 결과를 R&D 양식에 옮김.
3. 지원 규모는 백만 원대 → 천만 원대 → 1~3억대로 단계적 확장(작은 합격부터 쌓을 것).
4. 성능지표 설계: 선행기술 조사 → 현재 수준 파악 → 개선 목표 수치 설정 → 측정·검증 방법 결정.
5. R&D 지원 실무: IRIS에서 공고 확인 → 자격요건(업력 7년 이내, 매출 20억 이하) 확인 → 필요시 협업기관 섭외 → 접수.
6. 접수: 사업계획서 작성 → 사이트 가입 → 해당 서식만 작성 → 개인정보 동의 → 임시저장 후 마감 직전 최종본 교체 제출.
7. 연간 사이클: 매달 합격사례 프롬프트/양식 제공 → 지원 → 피드백 → 반복(1년 케어). 하반기 예산확대 대비 6~7월부터 준비.
8. 반려되면 게시판 재제출 → 강사가 히스토리 확인 후 재검토.

## 6월세미나 — 화법
"그죠?" 반복, 실시간 채팅 참여 유도, 자기 경험 근거("제가 이 사업 평가위원장입니다"), 과거-현재 대비 강조, 극단적 이분법("합격 아니면 탈락"), 겸손 후 전문성 반전("저도 잘 모르겠어요" → 구체적 수치 제시), 정보를 특별한 혜택처럼 프레이밍("이거 아는 사람 별로 없는데"), 격려형 마무리("겁내지 마시고"), 구어체 은어("잔잔바리"), 강한 단정어("무조건/절대/반드시/당연히").

## 7월세미나 — 판단기준
- 아이템의 독창성·혁신성은 필수가 아님 — 지역에서 꾸준히 사업하는 대표 위주로도 다수 선발.
- 자격/실적 요소(추론서 등)는 쌓일수록 합격률이 올라감.
- 벤처·이노비즈·여성기업 인증은 서류평가에서 가산점(보통 1점)으로 작용 — 근소한 점수차 경쟁에서 결정적.
- 매출 없어도 벤처 인증 가능(창업 3개월도 사례 있음). 반대로 매출 10억이어도 탈락 가능 — 결국 "아이템의 혁신성"을 봄.
- AI로 생성한 사업계획서를 그대로 베끼면 티가 남 — 반드시 본인 스타일로 각색해야 함.
- 전자동 생성 계획서만으로는 합격이 어려움.
- "우리 업종은 해당 없다"고 스스로 단정하지 말 것.
- 벤처 인증은 창업 3년 이내가 세제혜택 면에서 가장 유리하나, 3년 초과해도 인증 자체는 가능.
- 실사는 거의 없지만, 나올 경우 간판·인력·연구소장 재직 여부가 서류와 불일치하면 기존 혜택 환수.

## 7월세미나 — 프로세스
1. 실적·자격 쌓기(특허출원·고용·매출 등 점수 요소 확보).
2. 기업부설연구소(또는 연구전담부서) 설립: 코이타 사이트 가입 → 설립신고서 입력 → 조직도·도면·사진 첨부 → 접수(약 3일, 실사 없음).
3. 벤처기업 확인 신청: 시스템에 10개 항목(각 1000자 이내) 직접 입력해 접수.
4. 업무협약(MOU)·오픈이노베이션 참여로 신뢰도 추가 확보.
5. 사업계획서 작성: 기존 합격샘플 참고 → AI 초안 생성 → 반드시 본인 스타일로 수정·보완 후 제출.
6. 매달 그 달의 지원사업 공고 검토 → 최소 3건 이상 접수를 목표로 실행.

## 7월세미나 — 화법
"그죠?/그쵸" 거의 매 문장 반복, 확신 반복("됩니다. 됩니다. 제가 해봤습니다."), 실무경험 근거 제시("백 개 이상 해봤다"), 쉬운 비유로 개념 설명, 소유권 강조("이건 여러분들 겁니다"), 완곡한 지적("약간 돌려서 조금 아쉽네요"), 부정을 뒤집는 프레이밍("안된다는 생각보다 하겠다라고"), 질문 후 스스로 답변, 숫자 반복 강조, 자기 실수 소탈 인정, 수강생 공동체 애칭("동결이/도원결의")으로 소속감 조성.

## 통합 판단기준 요약 (교차 반복 테마)

여러 회차에서 공통으로 반복된 핵심 축:

1. **구체성 > 독창성** — 아이디어가 참신한지보다 "얼마나 구체적인가"가 합격/탈락을 가른다(2강·4강·7월세미나에서 반복).
2. **문제-해결 1:1 매칭** — 문제 정의가 좁고, 해결책과 정확히 대응해야 한다(2강·4강).
3. **정량화 최우선** — 계량목표·성능지표·정량 비교가 없으면 "허풍"으로 읽힌다(3강·4강·6월세미나).
4. **형식보다 문장** — 표·그림만 있고 설명 문장이 없으면 평가자가 안 읽는다(2강·3강).
5. **경제성/사업성이 전부** — 사회적 가치는 가점 요인이 아니며, 돈이 안 되면 무조건 탈락(3강·4강).
6. **증빙·인증이 결정적** — 특허출원·벤처인증·업무협약서 등 "진짜 서류"가 근소한 경쟁에서 당락을 가른다(1강·3강·4강·7월세미나).
7. **협약기간 내 실현가능성** — 협약기간(보통 6개월~1년) 안에 못 끝낼 목표는 감점(1강·4강).
8. **AI 산출물 그대로 제출 금지** — 반드시 본인 스타일로 각색해야 티가 안 남(7월세미나).
9. **자가판단으로 포기 금지** — 자격 미달로 보여도 반드시 접수(1강·7월세미나 공통 태도).
