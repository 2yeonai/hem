# 배치 결정 로드맵 (2026-07-13)

> Fable 5가 판단해서 작성. **실행자(Sonnet/Haiku)는 각 항목의 "실행 지시"를 그대로 따르고 판단을 새로 하지 않는다.**
> 이 문서로 2026-07-13 현재 볼트의 모든 CONFLICT·UNKNOWN·미결 항목이 결정됨.

## 결론 (한 문장)

**병목은 여전히 사람 입력 1건(방역 골든셋)이고, 그 외 전 항목은 오늘 결정 완료** — 잘림 버그는 sentinel 규칙으로 봉인, 인덱스 이슈 3건은 기계 작업으로 처리, 스크래핑·PPT·9개 후보는 전부 "지금 아님 + 트리거 명시"로 확정.

## C1. 잘림 버그 재발 방지 규칙 (신규 — 오늘부터 볼트 규칙)

- **결정**: 3중 규칙. ① md 신규 작성·전면 재작성은 bash heredoc + `wc -c` 바이트 확인으로만. ② 모든 md는 마지막 줄이 정확히 `<!-- ok -->` (sentinel). Edit/Write 도구로 수정한 직후에는 반드시 `verify_write.py <경로>` 실행 — sentinel 검사 통과(PASS) 출력이 없으면 그 작업은 미완료. ③ 잘림 발견 시 복구는 항상 git HEAD 대조.
- **근거**: 잘림은 항상 파일 맨 끝에서 발생(오늘 3회 실측) → 마지막 줄 고정 sentinel이면 잘림=sentinel 부재로 기계 판정 가능. 사후 휴리스틱보다 확실하고 저렴함.
- **실행 지시 (Sonnet, 45분, 혜미 몫: 없음)**:
  1. `ai공장짓기/scripts/verify_write.py`에 검사 추가: 대상 md의 마지막 비어있지 않은 줄이 `<!-- ok -->`가 아니면 FAIL 출력.
  2. 소급 배치 스크립트 작성·실행: 전체 md를 순회하며 마지막 줄에 `<!-- ok -->` 없으면 추가 (python 파일쓰기 — Edit/Write 도구 사용 금지). **실행 전 git 커밋 필수.**
  3. 매주 일요일 21:00 주간종합 예약 작업 프롬프트에 "전체 md sentinel 스캔, FAIL 시 git HEAD 복구" 1줄 추가.
  4. `0. Docs/명령어_사전.md`와 루트 CLAUDE.md에 규칙 ①② 기재.
- **완료 기준**: verify_write.py FAIL 검출 데모 1회 + 전체 md sentinel 스캔 FAIL 0 출력 붙여넣기.

## U1. `_inbox/옵시디언 변환본/00 인박스 자료 인덱스.md` 중복

- **결정**: 삭제하지 않고 `4. Archive/_inbox정리_2026-07/`로 **이동**. `3. Resources/정부지원사업_자료/` 판을 정본으로 확정.
- **근거**: 내용 거의 동일 + Resources판이 현행 경로 체계. "유일본 확인 전 삭제 금지" 규칙 준수 — 이동이면 되돌릴 수 있음.
- **실행**: Haiku, 5분. 혜미 몫: 없음 (이의 있으면 이동만 되돌리면 됨).
- **완료 기준**: 이동 후 `_inbox`에 해당 파일 없음 확인 출력.

## U2. 정본 인덱스의 깨진 위키링크 ~150개 ("옵시디언 변환본/..." → 실제는 `_원문/...__원문.md`)

- **결정**: 정본 인덱스(U1 이후 Resources판)의 링크를 실제 파일명으로 재매핑. 파일명 유사도 100% 확실한 것만 자동 치환, 애매한 것은 치환하지 말고 목록으로 보고.
- **근거**: 폴더명 변경 후 인덱스만 안 따라간 것이 유력 — 기계적 치환으로 충분, Fable 불필요.
- **실행 지시 (Sonnet, 1시간, 혜미 몫: 없음)**: python 스크립트로 `_원문/` 실제 파일명 목록 생성 → 인덱스 링크와 매칭표 작성 → 1:1 확정 건만 python 파일쓰기로 치환(C1 규칙 적용) → 잔여 미매칭 목록을 인덱스 하단에 "미해결 링크" 섹션으로 기록.
- **완료 기준**: 링크 해석 스캔 재실행 → 해당 인덱스의 깨진 링크 수 보고 (0 또는 미매칭 목록과 일치).

## U3. 원본 바이너리(hwp/pdf/pptx/mp4) 링크 ~250개 — vault에 실물 없음

- **결정**: **의도적 설계로 간주** (용량상 바이너리는 볼트 밖). 파일 추적·복사 안 함. 대신 링크 무결성 스캔 스크립트에 "비-md 확장자 링크는 검사 제외" 규칙 추가해 오탐 250건 재발 차단.
- **근거**: md 변환본이 전부 존재 = 정보 손실 없음. 바이너리를 볼트에 넣으면 sync·스캔 비용만 증가.
- **혜미 몫 (yes/no 1개)**: "원본 파일은 볼트 밖(원래 위치) 보관이 맞다" — 아니오일 때만 복사 계획 수립. 기본값: 예.
- **실행**: Haiku, 10분 (스캔 제외 규칙 추가).

## D1. 방역 미정봇 3개 (현장분리/견적금액산정/현장사진) 필요 여부

- **완료 기준**: 골든셋 3건+yaml 검증 통과 → 배치 PASS 로그 → decision-log에 3봇 판정 3줄.
