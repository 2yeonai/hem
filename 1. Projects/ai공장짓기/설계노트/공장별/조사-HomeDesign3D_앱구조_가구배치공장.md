# Home Design 3D 앱 구조 조사 — 가구배치 공장 벤치마크

> 목적: 3D 가구배치 공장(5-4)을 만들 때 참고할 기준 앱. 출처: App Store 공식 설명(2026-07-16 조사, v5.8.2 기준).
> 벤치마크(benchmark) = 잘 만든 기존 제품을 기준 삼아 비교·참고하는 것.

## 1. 앱 개요

- 개발사 Anuman(Microids), 전세계 6,000만+ 사용자, 평점 4.3(10.5만 리뷰)
- 무료 + 구독(월 $12.99~, 연 $44.99~). 오프라인 완전 동작, 튜토리얼 내장
- 핵심 한 줄: **"2D로 그리고, 3D로 걸어다니며 확인한다"**

## 2. 핵심 구조 (2D→3D 파이프라인)

앱의 작업 흐름이 우리 공장 stages와 거의 1:1로 대응된다.

| 앱의 단계 | 앱 기능 | 가구배치 공장 대응 stage |
|---|---|---|
| ① 평면 입력 | 2D로 벽 그리기, 치수 직접 입력(수동 정밀), 기존 도면 이미지 위에 덧그리기 | interpret (폼입력이면 스킵) |
| ② 구조 확정 | 벽 두께·높이·각도 조정, 다층(무제한 층) | validate |
| ③ 가구 배치 | 카탈로그(방 카테고리별 정렬)에서 드래그&드롭, 그리드 스냅, 크기·색 커스텀 | solve |
| ④ 배치 확인 | 3D 실시간 워크스루, 표면 위 오브젝트 배치, 3D 모드에서 치수 표시 | judge + render |
| ⑤ 결과 공유 | 버전 여러 개 저장·비교, 스크린샷, 커뮤니티 공유, VR/3D프린터 내보내기 | explain / 산출물 |

## 3. 우리 공장에 가져올 설계 포인트

1. **치수 수동 입력 옵션** — 앱도 "손으로 그리기"와 "정확한 치수 입력"을 병행 제공. 우리의 echo-back 승인(환각치수 방지)과 같은 철학. 폼 입력 경로를 1급으로 유지할 근거.
2. **그리드 스냅** — 배치 좌표를 격자에 붙임. 회전각도 이산화(각도를 몇 가지로 제한하는 것) 결정에 참고 — 앱도 연속 배치보다 스냅을 기본으로 한다.
3. **버전 저장·비교** — 같은 방에 배치안 여러 개를 저장해 나란히 비교. 우리 solve가 후보 배치 여러 개를 내놓는 구조와 맞음 → 출력에 "배치안 N개 + 비교" 형태 고려.
4. **2D 입력 / 3D 확인의 분리** — 입력은 단순한 2D, 확인은 3D 렌더. 우리도 solve/judge는 2D 좌표 수학으로, render만 3D(또는 등각뷰)로 하면 됨. 3D는 확인용이지 판정용이 아니다.
5. **오프라인 동작** — 판정 코어에 모델 개입 금지 원칙과 일치(코어는 로컬 결정론).
6. **치수 표시 오버레이** — 3D 뷰에도 치수를 띄움(v5.6.1 추가). render 산출물에 치수 주석 포함할 것.

## 4. 반면교사 (사용자 불만에서)

- 대형 프로젝트에서 성능 저하(구조물 3~4개 넘으면 멈춤) → 우리는 방 1개 단위로 범위 제한
- 구독 전환에 대한 반발 큼 → 과금 설계 시 참고
- "배치 불가"와 "오류"의 구분이 불명확하다는 불만 → 설계노트의 infeasible/not_found 분리 원칙 재확인
