# 4개 공장 감사 리포트 + 실행 로드맵 (2026-07-09)

> 이 문서는 Fable 5가 판단해서 작성했고, **실행은 Sonnet 이하 모델이 이 문서만 보고 할 수 있게** 쓰여 있다.
> 실행자(모델)는 각 항목의 "실행 지시"를 그대로 따르면 되고, 판단을 새로 할 필요 없다.

## 감사 결론 (한 문장)

**현재 병목은 AI도 코드도 아니라 "사람 입력 1건"이다** — 방역 골든셋에 실제 사례를 채우는 것. 이것이 끝나야 방역 검증 → 꽃집 재사용 → 플랫폼 설계로 이어지는 전체 라인이 움직인다.

## 프로젝트별 진단

| 공장 | 상태 | 진단 | 다음 행동 주체 |
|---|---|---|---|
| ① 정부지원 | v0.5.0, 90% | 중간구간 완성. 앞단(스크래핑)·뒷단(PPT) 미착수. **가장 좋은 검증법은 mo,on 본인 신청서에 실전 투입** | 사람+Sonnet |
| ② 꽃집 | 설계 완료, 코드 0% | 폴더 루트가 정부지원 사본으로 오염됨(혼동 유발). 방역 재사용률 64~86%라 방역 검증 후 착수가 효율적 | Sonnet |
| ③ 방역 | 구조 100%, 검증 대기 | **유일한 병목: golden_set_template.yaml 실사례 입력.** 구조 고민 5개는 골든셋 도착 전 판단 보류(올바른 결정) | **사람 (오늘)** |
| ④ 플랫폼 | 미착수 | ①②③ 경험 축적 전 착수 금지 유지. Fable용 총설계 프롬프트 3개는 이미 준비됨 — 크레딧 여유 있을 때 1회 실행 가치 있음 | Fable(승인 후) |

## 우선순위 로드맵 (기대수익 높은 순)

### R1. 방역 골든셋 채우기 — 이번 주 최우선
- **왜**: 전체 라인의 유일한 병목. 모델 등급 무관, 사람 기억 속 실제 사례가 필요.
- **모델**: Sonnet (인터뷰 진행자 역할)
- **실행 지시**: 새 Cowork 세션에서 아래 프롬프트 실행.
  ```
  1. Projects/클로드 방역 ai/golden_set_template.yaml 을 읽고, 나를 인터뷰해서 실제 사례로 채워줘.
  한 번에 질문 하나씩만. 내가 기억 안 나는 필드는 "미확인"으로 두고 넘어가.
  다 채우면 yaml 문법 검증(python3 -c "import yaml; yaml.safe_load(open(...))")까지 하고,
  검증 통과 결과를 붙여넣어 보고해.
  ```
- **완료 기준**: golden_set.yaml에 실사례 3건 이상 + yaml 검증 통과 출력 붙여넣기.

### R2. 방역 12건 배치테스트 재실행 + 미정 봇 3개 판단
- **모델**: Sonnet 실행. 결과가 기존 설계와 **충돌할 때만** Fable 호출(승인 후).
- **실행 지시**: `python3 scripts/test_12cases_batch.py` 실행 → 결과 전문 붙여넣기 → 현장분리/견적금액산정/현장사진 봇 필요 여부를 골든셋 근거로 판정. 판정 근거는 decision-log에 기록.
- **완료 기준**: PASS 로그 붙여넣기 + decision-log에 3개 봇 판정 항목 추가.

### R3. 정부지원 스킬 실전 투입 (mo,on 예비창업패키지)
- **왜**: 자기 사업 신청서가 최고의 실전 테스트. 스킬 검증과 실익이 동시에 나옴.
- **모델**: Sonnet (8인심사위원 서브시스템 그대로 사용). 접수 제출은 **반드시 사람**.
- **완료 기준**: 신청서 초안 + 8인심사위원 점수표 + 감점 항목 수정본.

### R4. 꽃집 폴더 정리 + 코드 착수
- **모델**: 정리는 Haiku/Sonnet, 코드는 Sonnet (방역 scripts/run.py 복제·수정).
- **실행 지시**: ⚠️ 삭제 금지 — 꽃집 루트의 정부지원 사본(SKILL.md, manifest.yaml, scripts/, test/)은 `4. Archive/꽃집폴더_정부지원사본_2026-07/`로 **이동**만. 이동 전 정부지원 원본 폴더와 diff로 동일본인지 확인하고 diff 결과 붙여넣기. 이후 14봇 분류표 기준으로 꽃집 자체 manifest.yaml 작성 → `ai공장짓기/scripts/validate_manifest.py`로 검증.
- **완료 기준**: 꽃집 루트에 꽃집 내용만 남음 + 새 manifest 검증 통과 출력.

### R5. 정부지원 스크래핑 — 설계만 Fable, 구현은 Sonnet
- **시점**: R1~R3 완료 후. 외부 사이트 의존이라 유지비가 있으므로 서두르지 않는다.
- **Fable 설계 시 질문 목록**(이대로 물으면 됨): 대상 공고 사이트 목록 / 수집 주기(triggers.schedule) / 실패 시 재시도 정책 / 수집물→매칭 스킬 io_contract.

### R6. AI회사 플랫폼 총설계 — 준비된 Fable 프롬프트 3개 실행
- **시점**: 방역 검증 완료 후, Fable 크레딧 여유 있는 날. 산출물은 반드시 `ai공장짓기/`에 저장 + decision-log 기록.

## 그만할 것 3가지 (근거 포함)

1. **트렌드 글 따라 세팅 갈아엎기** — VFF("Fable 단종" 전제부터 허위), NVIDIA 프록시(출처불명 스크립트 `curl|bash`, 보안 위험) 등. 현재 시스템(manifest/quality_gate/HANDOFF)이 이미 그 글들이 권하는 것보다 정교함. 새 도구는 "현 구조에 끼울 수 있는가"만 기준으로 평가.
2. **9개 신규 공장 후보 확장 검토** — ①②③ 완성 전 신규 라인 검토는 컨텍스트만 소모. 후보 목록은 이미 기록돼 있으니 잊어도 됨.
3. **.stale 파일 수동 정리 고민** — verify_write.py가 있으므로 방치가 정답. 단, 삭제는 유일본 여부 확인 전 금지(기존 규칙 유지).
