# 정부지원 UI 참조 #정부지원

근거 문서: `1. Projects/클로드 정부지원사업 ai/HANDOFF.md`, `manifest.yaml`(정부지원_manifest_v2.yaml 포함). 정부지원 공장은 매칭(어떤 지원사업이 맞는지) → 신청서 초안 생성 → 대표자 승인 → 발표자료 백엔드까지 이어지는 흐름입니다. 다른 3개 공장과 달리 **"자주 반복하는 작업"이 아니라 "가끔, 마감이 걸린 복잡한 작업"**이라는 점이 화면 설계에서 가장 중요한 차이입니다.

## 이 앱의 성격과 마법사(wizard) 패턴 채택 이유

조사 결과, 정부지원 신청서처럼 "복잡하고 드물게 하는 작업"에는 마법사형(단계별) 입력이 오류를 줄이는 데 효과적이라는 것이 공통적인 결론입니다(NN/g, Eleken 등). 한 화면에 모든 항목을 다 넣으면 대표자가 어디까지 채웠는지 잃어버리기 쉽고, 정부지원 신청서는 항목 수가 많아 단계별 진행이 특히 유리합니다. 단, 마법사는 10단계를 넘기지 않는 것이 권장되므로, 아래처럼 큰 묶음 5단계로 설계합니다.

## 화면 목록

| 화면 | 목적 | 사용자 |
|---|---|---|
| ① 매칭 시작 입력 | 사업 기본정보 입력(업종/규모/지역/필요자금 등) | 대표자 |
| ② 매칭 결과 목록 | 적합한 지원사업 리스트 + 적합도 | 대표자 |
| ③ 신청서 마법사(5단계) | 선택한 사업의 신청서 항목을 단계별로 입력 | 대표자 |
| ④ 초안 미리보기 | AI가 만든 신청서 초안 전체 보기 | 대표자 |
| ⑤ 문서승인(승인카드) | 승인/반려, 버전 관리 | 대표자 |
| ⑥ 마감 캘린더 | 신청 중인 사업들의 마감일 목록 | 대표자 |

## 화면별 구성 제안

### ① 매칭 시작 입력

- 단일 폼: 업종, 사업규모(예비창업/1년미만/3년미만 등), 지역, 필요자금 범위, 키워드(자유입력)
- 하단 [매칭 시작] 버튼 → AI가 검색·매칭 실행

### ② 매칭 결과 목록

- 카드 리스트: 사업명, 지원기관, 마감일(D-day 배지), 적합도(%) 또는 상/중/하 배지
- 마감이 임박한 순으로 기본 정렬, `--warn`/`--fail` 색으로 마감 임박 표시
- 카드를 누르면 ③ 신청서 마법사 시작

### ③ 신청서 마법사 (5단계, 상단 진행바 표시)

```
[1 사업개요] — [2 지원동기] — [3 사업화 계획] — [4 성과목표] — [5 첨부서류]
   ●진행중        ○              ○               ○              ○
```

- 각 단계는 해당 섹션의 항목만 보여줌(방역스킬_현황보고.md 사례처럼, 항목별 "여기에 뭘 적는지" 쉬운 말 설명을 함께 붙임 — 비개발자 사용자 기준)
- 이전/다음 버튼으로 이동, 다음 단계로 넘어가도 이전 입력값이 사라지지 않게 유지(마법사 패턴의 흔한 실수 방지 포인트)
- 각 단계마다 [AI 초안 생성] 보조 버튼 — 대표자가 키워드만 입력하면 AI가 문장으로 확장한 초안 제안(항상 초안일 뿐, 자동 제출 아님)
- 마지막 단계 완료 시 ④ 초안 미리보기로 자동 이동

### ④ 초안 미리보기

- 신청서 전체를 섹션별로 세로 스크롤로 보여줌(마법사 단계 구분 없이 한 문서로 합쳐서)
- AI가 생성한 문단은 배경 톤을 `--card-hover`로 살짝 구분
- 섹션별 [수정] 버튼 → 해당 마법사 단계로 되돌아가서 수정
- 하단 [검수 요청] → ⑤ 문서승인으로 이동

### ⑤ 문서승인 (공통_패턴.md 승인카드 그대로 적용)

- 머리말: "2026년 예비창업패키지 신청서 초안 v2"
- 버전 이력 표시(approval 블록의 version/version_history 그대로 노출) — 이전 버전과 비교 보기 링크
- 버튼: [승인] `--ok`, [반려] `--fail`(반려사유 필수 — 예: "예산 항목명 재확인 필요")
- 승인 시 문서 잠금 + "제출은 대표자가 직접" 안내 문구 고정 노출(회사 헌장상 AI가 실제 제출을 대행하지 않음)

### ⑥ 마감 캘린더

- 신청 진행 중인 사업들을 마감일 순으로 리스트(캘린더 그리드보다 리스트가 이 상황엔 더 실용적 — 진행 중 건수가 많지 않음)
- 상태 배지: 매칭됨/작성중/승인대기/승인완료/제출완료(제출완료는 대표자가 직접 표시하는 수동 체크 — AI가 자동으로 "제출완료" 처리하지 않음)

## 색상/상태 적용 요약 (정부지원 전용)

| 상태 | 화면 | 색 |
|---|---|---|
| 마감 임박(7일 이내) | 매칭결과/캘린더 | `--warn` |
| 마감 초과/마감 3일 이내 | 매칭결과/캘린더 | `--fail` |
| AI 초안 생성됨 | 마법사/미리보기 | `--accent` |
| 대표자 승인완료·잠금 | 문서승인/캘린더 | `--ok` |
| 반려 | 문서승인 | `--fail` |

## 조사 근거

마법사(wizard) UI는 "복잡하고 드물게 하는 작업"에 적합하며 10단계 이하, 진행 표시줄, 뒤로가기 허용이 권장된다는 조사 결과(NN/g, Eleken, VA.gov 디자인 시스템)를 반영했습니다. 정부지원 신청서는 정확히 이 조건(복잡함+드묾)에 해당해 마법사 패턴을 기본으로 채택했습니다.
