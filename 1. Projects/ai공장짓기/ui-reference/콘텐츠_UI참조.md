# 콘텐츠 UI 참조 #콘텐츠

근거 문서: `1. Projects/클로드 콘텐츠 ai/channel_config.yaml`, `HANDOFF.md`. 채널은 3곳(@maasittang 진주 맛집, @2yeon_sz 육아, 네이버 블로그 eunoia9496)이고, 주 3회(월·수·금 오후 5시) 발행 스케줄이 이미 확정돼 있습니다. 채널마다 형식은 다르지만 기초자료(분석카드+글스타일+사진)는 공유하는 구조라, 화면도 "하나의 콘텐츠에서 여러 채널로 파생"되는 걸 보여줄 수 있어야 합니다.

## 이 앱의 성격

콘텐츠 공장은 방역·꽃집처럼 현장 데이터를 다루는 게 아니라 **일정(캘린더) + 승인 + 게시 상태 추적**이 핵심입니다. 조사 결과, 콘텐츠 캘린더형 앱은 날짜/시간/플랫폼/본문/자산/링크를 최소 필드로 하고, 상태를 초안→검토중→승인→예약→게시 단계로 명시하는 패턴이 일반적입니다. 이 앱도 그 패턴을 기본 골격으로 씁니다.

## 화면 목록

| 화면 | 목적 | 사용자 |
|---|---|---|
| ① 콘텐츠 캘린더 | 이번 주/이번 달 발행 예정 콘텐츠를 한눈에 | 혜미 |
| ② 콘텐츠 작성/AI초안 | 기초자료 입력 → AI가 채널별 초안 생성 | 혜미 |
| ③ 채널별 미리보기 | 같은 콘텐츠가 채널마다 어떻게 나오는지 비교 | 혜미 |
| ④ 승인(승인카드) | 게시 승인/반려 | 혜미 |
| ⑤ 게시 이력/성과 | 게시된 콘텐츠와 상태 확인 | 혜미 |

## 화면별 구성 제안

### ① 콘텐츠 캘린더

- 주간 뷰 기본(월·수·금 발행 주기와 맞춤), 월간 뷰 전환 가능
- 각 날짜 칸에 채널별 색 점(마시땅/이연/네이버블로그 각각 다른 강조색 배지) + 상태 배지(초안/검토중/승인/예약/게시완료)
- 빈 칸(발행 예정인데 콘텐츠 없음)은 `--warn` 테두리로 표시해 놓침 방지
- 칸을 누르면 ③ 채널별 미리보기로 이동

### ② 콘텐츠 작성/AI초안

- 상단: 기초자료 입력 영역 — 분석카드 요약, 혜미가 직접 쓴 글(있으면 붙여넣기), 사진 업로드(여러 장)
- [채널별 초안 생성] 버튼 → 마시땅(맛집 톤)/이연(육아 톤)/네이버블로그(정보+수익화 구조) 3개 초안을 동시에 생성
- 각 초안은 접었다 펼 수 있는 카드로 나열, 채널명 배지로 구분

### ③ 채널별 미리보기

- 가로 3분할(데스크톱) 또는 세로 스와이프(모바일): 마시땅 미리보기 / 이연 미리보기 / 네이버블로그 미리보기
- 실제 게시 화면과 최대한 비슷한 모양으로(인스타그램은 정사각 이미지+캡션, 블로그는 제목+본문+사진 배치) — 조사에 따르면 콘텐츠 캘린더 앱들은 플랫폼별 미리보기(platform-specific preview)를 승인 전에 반드시 보여주는 것이 표준 패턴
- 각 미리보기 하단에 [이 채널만 수정] 버튼

### ④ 승인 (공통_패턴.md 승인카드 적용)

- 머리말: "8/3(월) 오후5시 발행 예정 — 마시땅 진주 맛집"
- 요약: 채널, 예정 발행 시각, 이미지 장수, 본문 글자수
- 미리보기 펼치기
- 버튼: [승인] `--ok`, [반려] `--fail`(반려사유 입력) — channel_config.yaml에 명시된 "혜미 승인 없이는 게시 절대 불가·게시봇 이중방어" 원칙을 화면에도 그대로 반영해, 승인 전에는 [지금 게시] 버튼이 항상 비활성화
- 승인 후에도 예약된 시각까지는 [승인 취소] 버튼을 남겨 발행 직전까지 되돌릴 여지를 준다(콘텐츠는 방역 문서보다 취소 여지가 있어야 하는 영역이라 판단, 다른 3개 공장의 "승인=완전 잠금"과 차이를 둠 — 필요시 혜미와 재확인)

### ⑤ 게시 이력/성과

- 리스트(표): 날짜, 채널, 상태(게시완료/게시실패), 링크
- 실패 시 `--fail` 배지 + 실패 사유(예: "티스토리 미개설로 대기") — channel_config.yaml 기준 티스토리는 아직 개설 대기 상태이므로 이 채널은 캘린더에서도 "개설 대기" 회색 처리

## 색상/상태 적용 요약 (콘텐츠 전용)

| 상태 | 화면 | 색 |
|---|---|---|
| 초안 | 캘린더/승인 | `--text-dim` |
| AI 생성중 | 작성화면 | `--accent` |
| 발행 예정인데 콘텐츠 없음 | 캘린더 | `--warn` 테두리 |
| 혜미 승인완료 | 승인/캘린더 | `--ok` |
| 반려 | 승인 | `--fail` |
| 게시 실패 | 이력 | `--fail` |
| 채널 미개설(예: 티스토리) | 캘린더 | `--text-dim` 회색 처리 |

## 조사 근거

콘텐츠 캘린더 UI는 날짜/시간/플랫폼/본문/자산/링크를 최소 필드로 하고, 상태를 draft→in review→approved→scheduled→live 단계로 명시하는 패턴이 여러 자료(Sprinklr, Hootsuite, Monday.com)에서 공통적으로 나타납니다. 또한 승인 전 플랫폼별 미리보기를 제공하는 것이 표준적인 관행이라는 점을 반영해 ③ 채널별 미리보기 화면을 승인(④) 앞에 배치했습니다.
