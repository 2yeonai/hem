# Fable 판단 대기 3건 — 마케팅팀 / 픽셀 AI 회사 / 가격표 (2026-07-19)

> 배경: [[_inbox/아이디어_대기줄|2026-07-19 대용량 설계안]] 검토 결과, 구조 결정이 필요한 항목만 추림.
> [[ai공장짓기/Fable_작업대기열|Fable 작업대기열]] F7~F9로 등록. 셋은 서로 독립적이라 순서 무관, 병렬로 Fable 세션에 넘겨도 됨.
> 사용법: 아래 질문 블록 하나를 통째로 복사해 Fable 5 세션에 붙여넣는다.

## F7 — 마케팅팀을 공통코어 필수 부서로 추가할지

**결정할 것**
1. 마케팅 코어를 몇 단계 파이프라인으로 볼지: 브랜드전략 입력 → 콘텐츠기획 → 마스터콘텐츠 → 채널별 변환 → 사실·브랜드 검수 → **사람 승인(게시)** → 게시 → 성과분석. 이 7단계가 기존 manifest v2 스키마(local/model/human, quality_gate, approval)와 그대로 맞는지, 별도 스키마가 필요한지.
2. 기존 3공장(꽃집 배송완료·방역 완료보고서·정부지원 사업계획서)의 실제 완료 결과를 마케팅 인풋으로 넘기는 **연결 지점을 어느 stage 뒤로 할지** — 예: 꽃집 `주문완료` stage의 io.writes에 `마케팅_소재후보` 필드를 추가할지, 아니면 완전히 별도 트리거(event: 업무완료)로 뗄지.
3. `channel-packs`(네이버블로그/스레드/인스타/구글비즈니스프로필 등)를 기존 `industry-pack`/`client-pack` 옆에 세 번째 팩 종류로 신설할지, 아니면 client-pack 하위 필드로 흡수할지.
4. 게시·발송은 [[1. Projects/클로드 ai 자동화/company_charter|company_charter]] §5 트리거 대상(비가역 행동)이 확실하므로, 마케팅팀 도입 자체가 헌장 개정 트리거인지 재확인.
5. **혼자 유지보수 가능 범위 안에서**, 처음엔 채널 전부가 아니라 어떤 채널 1~2개만 먼저 파이프라인에 넣을지(기존 콘텐츠 공장이 이미 블로그·스레드 테스트 글 3건 승인까지 가 있음 — 그 자산을 그대로 마케팅팀 v1으로 승격시키는 안 vs 새로 설계하는 안).

## F8 — "픽셀 AI 회사" 시각화 채택 여부

**결정할 것**
1. 지금 단계(정부지원/꽃집/방역 세 공장 모두 아직 실사용 골든셋 마무리 중)에서 이 화면을 만드는 게 우선순위상 맞는지, 아니면 "동결"에 다시 넣을지.
2. 만든다면: 실무형 관리자 화면(운영표에 이미 있는 개념)과 **별도 화면으로 병행**할지, 아니면 나중 확장으로만 미룰지.
3. 최소 프로토타입 범위 — 정적 층 구조 + 상태 아이콘까지만(애니메이션·엘리베이터·전시모드는 전부 후순위)로 합의할지.
4. 데이터 소스: 러너(`ai공장짓기/runner/`)의 실제 이벤트 스트림에 연결할지, 아니면 이번엔 완전 mock(가짜 상태)으로만 만들어 "느낌"만 검증할지.

## F9 — 상품 5단계 가격표(진단~업종용 AI운영체계) 채택 여부

**결정할 것**
1. 지금 판매 주체가 누구인지부터 명확히 — mo,on(혜미 본인 창업) 자체가 아직 예비창업패키지 탈락 후 재도전 여부 평가 중인 단계인데, "AI 자동화 컨설팅/구축 판매업"을 mo,on과 별도 사업으로 볼지, mo,on 안의 한 서비스 라인으로 볼지.
2. 제시된 가격대(진단 15~30만 / 반자동 89~149만+월 / 자동운영팀 249~490만+월 / 픽셀AI회사 490~990만+월 / 업종용OS 990~2000만+)가 지금 실제로 완성된 산출물(정부지원 매칭·꽃집 14봇·방역 18단계) 수준과 맞는지, 과대 책정인지.
3. 월 관리비 안에 API 실비용을 어떻게 반영할지 — 문서가 제안한 "고객 계정 분리 vs 혜미 계정에서 사용량만 분리" 중 어느 쪽을 원칙으로 할지 (기존 헌장에 비용 원칙이 아직 없으면 이번에 추가).
4. 첫 판매 대상을 이 3업종(정부지원 자주 신청하는 소상공인/꽃집형 주문제작업/방역·시설관리업) 중 어디로 좁힐지, 가족 사업(온천꽃식물원·방역앤클린) 자체를 베타 고객 0호로 볼지.
