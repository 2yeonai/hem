# UI 기획서 — 러너·공장 앱 공통 디자인 시스템 #플랫폼

> Fable 확정. 하위 모델은 이 문서만 읽고 "예쁜 UI"를 판단 없이 구현할 수 있어야 함 — 그래서 전부 형용사가 아닌 값으로 적음.

## 1. 디자인 토큰 (모든 앱 공통 — 복붙용)
```css
:root{
  --bg:#0f1220; --card:#1a1f35; --card-hover:#232a4a;
  --text:#e8ecff; --text-dim:#9aa3c7;
  --accent:#7fd4ff; --primary:#2f6fed; --ok:#3a9d5f; --warn:#d9a53a; --fail:#e05a5a;
  --radius:14px; --gap:16px; --font:'Pretendard',system-ui,sans-serif;
}
```
- 폰트: Pretendard(CDN: cdn.jsdelivr.net/gh/orioncactus/pretendard) 폴백 system-ui. 본문 15px/1.7, 제목 22px 700.
- 카드: 배경 --card, 모서리 --radius, 패딩 18~22px, 그림자 `0 4px 24px rgba(0,0,0,.35)`.
- 버튼: 높이 40px, 좌우 패딩 16px, radius 10px, hover 시 밝기 +8%. 주버튼 --primary, 성공동작 --ok.
- 상태색 고정: 검증통과=--ok, 경고=--warn, 실패=--fail, 실행중=--accent 점멸.
- 모바일: 640px 이하에서 카드 1열, 버튼 전체폭.

## 2. 러너 웹앱 v2 화면 명세 (다음 Sonnet 세션 작업분)
현재 v1: `ai공장짓기/runner/webapp.py` (기능 검증 완료). v2에서 추가:
1. 상단 헤더: "🏭 AI공장 러너" + 공장 수/최근 실행 시각.
2. 공장 카드에 상태 배지(마지막 실행 PASS/WARN/FAIL — logs 폴더에서 읽음), 폴더명은 --text-dim.
3. 실행 결과: FAIL 줄은 --fail, WARN은 --warn로 색칠(정규식 치환), 실행 중 스피너.
4. 완료 기준(체크 가능): ①위 토큰 그대로 사용 ②모바일 폭에서 안 깨짐 ③기존 검증·실행 기능 회귀 통과 ④스크린샷 저장해 증거 첨부.
지시 프롬프트: `(Sonnet) ai공장짓기/UI_기획서.md 2번대로 webapp.py를 v2로 개선. 완료 기준 4개 증거 포함 보고.`

## 3. 공장 앱 공통 레이아웃 (신규 공장 UI는 전부 이 틀)
[헤더: 공장 이름+한줄설명] → [입력 카드: 폼] → [실행 버튼 1개] → [진행 단계 표시: stage명이 순서대로 켜짐] → [결과 카드: 산출물+다운로드] → [푸터: 승인 필요 항목 경고 --warn]. 절대규칙: 게시/발송 버튼은 승인상태=승인완료일 때만 활성화.

## 4. Figma 활용 (연결 후 — 혜미가 설정>Connectors>Figma 연결하면)
- 용도①: 카드뉴스·썸네일 템플릿을 Figma로 만들고 클로드가 읽어 규격(색·폰트·여백)을 이 문서 토큰과 동기화.
- 용도②: 공장 앱 화면 시안 → 클로드가 시안 읽고 3번 레이아웃과 대조해 구현.
- 연결 전에도 작업 가능: 이 문서 토큰만으로 UI 구현 (Figma는 보강용이지 선행조건 아님).
