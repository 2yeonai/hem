# 꽃집 human_reviewer 승인화면 프로토타입 (v1)

이 폴더의 `승인화면_프로토타입_v1.html`은 꽃집(온천꽃식물원) 14단계 주문 파이프라인 중 유일한
사람 승인 단계인 `human_reviewer`가 실제로 어떤 화면일지 미리 보기 위한 **목업**입니다.
백엔드·AI 모델·데이터베이스 없이, 파일 하나만으로 브라우저에서 즉시 동작합니다.

## 여는 방법

`승인화면_프로토타입_v1.html` 파일을 **더블클릭**하면 기본 웹 브라우저(Chrome, Edge 등)로 바로 열립니다.
인터넷 연결이 없어도 열리고 작동합니다(외부 폰트·이미지·스크립트를 전혀 불러오지 않음 — 파일
하나에 모든 화면·글꼴 설정·동작 코드가 들어 있습니다).

열리면 "검수 대기" 목록이 먼저 보입니다. 카드를 누르면 그 주문의 상세 검수 화면으로 들어가고,
왼쪽 위 ← 버튼을 누르면 목록으로 돌아갑니다.

## 화면에 있는 내용

- **대기 큐**: 주문 3건(#001, #005, #006)이 검수 우선순위 색(빨강/노랑/파랑/초록) 배지와 함께
  나열됩니다. #005·#006은 실제로 같은 통화(`call_007_2026-07-07`) 한 건에서 분리된 두 개의
  주문이라, 카드에 "🔗 묶음 1/2·2/2" 표시가 붙고 상세 화면에서 서로를 바로 넘나들 수 있는
  버튼이 있습니다.
- **상세 화면**: 받는분/보내는분 기관·직함(AI 초안, 회색조 표시)과 이름·배송지·연락처(항상 빈칸
  + "⚠ 확인 필요" 배지 — golden_set 001에서 실제로 이름이 뒤바뀐 사례가 있어, 이름은 AI가 절대
  추정해서 채우지 않는다는 설계 원칙을 그대로 반영했습니다), 리본 문구·상품명·수량·가격, 검수
  체크리스트(체크박스), 원문 보기(펼치기)로 구성돼 있습니다.
- **승인 처리**: `manifest.yaml`의 `approval_action` 값 그대로 [검수저장]/[주문확정]/[출력준비]
  세 버튼이 있고, 체크리스트를 모두 확인해야 [주문확정]이 켜지며, [주문확정]을 눌러야만
  [출력준비]가 켜집니다. [주문확정]을 누르면 그 주문 카드는 잠기고(🔒 승인완료) 값을 다시
  고칠 수 없습니다. 추가로 [반려] 버튼도 넣었는데, 이건 원래 설계 문서(`12봇_kind분류표.yaml`)에는
  없는 개념이라 — 눌러보면 화면 안에 "return_to(반려 시 되돌아갈 단계)는 아직 확정되지 않은
  미정 항목"이라는 안내가 그대로 나옵니다. 지어내지 않고 미정이라는 사실 자체를 화면에 보여주는
  쪽을 택했습니다.

## 목업의 한계 (실제로 안 되는 것)

1. **실제 데이터와 연결되지 않습니다.** 화면에 보이는 주문 3건은 `golden_set.yaml`의 001/005/006
   실데이터를 그대로 옮겨 적은 것이고, 새 문자·전화가 와도 이 화면에 자동으로 올라오지 않습니다.
   실제 파이프라인(`order_classification_bot` ~ `review_manager_bot`)과 연결하는 작업은 아직
   안 돼 있습니다.
2. **저장이 안 됩니다.** [검수저장]/[주문확정]/[출력준비]/[반려]를 눌러도 그 결과는 이 브라우저
   탭 안 메모리에만 잠깐 남고, 새로고침하거나 파일을 다시 열면 전부 처음 상태로 돌아갑니다.
   실제 서비스에서 필요한 `storage_bot` 저장·버저닝은 이 화면에 연결돼 있지 않습니다.
3. **review_priority 색상 기준은 확정 사양이 아닙니다.** 빨강/노랑/파랑/초록 배지는
   `review_manager_bot.py`에 이미 있는 규칙기반 로직을 그대로 계산해 보여준 것이지만, 그
   로직 자체가 "실측 없이 제안된 값"이라고 소스 코드 주석에 명시돼 있습니다(`12봇_kind분류표.yaml`
   open_questions에도 동일하게 남아 있음). 색이 이상해 보이면 이 화면이 아니라
   `review_manager_bot.py`의 규칙을 고치는 게 맞습니다.
4. **[반려] 버튼과 배송지·연락처 입력칸은 이 프로토타입에서 새로 추가한 부분입니다.** 원래
   설계 문서에는 꽃집 파이프라인에 "반려" 흐름 자체가 없고(대표자가 카드에서 직접 값을 고쳐
   진행하는 3단계 버튼만 있음), 배송지 주소·연락처도 아직 정식 데이터 필드로 정의돼 있지
   않습니다. 화면에서는 자유 입력칸으로만 임시로 넣어뒀고, 실제 필드명·저장 위치는 다음
   세션에서 스키마 담당자와 확인이 필요합니다.
