# 꽃집 웹앱을 휴대폰에서 쓸 수 있게 만들기 (Render 배포 가이드)

이 폴더(`webapp/`)의 `app.py`는 이미 만들어져 있고 로컬 테스트도 끝났습니다.
남은 건 **혜미님이 직접 하셔야 하는 부분**(계정 가입, 클릭 몇 번)뿐입니다 —
이건 개인 이메일/계정 만드는 일이라 제가 대신 할 수 없습니다.

## 왜 Render인가
- 무료 요금제 있음 (카드 등록 불필요, 이메일만 있으면 가입)
- 이 볼트가 이미 GitHub에 올라가 있어서, GitHub 계정만 연결하면 자동 배포됨
- 항상 켜져 있어서 휴대폰에서 언제든 접속 가능 (단, 무료 요금제는 15분 정도
  아무도 안 들어오면 "잠들었다가" 다음 접속 시 10~30초 정도 깨어나는 데
  시간이 걸릴 수 있음 — 무료라서 생기는 유일한 불편, 유료 전환하면 해결)

## 1단계 — Render 가입 (혜미님이 직접)
1. https://render.com 접속
2. "Get Started" 클릭 → **"GitHub으로 가입"** 선택 (이미 있는 GitHub 계정 그대로 사용)
3. 이메일 인증만 하면 끝 — 카드 번호 입력 화면이 나오면 건너뛰어도 무료 요금제는 그대로 씀

## 2단계 — 새 웹 서비스 만들기 (혜미님이 직접, 5분)
1. Render 대시보드에서 **"New +"** → **"Web Service"** 클릭
2. GitHub 저장소 목록에서 이 볼트(hem 저장소) 선택 → "Connect"
3. 아래 칸들을 이렇게 채우기:
   - **Name**: `flower-shop-webapp` (아무 이름이나 가능)
   - **Root Directory**: `1. Projects/클로드 꽃집 ai/webapp`
   - **Runtime**: `Python 3`
   - **Build Command**: `pip install -r requirements.txt`
   - **Start Command**: `gunicorn -w 1 -b 0.0.0.0:$PORT app:app`
   - **Instance Type**: `Free` 선택
4. **"Environment Variables"** 섹션에서 "Add Environment Variable" 클릭:
   - Key: `ANTHROPIC_API_KEY`
   - Value: (발급받은 진짜 키 값 — 여기 이 문서에는 절대 적지 마세요, Render 화면에만 입력)
5. **"Create Web Service"** 클릭 → 몇 분 기다리면 배포 완료
6. 완성되면 화면 위에 `https://flower-shop-webapp.onrender.com` 같은 주소가 생김 —
   **이 주소를 휴대폰 브라우저 즐겨찾기(또는 홈 화면에 추가)해두면 앱처럼 쓸 수 있습니다.**

## 3단계 — 확인
1. 그 주소를 휴대폰으로 열어서 "검수 대기" 화면이 보이면 성공
2. "+ 새 문의 입력"을 눌러 아무 문자 내용이나 넣고 "AI 판단 시작" 눌러서
   실제로 AI가 판단해 상세 화면으로 넘어가는지 확인
3. 저장이 잘 안 되거나 화면이 이상하면, 다음 세션에서 "Render 배포 안 됨"이라고
   말씀해주시고 Render 화면의 "Logs" 탭 내용을 보여주시면 원인 확인 가능합니다.

## 알아두면 좋은 것
- 코드를 나중에 고치면(다음 세션에서), GitHub에 커밋·푸시만 하면 Render가 자동으로
  다시 배포합니다 — 따로 뭘 더 누를 필요 없음.
- 이 웹앱은 **1인 운영 기준**으로 만들어졌습니다(동시에 여러 명이 승인 버튼을 누르는
  상황은 고려 안 함) — 지금 규모엔 문제없습니다.
- 승인화면_프로토타입_v1.html(예전 목업)과 달리, 이 웹앱은 실제로 저장되고 실제
  AI가 판단합니다 — 목업이 아니라 진짜입니다.
