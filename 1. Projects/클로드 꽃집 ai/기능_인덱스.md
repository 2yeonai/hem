## 아직 없는 것 (착수 전이라 설계해도 됨)

- **kind:model 6개 전부 실 LLM 미연동** — 전부 규칙기반(정규식/사전/휴리스틱)으로만 동작. 각 `*_ENGINE` 변수 자리에 LLM 호출 함수만 갈아끼우면 되도록 구조는 이미 분리돼 있음.
- **human_reviewer(kind:human) 승인 UI 없음** — run_pipeline.py가 데모 목적으로 자동 통과시킬 뿐 실제 사람 승인이 아님.
- transcription_bot의 실 STT/OCR 연동 (현재 항등 목업).
- sms_ledger_bot의 실 문자 게이트웨이(알리고 등) 연동.
- order_classification_bot(golden_set 003 오판정) / order_split_bot(call_007 자동 2분리) 규칙기반 한계 — LLM 연동 시 해소 예상.
- review_manager_bot의 review_priority 색상 매핑 실측 검증(현재는 제안값).
- 리스크_메모_통화녹음_제3자동의.md의 통화녹음 제3자 동의 법적 이슈 — 결론 안 남(별도 세션 확인 필요, 실 STT/녹음 기능 출시 전 검토 필수).

1. 새 아키텍처 결정을 하기 전에 이 표 + `1. Projects/완전자동화_실행계획.md`의 T-테이블을 먼저 확인.
