# 완전 자동화 실행계획 v2 — "다음" 큐 (비개발자용)

> **사용법 (이것만 기억)**: 아무 새 세션(Sonnet)에나 아래 한 줄을 복붙한다.
>
> ```
> 1. Projects/완전자동화_실행계획.md 를 읽고 "다음" 프로토콜을 실행해.
> ```
>
> 모델이 첫 번째 미완료 작업을 찾아 실행하고, 끝나면 이 파일의 상태표를 직접 갱신한다.
> 혜미는 작업이 끝날 때마다 새 세션에 같은 한 줄을 다시 붙여넣기만 하면 된다. 날짜 아님, 순서다.

## "다음" 프로토콜 (실행 모델용 규칙 — 반드시 준수)

1. 상태표에서 위에서부터 첫 번째 `대기` 작업을 찾는다. `진행중`이 있으면 그 작업 블록의 `HANDOFF:` 줄부터 읽고 이어서 한다.
2. 시작할 때 상태를 `진행중`으로 바꾸고, 그 작업 블록의 지시만 수행한다. 다른 작업은 미리 건드리지 않는다.
3. **한 세션 = 한 작업.** 토큰/세션 한도가 걱정되면 작업을 스스로 절반으로 쪼개 앞부분만 끝내고, 작업 블록 끝에 `HANDOFF: (어디까지 했고, 다음 세션이 뭘 할지 한 줄)`을 남긴 뒤 정상 종료한다. 멈춤은 실패가 아니다 — 기록 없는 멈춤만 실패다.
4. 완료조건을 전부 충족하면 상태를 `완료`로 바꾸고 상태표 결과 칸에 한 줄 기록. 혜미에게는 "완료 + 혜미가 할 일(있으면)"만 보고한다.
5. 기존 설계와 충돌하거나 새 구조 판단이 필요해지면 **작업을 멈추고 "Fable 판단 필요"라고만 보고**한다. 임의 설계 금지.
6. **새 아이디어·새 설계는 이 큐에 넣지 않는다.** `_inbox/아이디어_대기줄.md`에 한 줄 적고 끝. 동결 해제 검토는 T5 완료 후 일괄로만.
7. md 파일 작성·수정 시 볼트 규칙(헤어독 + 센티널 `<!-- ok -->` + verify_write.py) 준수. **정정(2026-07-16)**: 한때 이 파일에 "`ai공장짓기/scripts/`가 비어있어 verify_write.py·validate_manifest.py가 없다"고 적었던 적이 있는데 그건 틀린 정보였다 — bash 마운트가 `1. Projects/ai공장짓기/` 폴더의 디렉토리 목록을 일시적으로 빈 것처럼 보여준 캐시 결함이었고, 실제로는 둘 다 처음부터 온전히 존재했다(파일 도구/Read로 확인). **bash `ls`/`cat`이 "없음"으로 보고해도 곧바로 "유실"로 단정하지 말고 Read 도구(Windows 경로)로 먼저 재확인할 것.** 스크립트 실제 경로: `1. Projects/ai공장짓기/scripts/validate_manifest.py`, `verify_write.py`. bash로 그 경로가 안 열리면 파일 도구로 내용을 읽어 다른 접근 가능한 폴더(예: 작업 대상 스킬 폴더)에 임시 복사해서 실행 후 삭제하는 우회로 사용.
