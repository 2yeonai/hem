# 완전 자동화 실행계획 (비개발자용, 순서대로)

> 사용법: 위에서부터 순서대로. 각 단계의 회색 박스를 **해당 모델을 켠 새 세션에 그대로 복붙**하면 된다.
> 모델 규칙은 `ai공장짓기/CLAUDE.md` 그대로: 기본 Sonnet, 단순분류 Haiku, 설계·충돌만 Fable(승인 후).

## 1단계 — 방역 골든셋 (이번 주 최우선, 유일한 병목)

새 세션(Sonnet)에 복붙:
```
클로드 방역 ai/golden_set_template.yaml 을 읽고 나를 인터뷰해서 실제 사례로 채워줘.
질문은 한 번에 하나씩. 기억 안 나는 필드는 "미확인"으로 두고 넘어가.
완료 후 yaml 문법 검증을 실행하고 통과 결과를 붙여넣어. 3건 이상 목표.
```

## 2단계 — 방역 검증 (골든셋 완료 직후)

```
(Sonnet) 클로드 방역 ai 폴더에서 python3 scripts/test_12cases_batch.py 를 실행하고
결과 전문을 붙여넣어. 그다음 골든셋 실사례를 근거로 미정 봇 3개
(현장분리/견적금액산정/현장사진)의 필요 여부를 판정하고,
판정 근거를 ai공장짓기/decision-log_skill-factory-architecture.md 에 추가해.
기존 설계와 충돌이 나면 작업을 멈추고 "Fable 판단 필요"라고만 보고해.
```

## 3단계 — 정부지원 스킬 실전 투입

```
(Sonnet) 클로드 정부지원사업 ai 스킬을 사용해서, 2. Areas/핵심맥락.md 의
mo,on 사업 내용으로 예비창업패키지 신청서 초안을 작성해줘.
8인심사위원 자기검수를 돌리고 점수표와 감점 항목, 수정본까지 만들어.
제출은 절대 하지 마 — 초안과 점수표만 파일로 저장해.
```

## 4단계 — 꽃집 정리 + 착수

```
(Sonnet) 감사_로드맵_2026-07-09.md 의 R4 항목을 그대로 실행해줘.
삭제 금지, 이동만. diff 확인 결과를 반드시 붙여넣어.
```

## 5·6단계 — Fable 사용 (승인 후에만)

- 5단계 설계 질문 목록과 6단계 프롬프트 위치: `ai공장짓기/감사_로드맵_2026-07-09.md` R5·R6 참고.
- Fable 세션 산출물은 반드시 `ai공장짓기/`에 저장 + decision-log 기록 (기존 규칙).

## 요금 폭탄 방지 규칙 (이 프로젝트 공통)

1. 모든 "완료" 보고는 증거(테스트 출력·diff·검증 로그) 붙여넣기 필수 — 말로만 "됐다" 금지.
2. 반복 실행엔 상한: 같은 작업 3회 실패 시 중단·보고 (루트 CLAUDE.md 규칙과 동일).
3. Fable은 설계·충돌 판단에만, 실행은 Sonnet, 분류는 Haiku.
4. 동시에 벌이는 목표는 최대 3개 (현재: 골든셋, 신청서, 자동기록).

문서 작성·수정, 코드 작성·실행·테스트, yaml 검증, 폴더 정리(이동만, 삭제 금지), 리서치·요약, 신청서 초안+8인심사위원 검수, PPT 제작, 세션 기록·주간 종합(자동), 새 자료 평가("볼트 기준으로 평가해줘" 한 줄이면 됨), 데스크탑 앱 조작(혜미님이 요청+권한 승인 시).
