# 1부. 대표님용 진행상황 요약

## 지금까지 무엇이 만들어졌나
방역 업무 문의 접수부터 → 고객DB 저장 → 방문 알림 → 현장 작업 완료 입력 → 완료보고서·소독증명서 초안 생성 → **대표자 승인/반려** → 승인된 문서만 발송·보관까지 이어지는 전체 업무 흐름의 설계도(manifest)와, 이 설계도대로 실제 돌아가는지 확인하는 모의 실행 프로그램(mock 실행기)이 만들어졌습니다.

## 이번 회차에 새로 반영된 것
1. **문서 승인 워크플로우** — Fable 5가 판단한 구조를 그대로 반영: 승인 상태 4단계(초안/승인대기/승인완료/반려), 반려 시 두 갈래 처리(문구 문제 vs 데이터 문제), 승인 안 된 문서는 절대 발송되지 않는 안전장치.
2. **문서 종류 구분** — 완료보고서와 소독증명서가 각각 독립적으로 승인/반려될 수 있음 (한쪽만 반려되는 경우도 처리됨).
3. **방문 리마인더의 실제 날짜 계산** — 방문주기(월1회/격월/분기/반기/연1회)에 따라 다음 방문 예정일을 계산하고, 도래/임박 고객을 골라내는 로직.
4. **사업자 정보 대조 확인 문구 추가** — 완료보고서·소독증명서 초안의 사업자 정보(상호명/대표자명/사업자등록번호) 아래에, 실제 발급 전 국세청 공식 사업자등록증명과 대조해야 한다는 경고 문구를 추가했습니다.

## 확인 결과 (가상 데이터 기준)
- 합성 테스트 데이터 12건 전부 끝까지 정상 처리됨을 확인했습니다.
- 문서 반려 → 재작업 → 재승인 흐름을 두 가지 경우(문구 문제, 데이터 문제) 모두 실제로 돌려서 확인했습니다.
- **승인 안 된 문서가 발송되려고 하면 시스템이 막는지**를 일부러 계속 반려시키는 상황으로 만들어 테스트했고, 실제로 차단되는 것을 확인했습니다.

> 주의: 위 확인은 전부 가상의 합성 데이터로 한 것이며, 실제 방역 문의·현장 데이터로는 아직 검증되지 않았습니다.

## 아직 사람이 결정해야 하는 것 (5가지)
아래는 AI가 스스로 판단해서 바꾸지 않고, 대표님(또는 필요시 Fable 5에게 물어본 뒤) 확인을 기다리고 있는 것들입니다.
1. 고객 정보와 현장 정보를 분리해서 관리할지
2. 법정 주기(2년 보존 등) 문서를 어떤 방식으로 저장할지
3. AI에게 넘기는 정보를 최소한으로 줄일지
4. 지금은 대표자 1명 권한 체계인데, 나중에 직원 등 추가 권한이 필요할지
5. 현장 인터넷이 끊겼을 때 어떻게 동작할지 (오프라인 정책)

## 아직 확정되지 않아 조심해야 하는 것
- 소독증명서의 법정 필수 기재항목은 국가법령정보센터 원문·관할 보건소 서식으로 최종 확인이 필요합니다. 지금 템플릿은 초안 검토용입니다.
- 사업자 정보 항목은 이제 시스템이 저장해뒀다가 자동으로 불러와 채웁니다(2026-07-07 갱신 — 이전엔 사람이 매번 입력해야 했음). 다만 그 값이 실제로 맞는지 검증하는 것은 여전히 사람 몫이므로, 실제 발급 전 반드시 국세청 공식 증명과 대조해야 합니다.

# 2부. 정부지원사업 제출 / 멘토링용 요약

## 사업 아이템 한 줄 요약
**방역업 AI 운영관리 MVP**: 정기 고객사의 방문주기·방역이력·문서 요청 이력을 모바일 AI 입력 흐름으로 구조화하여, 오더메모·고객DB·정기방문 D-day 리마인더·완료보고서/소독증명서 초안 생성·법정 필수항목 누락 점검·대표자 승인 후 문서 잠금·보관·만족도 수집까지 하나의 업무 흐름으로 연결하는 방역업 전용 모바일 AI 운영관리 MVP.

## 이번 단계에서 구현·검증된 범위
| 구분 | 상태 |
|---|---|
| 전체 업무 흐름 설계도(manifest, 18 stages) | 완료 — 스키마 검증 통과(오류 0건) |
| 문의 접수 → 고객DB → 방문알림 → 현장입력 → 문서초안 흐름 | 완료 — mock 실행기로 동작 확인 |
| 정기방문 D-day 계산 로직 | 완료 — 실제 날짜 연산으로 구현·검증 |
| 완료보고서·소독증명서 문서 종류 구분 | 완료 |
| 대표자 승인/반려 워크플로우 (4단계 상태, 반려 2갈래 처리) | 완료 — 승인/반려/재승인 시나리오 실제 실행 검증 |
| 승인 전 문서 발송 차단 안전장치 | 완료 — 강제로 반려시켜 실제 차단되는지 확인 |
| 문서 초안 템플릿 (완료보고서 2종, 소독증명서 1종) | 완료 — 사업자 정보 대조 확인 문구 포함 |
| 합성 테스트 데이터 12건 + 승인워크플로우 시나리오 3종 | 전체 PASS |

## 미확정 · 다음 단계로 남긴 항목 (임의로 확정하지 않음)
- 고객/현장/방문 분리 구조, 법정 주기 보존 방식, AI 전달 필드 최소화, 권한 체계(1인 vs 다역할), 오프라인 정책 — 5개 항목, 구조적 결정이 필요해 보류 중.
- 소독증명서 법정 필수 기재항목의 정확한 조문·별지서식은 국가법령정보센터 원문·관할 보건소 안내로 아직 최종 대조되지 않음.
- 실제 방역 현장 데이터(골든셋)로는 아직 검증되지 않음 — 지금까지는 전부 합성 데이터 기준 검증.

## 성과 지표 참고 (기존 리서치 문서 기준, 최종 문안은 원문 대조 후 확정)
| 지표 | 목표 |
|---|---|
| 완료보고서·소독증명서 초안 작성시간 | 5분 → 3분 이내 |
| 대표자 승인 없는 문서 발송 | 0건 (이번 세션에 안전장치로 구현·검증) |
| 필수항목 누락으로 인한 재작성 건수 | 0건 목표 |
| 만족도·불편사항 응답 | 30건 이상 확보 |

## 멘토링에서 점검받으면 좋은 부분
완료보고서·소독증명서 초안 템플릿, 승인/반려 기준(문구 문제 vs 데이터 문제 2갈래), 문서 잠금·보관 기준, 사업자 정보 대조 확인 절차, 5개 미확정 구조 항목의 우선순위.
