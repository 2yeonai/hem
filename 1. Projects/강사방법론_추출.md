# 강사 방법론 추출 프로젝트 — 관리기간이 끝나도 그 기준이 내 것이 되게 #정부지원

> 원리: 강의 자료를 복사하는 게 아니라 **판단 기준을 내 언어로 재구성**하는 것. 이미 설계된 [[ai공장짓기/설계노트/공장별/공장-자료정리__학습변환|학습변환 공장]](회차 누적+화자 스타일) 패턴을 그대로 사용.
> 경계: 강의 원본 자료의 재배포·판매는 안 함. 방법론을 배워 내 사업에 쓰는 것은 학습의 본질 — 정당.

## 1. 수집 (매 강의마다, 혜미 5분)
- 강의 중: 클로바노트(또는 녹음→텍스트) ON. 끝나면 텍스트를 `_inbox`에 저장 — 밤에 자동 분류됨.
- 코칭/피드백 받은 날: 강사가 내 계획서에 한 말을 그대로(토씨까지) `_inbox`에 메모. **강사의 지적 원문이 최고 원료.**

## 2. 추출 (회차마다 자동, Sonnet — 복붙 프롬프트)
```
(Sonnet) _inbox의 이번 강의 녹취를 읽고 강사방법론/ 폴더에 3층으로 정리해줘:
①판단기준 — "어떤 계획서가 붙는다/떨어진다"고 말한 모든 기준을 규칙 문장으로
②프로세스 — 작업 순서(공고분석→아이템프레임→작성→검증)로 말한 것
③화법 — 자주 쓰는 표현·강조 패턴 (스타일 카드에 누적)
그리고 누적 종합본(강사방법론/종합본.md)을 갱신해. 원 녹취는 수정 금지.
```

## 3. 자산화 (관리기간 안에 완성할 것)
- **강사 페르소나 카드** = 판단기준+화법의 압축본 → 정부지원 공장의 **9번째 심사위원**으로 주입: 앞으로 모든 신청서가 "8인 심사위원 + 강사 기준"으로 자기검수됨. 이게 "뇌를 훔친다"의 실체 — 강의가 끝나도 그 기준이 시스템 안에서 영원히 작동.
- **질문 큐**: 남은 코칭 기회에 물어볼 것을 미리 준비 — 탈락분석의 급소 5개가 1순위 질문 목록:
  1. "엑셀·카톡으로 하던 업체가 왜 돈을 낼까요 — 지불의사를 어떻게 증명하죠?"
  2. "비개발자인데 SaaS 개발 역량을 심사에서 어떻게 보여주죠?"
  3. "수요조사는 몇 건이면 심사에서 '했다'로 인정받나요?"
  4. "MVP 단계에서 데이터 차별성을 정량으로 보여주는 좋은 예시가 있나요?"
  5. "예창패 다음으로 mo,on에 맞는 공고는 뭐라고 보세요?"

## 4. 종료 후 루프 (강의 끝난 뒤)
매 신청서: v작성(Sonnet) → 9인 심사(8인+강사 카드) → 감점 수정 → 혜미 승인 → 제출(혜미). 탈락 시 심사평을 골든셋에 추가 — **탈락도 시스템을 강하게 만드는 원료.**
