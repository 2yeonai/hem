# F3(손 위치·압력 방향 이미지 오버레이) — 소책자 이미지 추출 가능성 조사 보고

조사일: 2026-07-19
대상 파일: 프로젝트 루트 `테크닉 소책자.pdf` (40페이지, 46MB, iOS Print로 생성된 스캔본)

## 조사 방법
1. `pdfinfo`로 PDF 메타 구조 확인.
2. `pdfimages -list`로 각 페이지의 embedded image 개수·해상도·인코딩 확인.
3. `pdftoppm -r 150 -jpeg`로 실제 페이지 5~7장(앞/중간/뒤 섞어서: PDF 페이지 1, 2, 10, 20, 30, 39, 40)을 300~150dpi 이미지로 추출해 직접 열어봄(텍스트 추정이 아니라 실물 확인).
4. 카드 폴더(`카드/테크닉_*.md`, 60장)의 근육명과 소책자에서 육안으로 확인한 기법명을 대조.
5. 조사에 사용한 임시 크롭 이미지는 확인 후 전부 삭제함(산출물 아님).

## 실측 결과

### 1. PDF 구조
- `pdfinfo`: 40페이지, A4(841.89×595.276pt, 가로형), PDF 1.3, `Producer: iOS ... Quartz PDFContext` — 아이폰 사진을 Photos/Files 앱에서 "Print" 기능으로 PDF화한 전형적인 스캔본.
- `pdfimages -list`: **40페이지 = embedded image 정확히 40장(페이지당 1장)**, 각각 JPEG, 폭 1573~2167px × 높이 2072~2876px (세로형 사진), 실효 해상도 약 190~262ppi. 페이지가 가로형인데 이미지는 세로형 → 실제로는 휴대폰 카메라로 찍은 세로 사진이 가로 페이지 위에 얹힌 구조.
- 텍스트 레이어 없음(스캔 이미지이므로 `pdftotext`로 뽑을 문자 없음) — 카드에 적힌 "OCR 불가" 서술과 일치.

### 2. 실제 페이지 내용 (직접 열람 확인)
**결정적 발견: PDF의 각 "페이지"(=1장의 embedded 사진)는 책을 편 상태로 찍은 "두 페이지 스프레드" 사진이다.** 즉 PDF 1페이지 안에 실제 소책자의 물리적 페이지 2쪽(좌/우)이 나란히 들어있고, 각 쪽 하단에 인쇄된 페이지 번호(3, 4, 5, 6, 21, 22, 43, 44, 47, 48, 49, 50, 69, 70 등, 직접 확인)가 선명하게 보인다.

각 물리 페이지(기법 1개당 1쪽)의 레이아웃은 매우 정형화되어 있다:
- 상단: 기법명(영문, 깔끔한 sans-serif 디지털 폰트 — 손글씨 아님, 예: "GH posterior capsule ART", "Infraspinatus & Teres minor MET 1 (PIR)", "Hamstring ART", "Scalene MET")과 `position : ...` 자세 라벨.
- 중단: **둥근 모서리 마커(4개 원)로 테두리가 명확히 표시된 사진 박스** 1~2장(ART/MFR류는 Start→End 사진 2장, MET류는 사진 1장) — 손 위치·압력 방향이 담긴 바로 그 이미지. 박스 경계가 시각적으로 뚜렷해 크롭 기준선으로 쓰기 좋음.
- 하단: 필드 라벨(Contact Point/Tension or Compression, 또는 Barrier/Resistance, 또는 Tool/Technique/Land&Sink/Exit) + 빈 Note 줄.
- 일부 페이지엔 손글씨 메모도 존재(예: "안함"/스킵 표시, "편함 가까울수록" 등) — 부차 정보.

**목차(TOC) 페이지 존재 확인**: PDF 39페이지(=물리 페이지 47-48) 우측이 "Trunk Part" 섹션 목차였고, 여기 나열된 기법명·페이지번호(Thoracolumbar fascia MFR1→49, Erector spinae MET→51, Quadratus lumborum ART1→52 등)가 바로 다음 PDF 40페이지(=물리 49-50)의 실제 내용과 정확히 일치했다. 즉 **책이 신체 부위별 섹션(Shoulder/Cervical/Trunk/Hip/Knee/Ankle 등으로 추정)으로 나뉘고, 각 섹션 시작에 목차가 있어 "기법명→페이지번호" 매핑을 OCR 없이도 목차만 읽으면 얻을 수 있다.**

**PDF 페이지 순서는 물리 페이지 순서와 완전히 일치하지 않음**: 앞부분(PDF 1,2,10페이지 = 물리 3-4, 5-6, 21-22)과 뒷부분(PDF 39,40페이지 = 물리 47-48, 49-50)은 순서가 맞지만, 중간(PDF 20페이지=물리43-44, PDF 30페이지=물리69-70)은 건너뛰거나 뒤섞여 있다. 스캔이 여러 세션에 나뉘어 촬영·병합되며 순서가 흐트러진 것으로 보인다. → **PDF 페이지 번호만으로 내용을 예측하는 자동 인덱싱은 불가능, 각 페이지를 열어 제목/페이지번호를 확인해야 함.**

### 3. 카드 근육명과의 대응 확인
직접 열람한 페이지만으로도 다음이 60장 테크닉 카드와 명확히 일치했다: GH전방/후방관절낭(GH capsule ART), 극하근·소원근(Infraspinatus & Teres minor), 소흉근(Pectoralis minor), 사각근(Scalene), 흉쇄유돌근(SCM), 요방형근(Quadratus lumborum), 척추기립근(Erector spinae), 횡격막(Diaphragm & AB complex), 대퇴이두근/반건양근/반막양근(Hamstring — 단, 소책자는 "Hamstring"으로 묶어 한 기법만 제공, 카드는 개별 근육 3장으로 분리되어 있어 1:N 매핑 필요). Coracobrachialis처럼 소책자엔 있지만 손글씨로 "안함"(스킵) 표시된 기법도 있었다.
카드 파일 자체에 이미 `책_페이지: 소책자 페이지 확인 안 됨 (스캔 PDF OCR 불가)` 필드가 있어, 애초에 대응 관계를 기록하려는 설계였음이 확인됨.

## 실현 가능성 결론
**(b) 수작업(또는 반자동) 크롭으로 실현 가능. 완전 자동화(a)는 어렵지만, 완전 불가능(c)은 아님.**

근거:
- 40페이지 = 물리 최대 80쪽 분량이지만 실질 기법 수는 이보다 적고(목차 확인 결과 섹션당 10여 개 내외), 사진 박스가 둥근 모서리 마커로 명확히 구분되어 크롭 좌표를 특정하기 쉽다.
- 각 페이지 제목이 손글씨가 아닌 깔끔한 디지털 폰트라 사람이 눈으로 읽는 것도 빠르고, 필요하면 제목 영역만 잘라 OCR을 걸어도 인식률이 높을 것(다만 전체 페이지 자동 OCR 파이프라인을 새로 구축할 만큼 페이지 수가 많지 않음 — 배보다 배꼽이 커짐).
- 다만 PDF 페이지 순서가 물리 페이지 순서와 어긋나 있어 완전자동 인덱싱(a)은 신뢰할 수 없고, 목차 페이지를 먼저 찾아 "기법명→물리페이지" 지도를 만든 뒤 사람이 확인하며 진행해야 한다.
- 소책자의 기법 단위(예: "Hamstring" 통합)와 카드의 근육 단위(대퇴이두근/반건양근/반막양근 개별)가 항상 1:1은 아니므로, 매칭 시 근육별로 어떤 사진을 재사용할지 판단이 필요하다(자동 로직으로 못 미룸).

## 다음 단계 제안 (구체적)
1. 40개 PDF 페이지를 300dpi로 개별 추출(`pdftoppm -r 300 -jpeg`, 총 40장, 각 약 2~4MB 예상).
2. 섹션 목차 페이지(현재 확인된 것: PDF 39페이지=Trunk Part 목차)를 전수 스캔해 "기법명 → 물리 페이지번호" 목록을 먼저 만든다 — 목차 페이지는 페이지 하단 번호가 크게 튀는 지점(섹션 경계) 근처에 있을 가능성이 높음.
3. 목차 기반으로 필요한 물리 페이지만 골라 좌/우 절반으로 분할(각 스프레드 이미지를 폭 절반 기준으로 crop) 후, 사진 박스(둥근 모서리 마커 안쪽)만 2차 크롭.
4. 카드 60장의 근육명과 대조해 1:1 또는 1:N(예: Hamstring→대퇴이두근/반건양근/반막양근 공용)으로 매핑표 작성.
5. 매핑 확정된 것부터 `app/`에서 쓸 이미지 파일로 저장(예: `app/images/기법명.jpg`)하고 카드 md의 F3 관련 필드(현재 텍스트만 있는 손 위치·압력 방향 서술)에 이미지 경로를 연결.
6. 소책자에 없는 근육(대퇴직근 등 카드 자체가 없는 경우 제외, 그 외 소책자 미수록 근육)은 F3 이미지 없이 텍스트만 유지하거나 별도 촬영 필요함을 명시.
7. 페이지 수(40장, 기법 수 추정 50~80개 내외)를 고려하면 전량 수작업 크롭도 실무적으로 하루 이내 소화 가능한 규모로 판단됨.
