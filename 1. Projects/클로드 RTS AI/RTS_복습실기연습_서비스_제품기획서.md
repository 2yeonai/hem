# RTS 복습·실기 연습 서비스 — 제품 기획서 v1

## 0. 결론

**"검사 → 제한 움직임 → 의심 근육 → 촉진 → ART → MET → 재검사 → 운동 → 복습"이라는 임상 추론 체인 자체를 제품의 뼈대(데이터 모델이자 화면 흐름)로 삼는 개인용 복습·실기 훈련 서비스**를 만든다. 콘텐츠는 이미 보유한 RTS Level 1·2 자료(강의 전사문, 노트, 검사_전체정리 16개 검사, 테크닉_전체정리 13개 테크닉, 소책자 PDF, 시연 영상)를 구조화해서 채운다.

MVP는 1인용 3종 세트다: ① 체인 탐색기(검사에서 시작해 재검사·운동까지 클릭으로 따라가는 구조), ② 검사·근육·테크닉 카드(촉진/손 위치/압력·방향 포함), ③ 퀴즈 + 오답·약점 관리. 랜덤 실기시험, 해부 시각화 고도화, 다중 사용자는 확장 단계다. 성공 기준은 하나 — **RTS를 끝낸 사람이 강사 없이 혼자서, 실제 시술 순서 그대로 반복 연습할 수 있는가.**

---

## 1. 이 서비스가 해결하는 핵심 문제

1. **수료 후 지식 증발**: RTS는 검사→치료→재검사의 절차 지식인데, 수료 후엔 절차의 "연결"이 먼저 무너진다. 개별 근육·검사는 기억나도 "업 스크래치 양성 → 어떤 근육 순서로 볼지"가 흐려진다.
2. **자료는 있는데 검색 구조가 없다**: 전사문·노트·PDF·영상이 날짜별 파일로 흩어져 있어, "견갑하근 MET"를 다시 보려면 어느 파일 몇 페이지인지 뒤져야 한다.
3. **실기는 혼자 연습할 방법이 없다**: 손 위치, 압력 방향, 환자 자세는 텍스트만으로 재현이 어렵고, 피드백해 줄 강사도 없다.
4. **약점을 모른 채 전체를 반복한다**: 이미 아는 것을 다시 보고, 헷갈리는 것(예: 능형근 vs 견갑거근 촉진 구분)은 계속 놓친다.

## 2. 핵심 사용자 (Persona)

| ID | 페르소나 | 상황 | 핵심 요구 |
|---|---|---|---|
| P0 | **본인** (RTS Level 1·2 수료, 실무 병행) | MVP의 유일한 사용자이자 콘텐츠 소유자 | 시술 전 5분 리마인드, 주기적 전체 복습, 실기 감각 유지 |
| P1 | RTS 교육생 (수강 중~수료 직후) | 시험·실기 평가 대비 | 검사표 암기, 체인 암기, 실기시험 시뮬레이션 |
| P2 | 마사지사 | RTS 미수료, 기법 관심 | 근육별 촉진·테크닉 사전, 안전 범위(금기) 안내 |
| P3 | 운동지도자·재활운동 전문가 | 회원 평가에 검사 활용 | 검사→운동 연결(치료 단계는 열람만), 운동 처방 근거 |

P0의 사용성이 곧 MVP 기준이다. P1~P3은 데이터 구조에 "직군별 노출 범위" 필드만 미리 심어두고 화면은 만들지 않는다.

## 3. 사용자 여정 (User Journey)

**여정 A — 시술 직전 리마인드 (P0, 5분)**
내일 어깨 고객 예약 → 앱 열기 → "어깨 복합관절" 파트 → 업 스크래치 검사 카드 30초 확인 → 양성 시나리오 탭 → 의심 근육 4개(소흉근·능형근·견갑거근·광배근) 순서 확인 → 소흉근 ART 손 위치 이미지 + 영상 20초 구간 재생 → 끝.

**여정 B — 주간 정기 복습 (P0, 20분)**
홈의 "오늘의 복습 큐"(간격 반복이 고른 항목 10개) → 퀴즈 형식으로 진행 → 오답 2개는 해당 카드로 즉시 이동해 재학습 → 오답은 3일 뒤 큐에 자동 재등장.

**여정 C — 실기 셀프 훈련 (P0/P1, 30분)**
"실기시험 모드" 시작 → 랜덤 케이스 제시("40대, 팔을 뒤로 못 돌림, 다운 스크래치 -5cm") → 사용자가 체인을 단계별로 직접 선택(검사 선택 → 판정 → 의심 근육 지목 → 테크닉 선택 → 재검사) → 각 단계 정답·해설 → 결과가 약점 프로필에 기록.

**여정 D — 원자료 회귀 (P0)**
어떤 카드에서든 "원자료 보기" → 해당 내용의 출처(전사문 날짜·구간, 소책자 페이지, 영상 타임스탬프)로 점프. 카드가 요약이고 원자료가 근거라는 위계를 유지한다.

```

├─ 1. 학습 (라이브러리)

│   ├─ 근육 라이브러리 (부위별 / 체인 역할별)
│   ├─ 테크닉 라이브러리 (ART / MET 필터)
│   └─ 운동 라이브러리
├─ 2. 실습 (체인)
│   ├─ 체인 탐색기 (검사에서 시작하는 가이드 흐름)
│   └─ 실기시험 모드 (랜덤 케이스) [확장]
├─ 3. 복습
│   ├─ 퀴즈 (파트별 / 유형별)
│   ├─ 오답 노트
│   └─ 약점 프로필 + 복습 캘린더(간격 반복)
├─ 4. 자료실 (전사문 · 노트 · PDF · 영상 — 엔티티 링크 포함 원본)
└─ 5. 설정 (직군 모드 · 데이터 관리) [확장]
```

## 5. 화면 구조 (Screen Flow)

핵심 화면 8개. 화살표는 주 이동 경로.

```
[S1 홈]
  → [S2 검사 라이브러리] → [S3 검사 상세]
       S3 탭: 방법 | 정상 기준 | 양성 판단 | 보상 관찰
       S3 하단 CTA: "양성이면? 의심 근육 보기"
  → [S4 의심 근육 목록] (우선순위 순, 근육별 한 줄 단서)
  → [S5 근육 상세]
       S5 탭: 개요·해부 | 촉진 | ART | MET | 운동
       각 테크닉 탭: 자세 → 손 위치 → 압력·방향 → 환자 동작 → 단계 진행
       S5 하단 CTA: "재검사하기" → S3(동일 검사)로 복귀 + 전/후 기록
  → [S6 운동 상세] (재검사 개선 확인 후 연결)
[S1] → [S7 퀴즈/시험] → 결과 → 오답은 해당 S3/S5로 딥링크
[S1] → [S8 복습 대시보드] (약점 Top5, 캘린더, 파트별 숙련도)
```

설계 원칙: **어느 화면에서든 체인의 앞·뒤 단계로 한 번에 이동 가능**해야 한다(상단에 체인 브레드크럼 고정: 검사 > 의심근육 > 촉진 > ART > MET > 재검사 > 운동).

## 6. 핵심 기능 정의 및 명세

**F1. 검사 복습** — 검사 카드 16종(검사_전체정리 기반). 필드: 검사명, 관절, 대상, 자세, 방법(단계), 보상 관찰 포인트, 정상 기준, 양성 판단, 의심 원인, 주의, 연관 검사. 입력: 검사 선택. 출력: 카드 + 미디어. 상태: 열람/암기완료 표시.

**F2. 근육 해부학 시각화** — 근육별 기시·정지·작용 + 부위 이미지. MVP는 보유 이미지·소책자 발췌로 시작, 확장 단계에서 인터랙티브 2D(부위 탭 → 근육 하이라이트) → 3D 순으로 고도화. 코드 구현이 아니라 "시각 자산의 단계적 상향" 로드맵으로 정의. 벤치마크는 아나토미마스터([[1. Projects/클로드 RTS AI/조사-아나토미마스터_앱기능분석|기능 분석 노트]]): ① 근육의 질감·결 방향까지 보여주는 3D 이미지(결 방향은 F4 압력·근섬유 방향 표시와 직결), ② 모션으로 "이 움직임에 쓰이는 근육" 하이라이트, ③ 전신/부위 맥락 뷰 안에서 개별 근육을 하이라이트(개별 근육만 떼어 보여주면 위치 파악이 안 된다는 리뷰 불만 반영 — 카드마다 위치 미니맵 포함).

**F3. 손 위치 안내** — 테크닉별 시술자 손 모양·접촉 위치·랜드마크. 표현: 사진/일러스트 + 텍스트("쇄골 아래 2횡지, 손가락 끝으로 사선 진입" 식). 촉진(Palpation) 가이드와 공유되는 컴포넌트.

**F4. 압력·움직임 방향 표시** — 이미지 위 화살표 오버레이(압력 방향, 근섬유 방향, 환자 움직임 방향을 색으로 구분). MVP는 정적 이미지에 주석, 확장에서 애니메이션.

**F5. ART/MET 단계별 안내** — 테크닉 카드 13종(테크닉_전체정리 기반). 필드: 적응증("이 사람에게 해!"), 핵심 한 줄, 짧아지는/늘어나는 자세, 자세 변형(Prone/Standing 등), 단계(시술자 행동·환자 동작·호흡·유지 시간), 임상 포인트, 금기. "실습 모드"에서는 한 단계씩 큰 글씨로 표시(시술 중 힐끗 보는 용도).

**F6. 재검사 연결** — 테크닉 완료 후 동일 검사로 원클릭 복귀. 전/후 결과(각도·거리·통증)를 간단 기록 → 개선 여부가 세션 로그에 남는다. "치료했으면 반드시 재검사"라는 RTS 원칙을 UI 강제로 구현.

**F7. 운동 연결** — 재검사 개선 확인 후 해당 근육/움직임의 운동 카드 제시(활성화→강화→패턴 순 진행 단계 포함). 근육이 아니라 "회복된 움직임"에 매핑되는 것이 원칙.

**F8. 퀴즈** — 유형: 4지선다(양성 판단 기준), 매칭(검사↔의심근육), 순서 배열(체인·테크닉 단계), 이미지 문항(촉진 위치 지목), 빈칸 채우기(기시·정지·작용, 아나토미마스터 방식). 카드 데이터에서 자동 생성 가능한 유형부터 시작. 카드 열람 시 "명칭 가리기" 모드(용어를 가리고 스스로 말해본 뒤 확인 — 아나토미마스터 암기카드 방식, Muscle의 3표기 필드 활용)를 F1·F5 카드에 공통 제공.

**F9. 랜덤 실기시험** — 케이스 시나리오(주호소+검사 소견) 랜덤 제시 → 사용자가 체인 전 단계를 선택으로 완주 → 단계별 채점·해설. 출제 풀은 CaseScenario 데이터로 관리.

**F10. 오답·약점 관리** — 모든 퀴즈·시험 결과가 엔티티(검사/근육/테크닉) 단위 숙련도 점수로 누적. 간격 반복(SRS: 맞히면 복습 간격이 늘고 틀리면 짧아지는 방식)으로 "오늘의 복습 큐" 자동 생성. 약점 Top 5는 홈에 상시 노출. 아나토미마스터에서 가져올 보조 기능: 헷갈리는 카드 북마크(나만의 암기노트), 학습 리포트(오늘/누적 복습량·시간), 연속 학습 스트릭·배지(꾸준함 유도 장치).

**F11. 통합 검색** — 키워드 하나로 검사·근육·테크닉·운동을 한 번에 검색해 해당 카드로 점프(아나토미마스터의 해부학 검색 방식). 여정 A("시술 전 5분 리마인드")의 진입 속도를 좌우하므로 MVP에 포함.

## 7. 기능 간 연결 (체인이 데이터로 구동되는 방식)

```
Assessment ──양성──▶ LimitedMovement ──▶ SuspectMuscle(가중치 순)
                                             │
                              Muscle ◀───────┘
                                │
              ┌── PalpationGuide (F3 손위치 공유)
              ├── Technique[ART] ─┐
              ├── Technique[MET] ─┼─▶ 완료 ─▶ 동일 Assessment 재검사 (F6)
              └── Exercise ◀──────┘        └─ 개선 시 ─▶ Exercise (F7)
                                │
    QuizItem·CaseScenario ◀── 모든 엔티티에서 자동 파생 (F8·F9)
                                │
    AttemptLog ─▶ WeaknessProfile ─▶ 복습 큐 (F10) ─▶ 다시 카드로
```

즉 화면·퀴즈·시험·복습 큐는 전부 같은 엔티티 그래프를 다른 각도에서 보는 것이다. 콘텐츠를 한 번 구조화하면 기능 10개가 전부 그 위에서 동작한다 — 이것이 이 제품의 구조적 핵심이다.

## 8. 데이터 구조 (개념 스키마)

구현 코드가 아닌 개념 정의. 괄호는 관계.

- **Assessment** — id, 이름(한/영), 레벨(1|2), 파트(어깨/경추/기타), 관절, 대상 조직, 자세, 방법 단계[], 보상 관찰[], 정상 기준, 양성 기준, 주의, 연관검사[](→Assessment), 의심근육[]{→Muscle, 우선순위, 단서}, 미디어[](→ContentAsset), 직군노출범위
- **Movement** — id, 이름, 면(plane), 정상 범위, 관련 검사[](→Assessment)
- **Muscle** — id, 이름(해부학/일반/영문 3표기), 부위, 기시·정지·작용, 짧아지는 자세/늘어나는 자세, 관련 검사[](→Assessment), 촉진(→PalpationGuide), 테크닉[](→Technique), 운동[](→Exercise), 해부 자산[](→ContentAsset)
- **PalpationGuide** — muscle_id, 환자 자세, 랜드마크, 손 위치·손 모양, 깊이·압력, 진입 방향, 흔한 실수[], 미디어[]
- **Technique** — id, muscle_id, 유형(ART|MET), 변형(자세: Prone/Standing/Side 등), 적응증 문구, 핵심 한 줄, 단계[]{순서, 시술자 행동, 환자 동작, 압력·방향, 호흡, 시간}, 임상 포인트, 금기, 미디어[]
- **Exercise** — id, 대상(→Muscle | →Movement), 목적(활성화|강화|패턴), 단계[], 세트·반복 가이드, 진행 규칙(다음 단계 조건), 미디어[]
- **ContentAsset** — id, 유형(전사문|노트|검사표|교재PDF|이미지|영상), 파일 경로, 위치(페이지|타임스탬프|문단), 연결 엔티티[], 검증 상태(원문확인|미확인)
- **QuizItem** — id, 유형(선다|매칭|순서|이미지지목), 문제, 정답·오답, 연결 엔티티, 난이도, 생성 방식(자동|수동)
- **CaseScenario** — id, 주호소, 인구학 정보, 검사 소견{}, 정답 체인[](단계별 기대 선택), 해설
- **AttemptLog** — 사용자, 대상(QuizItem|CaseScenario|체인 세션), 결과, 시각
- **WeaknessProfile** — 사용자, 엔티티, 숙련도(0~1), 최근 복습일, 다음 복습 예정일
- **User** — 확장 단계용. MVP는 단일 사용자 고정, 필드만 예약(직군, 수료 레벨, 콘텐츠 접근 범위)

## 9. 기능 우선순위 (MVP → 확장)

| 순위 | 범위 | 포함 기능 | 판단 기준 |
|---|---|---|---|
| **MVP** | 혼자 쓰는 복습 도구 | F1 검사 카드, F5 ART/MET 카드, F3·F4(정적 이미지 수준), F6 재검사 연결, F7 운동 연결(기본), F8 퀴즈(자동생성 2유형), F10 오답노트(단순 목록), F11 통합 검색 + 체인 탐색기 + 자료실(원자료 링크) | 여정 A·B가 앱 안에서 완결되는가 |
| **확장 1** | 훈련 도구화 | F9 랜덤 실기시험, F10 고도화(SRS 큐·약점 Top5), F8 전 유형, 영상 타임스탬프 딥링크 | 여정 C 완결 |
| **확장 2** | 시각화 고도화 | F2 인터랙티브 2D 해부, F4 방향 애니메이션, 촉진 이미지 지목 문항 | 실기 재현도 향상 |
| **확장 3** | 플랫폼화 | 멀티유저, 직군 모드(P1~P3), 강사용 콘텐츠 팩, Level 1 콘텐츠 확충 | 외부 사용자 유입 |

MVP에서 의도적으로 뺀 것: 3D 해부, 계정 시스템, 커뮤니티, AI 자동 피드백. 이유: 여정 A·B의 완결에 기여하지 않음.

## 10. 향후 로드맵

- **Phase 0 — 콘텐츠 구조화**: 검사_전체정리·테크닉_전체정리를 위 스키마로 변환(이미 필드 구조가 거의 일치), 전사문·소책자·영상에 엔티티 링크 부여. 제품의 성패가 여기서 갈린다.
- **Phase 1 — MVP**: 위 표의 MVP 범위. 본인이 4주 이상 실사용하며 여정 A·B 검증.
- **Phase 2 — 훈련 도구**: 실기시험 모드 + SRS. 실사용 데이터로 케이스 풀·오답 패턴 보강.
- **Phase 3 — 베타 확장**: RTS 동기·교육생 소수에게 개방(P1 검증). 강사 감수 절차 도입(콘텐츠 검증 상태 필드 활용).
- **Phase 4 — 플랫폼**: 직군별 모드(P2 마사지사=테크닉 사전 중심, P3 지도자=검사·운동 중심, 치료 단계는 열람 제한), 강사가 자기 강의 자료로 콘텐츠 팩을 만드는 구조로 일반화. Level 1 및 타 부위(하지·체간) 콘텐츠 추가.

## 11. 다른 서비스와의 차별점

1. **체인이 1급 구조**: Complete Anatomy류 해부 앱은 "지식 사전", Anki·아나토미마스터류는 "암기 카드"다. 이 서비스는 검사→치료→재검사라는 **의사결정 절차 자체**가 데이터 모델이고, 모든 기능이 그 절차를 훈련시킨다.
2. **본인 강의 자료 원문 기반**: 강사의 표현("이 사람에게 해!", 핵심 한 줄)을 그대로 보존하고 원자료로 역추적 가능 — 일반 교과서 요약 앱과 학습 일관성이 다르다.
3. **재검사 루프의 UI 강제**: 치료 후 재검사를 화면 흐름으로 강제하는 서비스는 사실상 없다. RTS 철학의 제품화 지점.
4. **실기 중심**: 손 위치·압력 방향·환자 자세 등 "손으로 하는 지식"을 시각 자산 단위로 관리 — 텍스트 요약 도구와의 근본 차이.
5. **확장 시 진입장벽**: 콘텐츠 팩 구조(강사 자료 → 스키마 변환) 자체가 자산이 되므로, RTS 외 다른 교육과정으로도 같은 엔진을 재사용할 수 있다.
