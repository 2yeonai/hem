# 클로드 RTS AI — 임상 추론 훈련 카드 + PWA 앱

## 목표 (Outcome)
어깨·경추 임상 추론(검사 → 의심근육 → 촉진/ART/MET → 재검사) 수업 전사문을 근거로, 강의를 듣고 만든 티가 나지 않는 전문 레퍼런스 톤의 학습 카드(md)를 만들고, 이를 파싱해 오프라인 PWA 앱(`app/`)에서 카드 열람·체인 탐색·퀴즈·오답노트로 복습할 수 있게 한다.

## 완료 기준 (Definition of Done)
- [x] 카드 29장(검사 16 + 테크닉 13, 어깨·경추 유닛)이 `검증상태: 전사원문확인`(날짜별 원본 전사문 근거)로 작성되어 있다 — 정리본(`검사_전체정리.md`/`테크닉_전체정리.md`) 근거는 남기지 않는다.
- [x] `app/data.json`이 (당시) 카드 29장 최신 내용을 반영해 재생성되어 있고, `app/index.html`을 로컬 서버로 열었을 때 스키마 깨짐·콘솔 에러 없이 렌더링된다.
- [ ] **⚠️ 정정(2026-07-17 재확인): 29장은 전체가 아니라 "어깨·경추 유닛"만의 완료였다.** 혜미가 "카드 절대 29개 아니야, 소책자 페이지만해도 훨씬 넘어"라고 지적 — 실제로 미확인 전사문(3.18~3.27, 4-15/4-17.txt, 약 5,000줄)을 전량 읽어 인벤토리 작성 완료. **목/몸통(코어)/엉덩이/무릎/발목** 유닛 전체가 아직 카드화되지 않았음이 확인됨. 상세 목록은 `HANDOFF_카드재작성_2026-07-17.md`의 "배치E 인벤토리(2026-07-17 완료)" 섹션 참고.
- [ ] **배치E — 신규 유닛 카드 작성 (인벤토리 완료, 작성 미착수):** 새 검사카드 17장 + 새 테크닉카드 47장 = 총 64장 신규 필요. 최종 목표 총 카드 수 **약 93장**(29+64) — 이것도 바닥값(floor estimate)이며, 4-15/4-17.txt에 "이미 배운 것으로 전제하고 넘어간" 근육명(예: 대퇴직근)이 있어 더 늘어날 수 있음.
- [ ] F3(손 위치·압력 방향 이미지 오버레이)는 여전히 텍스트만 있고 미구현 — 소책자(스캔 이미지, OCR 불가)에서 그림만 추출하는 방법을 시도할지 여부 확인 필요.
- [ ] 체인 링크(의심근육→/연관검사→) 자동 연결 해상률 재계산 — 이전 측정값 81%(카드 29장 기준), 93장 기준으로 재확인 필요.
