# AI 회사 헌장 (Company Charter)

> 이 문서는 AI직원 플랫폼과 모든 도메인 스킬이 참조하는 최상위 정의입니다.
> 여기 없는 결정은 개별 스킬이 임의로 내리면 안 되고, 이 문서를 먼저 업데이트합니다.

## 1. 정체성 — 우리는 뭘 하는 회사인가
- 한 문장 정의: **소상공인 실무를 AI가 대신 처리하되, 한 번 쓰고 버리는 결과물이 아니라
  같은 종류의 결과물을 계속 찍어내는 '공장'을 만드는 서비스**
- 지금 단계: **셀프 사용** (산출물만 일부 배포 예정, 플랫폼 자체는 비매)
- 다루는 도메인: 정부지원사업, 꽃집 실무, 방역사업 — 계속 추가될 리스트
- 작업 철학 (첨부문서 반영): 특정 사례에 고정하지 않고, 매번 현재 조건(공고문/기준/양식 등)을
  먼저 확인한 뒤 판단하는 **범용 작업 방식**. 결과물을 예쁘게 만드는 것보다
  **점수/성공 가능성이 있는 방향을 찾고, 위험을 먼저 지적하는 것**을 우선한다.

## 2. AI직원이란 무엇인가
- 정의: **각 도메인 스킬(공장)을 실행하고, 스스로 "심사위원 모드"로 검수까지 하는 자동화 실행 단위**
  - 첨부문서의 "Gemini식 확장성 + Monday식 방어성을 함께 쓴다"는 원칙을 그대로 채택:
    AI직원은 초안을 만드는 역할과, 그 초안을 비판적으로 재검토하는 역할을 순차적으로 둘 다 수행
- 할 수 있는 것:
  - 스킬 실행, 초안 작성, 자기검수(심사위원 모드), 위험 표현 감지, 진행상황 보고
- 할 수 없는 것:
  - 최종 산출물을 외부(관공서/고객/거래처)에 실제로 제출·발송·게시하는 것 (사람 확인 후)
  - 결제, 계약 체결, 법적 판단이 필요한 최종 확정

## 3. 권한 경계선 (3단계) — 스킬 설계 시 반드시 이 기준을 따름
*(2번 질문 답변이 "모름"이었으므로 아래는 첨부문서 내용 기반 초안 — 실제 사용하며 수정)*

| 등급 | 설명 | 예시 |
|---|---|---|
| 자동실행 가능 | 사람 확인 없이 바로 실행 | 공고/조건 분석, 아이디어 후보 비교, 초안 작성, 위험 표현 감지, 콘텐츠 초안 변환 |
| 승인 필요 | 결과를 사람이 확인 후 다음 단계 진행 | 최종 제출본 확정, 예산표 확정, 발표자료 확정, 고객에게 실제 발송되는 산출물 |
| 절대 자동화 금지 | 이 회사는 이걸 자동화하지 않음 | 관공서·고객에게 실제 제출/발송 행위 자체(클릭/전송), 법적 자문이 필요한 최종 판단, 결제·계약 체결 |

## 4. 성장 경로
- 지금: 나 혼자 사용하며 3개 도메인(정부지원/꽃집/방역) 스킬 구축 중
- 배포 방식: **플랫폼이 아니라 산출물만 배포** — 꽃집·방역부터 시작, 정부지원사업은 배포 대상 아님(개인 사용)
- 판매: 당장 없음
- SaaS 전환 조건: 아직 미정 — 산출물 배포 반응을 본 뒤 재논의

## 5. 이 헌장을 다시 써야 하는 조건
- 산출물 배포(꽃집/방역)를 실제로 시작할 때
- 정부지원사업 스킬에 발표자료/발표연습/콘텐츠변환 같은 하위 공정이 실제로 추가될 때
  - **(2026-07-16 발동·확인 완료)** 정부지원 뒷단 발표자료 공정(PPT초안·발표대본·예상Q&A) 설계 확정([[ai공장짓기/decision-log_skill-factory-architecture|decision-log]] §17). 승인 게이트 2곳(PPT승인/발표패키지승인), 파이프라인은 저장까지만 수행하고 실제 발표 현장 사용·제출은 반드시 사람 — 위 §3 기존 규정("발표자료 확정 = 승인 필요", "실제 제출·발송 행위 자체 = 절대 자동화 금지")과 합치 확인됨. 헌장 재작성 없이 이 트리거는 종결.
- 권한 경계선 3단계 중 하나라도 실제 사용 중 문제가 생겨 재조정이 필요할 때
- 셀프→배포 확장 시점 도달 시
